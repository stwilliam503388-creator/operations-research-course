## 这份教程是什么

这是一份给**有高中数学基础、会写 Python、但对运筹学完全陌生**的技术人写的自学教程。它不讲虚的——每一章都围绕「你为什么需要知道这个」和「这个东西到底在干什么」展开。读完以后，你不一定会成为运筹学专家，但你一定会：

- 跟同事解释清楚「MIP 求解器是干什么的、里面怎么工作的」
- 用 Python + 求解器把工作中的优化问题建模并求解
- 看懂 Gurobi/COPT 的求解日志，知道什么时候该停、什么时候该调参

## 术语速查表

下面的表是你翻任何一章时会反复回来查的东西。每个术语只有一句话解释——够你「认出来」就行，不需要背。

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 决策变量 | Decision Variable | Excel 表里你要填的那个格子——你要决定的东西 |
| 目标函数 | Objective Function | 你想让什么东西最大或最小 |
| 约束 | Constraint | 现实不允许你做的事情——预算上限、时间上限、物理定律 |
| 可行解 | Feasible Solution | 满足所有约束的一组合法答案 |
| 最优解 | Optimal Solution | 所有合法答案里，目标值最好的那个 |
| LP 松弛 | LP Relaxation | 假装整数不存在，先算个大概——像做预算先不算零头 |
| 对偶 | Duality | 同一个问题从相反角度看——买方视角 vs 卖方视角 |
| 分支定界 | Branch & Bound | 猜谜游戏——每次二分，不对就回头 |
| 割平面 | Cutting Plane | 切掉迷惑人的假答案，不碰任何真答案 |
| 预处理 | Presolve | 求解器动手前的自动瘦身——像编译器的优化 pass |
| 启发式 | Heuristic | 不求最优但求快——不求最好但求还行 |
| 上界 | Upper Bound / Primal Bound | 目前找到的最好答案的值 |
| 下界 | Lower Bound / Dual Bound | 最优答案不可能低于这个值 |
| Gap | MIP Gap | 你现在离真正的第一名还差百分之几——0% 就是追平了 |
| NP-hard | NP-hard | 计算机「天生」算不快的问题——但可以聪明地绕过去 |
| 影子价格 | Shadow Price | 多一个单位资源你能多赚多少钱 |
| 对称性 | Symmetry | 两个方案本质上一样——求解器在重复劳动 |
| Big-M | Big-M | 一个「足够大但别太大」的数——建模的万能胶水也是万恶之源 |
| 回调 | Callback | 求解过程中插入你自己的代码——比如动态加约束 |
| 分支策略 | Branching Strategy | 先猜哪个变量的值——猜得准能省 90% 时间 |

> 读到任何一个术语忘了什么意思，回到这里看「一句话人话」列，3 秒搞定。

### 概念边界：求解器最优不等于业务最优

MIP 求解器报告 `optimal` 时，严格含义是：**在你写下的变量、约束、目标函数和输入数据下，找到了数学最优解**。它不能自动保证：

- 输入数据真实、完整、没有延迟。
- 业务约束都被建进模型，例如人工规则、服务承诺、政策限制。
- 目标函数真的代表业务偏好，例如只最小化成本可能牺牲稳定性。
- 方案在未来需求变化、参数误差或执行偏差下仍然稳健。

所以工业建模通常还要做数据校验、极端场景测试、灵敏度分析和业务复核。求解器是强大的计算工具，不是替代建模判断的黑盒。

## 环境准备

本节所有案例用 **HiGHS**（免费开源，`pip install` 即用）作为默认求解器。如果你有 Gurobi 或 COPT 的许可，可以直接替换——性能更强。

```bash
# 1. 创建虚拟环境
python3 -m venv mip-env
source mip-env/bin/activate

# 2. 进入课程目录（从仓库根目录）
cd "04-MIP求解器技术"

# 3. 安装依赖
pip install -r code/requirements.txt
```

`code/requirements.txt` 内容：
```
highspy
numpy
pandas
matplotlib
```

可选（需单独申请许可）：
- **Gurobi**：去 [gurobi.com](https://www.gurobi.com/downloads/) 申请免费学术许可 → `pip install gurobipy`
- **COPT**：去 [shanshu.ai/copt](https://www.shanshu.ai/copt) 申请免费学术许可 → `pip install coptpy`

> ⚠️ 本教程所有案例均可改用 HiGHS 跑通。遇到 Gurobi 代码的地方，案例文件里会标注 `# 将 Gurobi 替换为 HiGHS 的写法`。

C++ 对照案例默认不调用商业求解器，只演示求解器日志与分支定界的核心思想：

```bash
g++ -std=c++17 -O2 code/cpp/case04_branch_bound_knapsack.cpp -o /tmp/case04_branch_bound_knapsack
/tmp/case04_branch_bound_knapsack
```
