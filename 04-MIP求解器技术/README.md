> 一个周末，从零到能自己写 MIP 模型解决问题。

## 这份教程是什么

这是一份给**有高中数学基础、会写 Python、但对运筹学完全陌生**的技术人写的自学教程。它不讲虚的——每一章都围绕「你为什么需要知道这个」和「这个东西到底在干什么」展开。读完以后，你不一定会成为运筹学专家，但你一定会：

- 跟同事解释清楚「MIP 求解器是干什么的、里面怎么工作的」
- 用 Python + 求解器把工作中的优化问题建模并求解
- 看懂 Gurobi/COPT 的求解日志，知道什么时候该停、什么时候该调参

## 术语速查表

下面的表是你翻任何一章时会反复回来查的东西。每个术语只有一句话解释——够你「认出来」就行，不需要背。

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 决策变量 | Decision Variable | 像 Excel 里真正要你填的格子：派哪辆车、排哪台机器、买多少货 |
| 目标函数 | Objective Function | 像打分规则：你到底想省钱、提利润、少迟到，还是几个目标一起折中 |
| 约束 | Constraint | 像现实世界给方案画的红线：预算不能超、车不能超载、人不能一天干 30 小时 |
| 可行解 | Feasible Solution | 先别管好不好，只要不违规、不越界、能执行，就是一个可行解 |
| 最优解 | Optimal Solution | 在所有不违规的方案里，按你的打分规则拿第一名的那个 |
| LP 松弛 | LP Relaxation | 先把“必须是整数”这件事放一放，让求解器先看清大方向 |
| 对偶 | Duality | 像从资源价格反过来看问题：不是问怎么排方案，而是问资源到底值多少钱 |
| 分支定界 | Branch & Bound | 像查字典找答案：先分叉缩小范围，发现这一大片不可能更好就整片划掉 |
| 割平面 | Cutting Plane | 像用尺子切掉明显不靠谱的区域，只删假答案，不删真正可行的好答案 |
| 预处理 | Presolve | 像开工前先收拾桌面：删掉废变量、合并重复条件，让后面少算冤枉路 |
| 启发式 | Heuristic | 像先找一个“还不错能用”的方案救急，不保证冠军，但通常来得很快 |
| 上界 | Upper Bound / Primal Bound | 对最小化问题来说，像目前手里已经拿到的最好成绩，至少知道能做到这么好 |
| 下界 | Lower Bound / Dual Bound | 像理论上的底线：再怎么努力，最优值也不可能比它更低 |
| Gap | MIP Gap | 像离冠军还有多大差距：Gap 越小，越能放心说“差不多就是最优了” |
| NP-hard | NP-hard | 像迷宫规模一大就爆炸的问题：不是电脑笨，而是组合可能性太多 |
| 影子价格 | Shadow Price | 像多给你一小时工时或一辆车，目标值大概能改善多少 |
| 对称性 | Symmetry | 像两辆完全一样的车互换编号，业务上没区别，但求解器可能重复想两遍 |
| Big-M | Big-M | 像建模里的大锤子：能把开关逻辑砸进去，但太大会把模型砸得很难算 |
| 回调 | Callback | 像求解器跑到一半喊你插话：你可以趁机加约束、砍坏解或记录信息 |
| 分支策略 | Branching Strategy | 像猜谜先猜哪个关键线索：猜对了搜索树会小很多，猜错了就绕远路 |

> 读到任何一个术语忘了什么意思，回到这里看「一句话人话」列，3 秒搞定。

### 概念边界：求解器最优不等于业务最优

MIP 求解器报告 `optimal` 时，严格含义是：**在你写下的变量、约束、目标函数和输入数据下，找到了数学最优解**。它不能自动保证：

- 输入数据真实、完整、没有延迟。
- 业务约束都被建进模型，例如人工规则、服务承诺、政策限制。
- 目标函数真的代表业务偏好，例如只最小化成本可能牺牲稳定性。
- 方案在未来需求变化、参数误差或执行偏差下仍然稳健。

所以工业建模通常还要做数据校验、极端场景测试、灵敏度分析和业务复核。求解器是强大的计算工具，不是替代建模判断的黑盒。

## 环境准备

本节所有案例用 **HiGHS**（免费开源，`pip install` 即用）作为默认求解器。如果你有 Gurobi 或 COPT 的许可，可以直接替换——性能更强。

```bash
# 1. 创建虚拟环境
python3 -m venv mip-env
source mip-env/bin/activate

# 2. 进入课程目录（从仓库根目录）
cd "04-MIP求解器技术"

# 3. 安装依赖
pip install -r code/requirements.txt
```

`code/requirements.txt` 内容：
```
highspy
numpy
pandas
matplotlib
```

可选（需单独申请许可）：
- **Gurobi**：去 [gurobi.com](https://www.gurobi.com/downloads/) 申请免费学术许可 → `pip install gurobipy`
- **COPT**：去 [shanshu.ai/copt](https://www.shanshu.ai/copt) 申请免费学术许可 → `pip install coptpy`

> ⚠️ 本教程所有案例均可改用 HiGHS 跑通。遇到 Gurobi 代码的地方，案例文件里会标注 `# 将 Gurobi 替换为 HiGHS 的写法`。

C++ 对照案例默认不调用商业求解器，只演示求解器日志与分支定界的核心思想：

```bash
g++ -std=c++17 -O2 code/cpp/case04_branch_bound_knapsack.cpp -o /tmp/case04_branch_bound_knapsack
/tmp/case04_branch_bound_knapsack
```

## 学习验收标准

学完本课程后，建议用下面 5 条自检：

- 能把业务约束翻译成变量、目标函数和约束，而不是先写代码再猜模型。
- 能解释 LP 松弛、分支定界、割平面和 MIP Gap 分别在求解器里起什么作用。
- 能说明 `optimal` 只代表当前模型内最优，不代表真实业务一定可执行。
- 能识别 Big-M 过大、变量对称、缺少子环消除等常见建模风险。
- 能运行 Python MIP 案例和 C++ 分支定界对照案例，并解释界估计为什么能剪枝。
