## 1.1 没有优化之前，世界是什么样子的

先想象一个场景。

你是一家物流公司的调度经理。你有 50 辆货车，每辆载重 5 吨。今天有 300 个配送点等着发货。每个点有不同重量、不同地址、不同时间窗——有些必须在上午 10 点前送到，有些下午 4 点以后才可以。你的目标很简单：用最少的油、最短的时间，把所有货送到。

你怎么做？传统做法是找个经验最丰富的老调度，对着地图和 Excel 表，从早上 8 点排到晚上 10 点。他排完以后，你会发现：一天过去了，他只排了大概 150 个点。剩下的一半——要么加班，要么加车，要么跟客户道歉改明天。

这不是某个调度员的问题。这是**人类大脑在面对组合爆炸时的天然瓶颈**。300 个点的排列方式，比宇宙中的原子还多。任何一个靠「经验」和「直觉」来做这类决策的人，无论多资深，都只是在「差不多就好」和「完全没法算」之间找一个折中。

### 从直觉到数学：三个概念，像做饭一样简单

数学优化听起来唬人，但它本质上只关心三件事。我用一顿饭来类比：

| 概念 | 做饭类比 | 正式定义 |
|------|----------|----------|
| *决策变量* | 买什么菜？做几道菜？每道做多少？ | Excel 里你要填的那个格子——它是你还不知道、需要你决定的量 |
| *约束* | 冰箱只能装 10 样东西。你只有 2 小时。儿子不吃辣。总预算 200 块。 | 现实世界不允许你做的事情——物理限制、时间限制、资源限制、规则限制 |
| *目标函数* | 让全家吃饱且不超预算——或者花最少的钱做到「能吃饱」 | 你想要最大化或最小化的那个东西 |

任何一个优化问题，拆开来看就这三个要素。你工作中遇到的也是这样——你要决定什么（变量）、你不能做什么（约束）、你想达到什么（目标）。没有第四个。

### 连续 vs 离散：水量和人数的区别

想象你在调淋浴水温。你可以把旋钮拧到 37.5°C、37.8°C、38.1°C——任何一个小数都行。这就是**连续变量**（Continuous Variable），你可以任意微调。

但如果你在排班——一个人要么上班，要么不上班。你不能派「0.7 个人」去干活。这就是**离散变量**（Discrete Variable），它只能取特定的、一个个跳着的值。

更常见的离散变量是整数：你不能生产 2.7 台发动机，不能建 0.4 座仓库，不能让 3.2 个工人加班。真实世界充满「整数」要求。

### LP → IP → MIP：从线性到整数

这是很多人第一次接触这个概念时最容易晕的地方。我们用一张 ASCII 图把它讲清楚：

```
         ┌──────────────────────────────┐
         │          LP (线性规划)          │
         │   所有变量都是连续的              │
         │   可以取任意实数                  │
         │    ┌──────────────────────┐    │
         │    │    MIP (混合整数规划)    │    │
         │    │  一部分变量是连续的，     │    │
         │    │  一部分变量必须是整数     │    │
         │    │  ┌──────────────┐    │    │
         │    │  │ IP (整数规划)  │    │    │
         │    │  │ 所有变量都是   │    │    │
         │    │  │ 整数          │    │    │
         │    │  └──────────────┘    │    │
         │    └──────────────────────┘    │
         └──────────────────────────────┘
```

- **LP**（Linear Programming，线性规划）：所有变量连续。求解很快。这是基础。
- **IP**（Integer Programming，整数规划）：所有变量必须是整数。求解可能很慢。这是 LP 的子集。
- **MIP**（Mixed Integer Programming，混合整数规划）：一部分连续，一部分整数。这才是真实世界的形态——你可能同时需要决定「建不建仓库」（整数）和「仓库里存多少货」（连续）。MIP 是 IP 的超集。

一句话总结：**MIP = 有些变量必须是整数的线性规划。这个「有些」让问题从简单变得极难，但也让它能描述几乎所有真实世界的决策问题。**

> 🆘 **逃生通道**：如果你感觉这一节信息量太大——只需要记住三件事：*决策变量*、*约束*、*目标函数*。这就是全部的数学优化。后面所有章节都在围绕这三件事展开。把这三个词刻在脑子里，你已经比 90% 的入门者更清楚自己在干什么了。

## 1.2 一段八卦：MIP 是怎么从一个数学游戏变成千亿美元产业的

你不需要记住每个年份和名字。但你需要知道一件事：MIP 不是某个大学教授在办公室里捣鼓出来的学术玩具。它的每一步发展，都有人砸了真金白银。

### 1947：一个被当成课后作业的天才发明

故事要从乔治·丹齐格（George Dantzig）说起。1947 年，他在加州大学伯克利分校读博士。有一天他上课迟到，看到黑板上写着两道题，以为是课后作业，抄下来回去做了。几天后他交上去，跟教授道歉说「这次的作业有点难，我多花了几天」。

几周后，教授冲到他家门口，激动地告诉他：那两道题不是作业——是统计学领域两个当时还没人证出来的未解问题。而丹齐格把它们当作业做完了。

这个迟到生后来去了 RAND 公司（美国军方智库），在解决军事后勤调度问题时，「顺便」发明了**单纯形法**（Simplex Method）。这个方法至今仍是 MIP 求解器最核心的发动机。

生活类比：你老板丢给你一个没人能搞定的烂摊子，你以为就是日常任务，结果搞完以后发现全公司在给你鼓掌。

工作类比：很多时候你以为的「日常」，其实是别人眼里的「不可能」。带着这种心态去面对问题，往往能做出意想不到的结果。

### 1954：三个人证明了一个灾难

丹齐格的单纯形法问世后，大家以为「数学优化」问题基本解决了——无非就是算力不够嘛，等计算机再快一点就好了。

然后旅行商问题（Traveling Salesman Problem, TSP）给了所有人一记耳光。问题本身简单到小学生都能理解：一个推销员要去 N 个城市，每个城市去一次，怎么走总路程最短？1954 年，Dantzig、Fulkerson 和 Johnson 用当时最先进的方法求解了 49 个城市的 TSP，需要手工计算数周。更重要的是，他们初步认识到了这类问题的组合本质——问题规模每增加一个城市，可能的路线数量就会爆炸式增长。

后来人们才知道，TSP 属于 *NP-hard* 问题——这意味着**不存在已知的多项式时间算法**，而且很可能永远不会有。这不是「现在算得慢」，而是「问题的本质决定了它难」。

### 1960：分支定界——猜谜游戏的开端

1960 年，Land 和 Doig 提出了*分支定界*（Branch & Bound）框架。这个思想的精妙之处在于：它不是直接求解整数问题，而是通过不断「猜」整数变量的值，每一次猜完就验证一下——如果猜错了，就回头换一个答案；如果猜的方向一看就不可能更好了，就直接剪掉不走了。

生活类比：你在一个巨大的迷宫里找出口。到每个岔路口，你选一条路走。走了几步发现是死胡同，就退回来换另一条。如果某条路上已经看到前面是悬崖了，就别浪费体力了。

### 1984：AT&T 的「另一条路」

1984 年，AT&T 贝尔实验室的一个年轻数学家 Narendra Karmarkar 发表了*内点法*（Interior Point Method）。他没有像单纯形法那样在多面体的边界上「沿着边走」，而是直接从内部穿过去。

AT&T 意识到这个算法可以帮他们优化长途电话网络路由，于是申请了专利——这在当时的学术界引起了巨大争议（数学家申请算法专利？），也从侧面说明这个东西的商业价值有多惊人。

### 1990s 之后：求解器变成军火商

从 1990 年代开始，MIP 不再只是学术界的事情。几家公司先后入场，把 MIP 求解器变成了一项利润丰厚的生意：

- **CPLEX**（1990s）：最早商业化的 MIP 求解器之一，被 IBM 收购后成为行业标杆。很多大企业的供应链系统底层跑的就是它。
- **Gurobi**（2008）：CPLEX 的核心开发团队出走创业。现任 CEO 是单纯形法领域的世界级专家。Gurobi 现在是求解器市场的绝对领导者——每年授权费可以从几万到几十万美元不等。
- **SCIP**（2000s）：柏林 ZIB 研究所开发的开源求解器。非商业用途免费，是学术界的灯塔项目。
- **COPT**（2019）：中国团队在杉数科技开发的商业求解器。从 2019 年发布到现在，性能已经进入全球第一梯队——这件事放在十年前没人敢信。
- **HiGHS**（2020s）：爱丁堡大学开发的开源求解器，性能不俗且完全免费，是本教程的默认求解器。

### 关键时间线

```
1947    1954      1960        1984      1990s     2008      2019     2020s
 │       │         │           │          │         │         │        │
 ▼       ▼         ▼           ▼          ▼         ▼         ▼        ▼
单纯形法  TSP     分支定界    内点法     CPLEX     Gurobi    COPT    HiGHS
 (Dantzig) NP-hard (Land-Doig) (Karmarkar) 商业化   创业      中国突围  新生代
```

**为什么要讲这段历史？** 因为 MIP 不是什么「也许未来有用」的前沿研究。过去 70 年里，工业界在求解器上砸了上百亿美元——原因只有一个：他们真的需要这东西。联邦快递用它规划航线、亚马逊用它调度仓储、国家电网用它调配电力。它不是玩具，它是工具。

> 🆘 **逃生通道**：这一节没有任何技术考点。你只需要记住一个结论：工业界拼命砸钱造求解器，不是因为数学家闲得慌，而是因为**不优化就亏钱**。当你的车队规模到 100 辆、配送点到 500 个的时候，人工调度已经不可能了——那多出来的 10% 成本，就是 MIP 的市场。

## 1.3 数学工具箱：你需要的不是证明，是直觉

先说清楚：这一节不会教你怎么证明定理，也不会推导任何公式。如果你需要严格的数学——任何一本运筹学教材都能满足你。这里的目的是让你**建立起直觉**：这些概念大概在说什么，以及当你看到求解器日志里出现这些词的时候，你大概知道它在干什么。

### 多面体与极点：橡皮筋的故事

把所有的*约束*想象成很多面墙。这些墙围起来形成一个多面体（Polytope）——可行域（Feasible Region）。在这个多面体内部的每一点，都是「合法的」。

现在想象你在这个多面体外面套一根橡皮筋，沿着某个方向绷紧它。橡皮筋最终会「崩」到多面体的哪一个位置？它一定是停在某个尖角上——那个尖角就是**极点**（Extreme Point）。

为什么这很重要？因为单纯形法的核心操作就是：在多面体的极点之间跳跃。它不去搜索内部的所有点——它只在「角」上跳。这是一个极其聪明的偷懒。

生活类比：你在一间有很多角落的房间找最亮的那个位置（比如靠窗的角落）。你不会在房间正中央一寸一寸地挪——你直接去每个角落看一眼，哪个最亮站哪。

工作类比：你面对 100 个方案时，不需要逐个评估。先把那些「显然不可能更好」的方案排除——那些不在「角」上的方案（即在约束内部、没有推到极限的），永远不可能最优。

### 对偶理论：同一枚硬币的两面

这是本章最难的概念。如果没有完全理解——没关系，先记住一句话：**对偶 = 同一个最优化的两种等价描述**。

**生活类比**：你开了一家奶茶店。你想定一个价格表——珍珠奶茶卖多少、椰果奶茶卖多少、每种配料怎么收费——使得总收入最大化（给定顾客的购买力）。与此同时，你的竞争对手也在开奶茶店。他想定一个「竞价策略」——他愿意为每克珍珠付多少钱、每毫升牛奶付多少钱——使得他愿意出的总价最小化，但又要高到足够把你手里的原料都买走。

神奇的事情是：**在你定价最优的那一刻，你的最大化收入正好等于他出的最小化竞标价。** 这两个数字完美相等。这就是*对偶*（Duality）的核心直觉。

**工作类比**：你团队有个项目，可以内部做，也可以外包。内部的成本是「我做这件事最少要多少人、多少天」。外部报价是「市场上做这件事最少要多少钱」。如果内部成本 > 市场价，就应该外包。如果内部成本 < 市场价，就应该自己做。当两者相等的时候——你正好在最优决策的边界上。这就是*影子价格*（Shadow Price）的直觉：多一个单位的资源，你到底愿意为它付多少钱。

在 MIP 求解中，对偶之所以重要，是因为它给求解器提供了「进度条」：每次计算最优解时，求解器同时维护一个*上界*（Primal Bound，找到的最好答案）和一个*下界*（Dual Bound，不可能低于这个数）。当这两个数越来越近、直到相等——求解完成。你的 Gap 降到 0%。

### NP 困难：不是锁打不开，是钥匙不存在

这是被误解最多的概念。很多人听到 *NP-hard* 的反应是「那没法算了」——不对。

**生活类比**：你不是打不开这把锁——而是**全宇宙的计算机加在一起，也没有一把通用钥匙能快速打开所有这种锁**。但这不妨碍你请一个熟悉这个品牌锁的锁匠来开——他知道这把锁的结构特点，可以绕开一些通用的低效尝试。

**工作类比**：MIP 的 *NP-hard* 意味着：没有一种「万能解法」能对所有的 MIP 问题都快速求解。但这不代表你的具体问题没法解。关键在于——**你会不会根据你的问题结构，挑选合适的技术**。有些问题虽然理论上是 NP-hard，但实际上有特殊结构（比如网络流），可以用很巧妙的方法秒解。

*NP-hard* 告诉你的不是「放弃」，而是「别指望万能药」。

### 松弛：先拿基础分，再冲附加题

**生活类比**：你考试时拿到一张卷子。大题很难，但选择填空题你都会。聪明做法：先跳过大题，把所有小题做完拿满基础分（可能已经 70 分了），然后再回头攻克大题。这就是*LP 松弛*（LP Relaxation）——先假装整数约束不存在，求一个「轻松版」的最优解。虽然这个解可能不是合法整数解（就像基础分还没到 100），但它给你标定了一个天花板：最优的整数解，绝不比这个「轻松版」更好。

**工作类比**：你在做年度预算规划。精确的数字需要跟每个部门反复确认。但你老板下午就要一个大数。你怎么办？先去掉最难满足的约束（比如跨部门调拨的限制），算出一个大概的数。这个数不精确，但它给你一个方向感。然后再回头把约束一个一个加回去，逐步逼近最终方案。

求解器做 MIP 时就是这样——先做 *LP 松弛*，拿一个乐观估计作为基准，然后通过*分支定界*一步步把整数约束「加回去」，直到找到合法且最优的整数解。

> 🆘 **逃生通道**：如果你只记住四个概念中的一个——选*对偶*。在本节中，「对偶 = 同一个最优化的两种等价描述」这句话就足够了。进度条（Gap）= 上界和下界的差距。Gap 到 0%，你找到了最优解。Gap 很大，继续算。其他三个概念在后面的章节中都会反复出现，到时候再理解也不晚。

## 1.4 六大经典模型：你工作中遇到的所有优化问题都长得像它们

下面的六个模型，不是让你背的。它们是「模板」——当你面对一个新问题时，你大概率会发现它可以归结为这六个中的某一个（或某几个的组合）。每种模型我们都用同一个结构来讲：场景 → 表格 → 公式（逐符号注释）。

---

### 模型一：分配问题

**难度：★☆☆☆☆**

**场景**：你有 5 个工人和 5 个任务。每个工人做每个任务的效率不一样（有人擅长搬货，有人擅长点数）。你的目标是——把任务分配给工人，使得总工时最少，且每个工人只做一个任务，每个任务只分配给一个工人。

| 要素 | 内容 |
|------|------|
| 决策变量 | x_{ij} ∈ {0,1}：工人 i 是否被分配去做任务 j（是=1，否=0） |
| 约束（1） | 每个工人只做一个任务：对每个 i，∑_j x_{ij} = 1 |
| 约束（2） | 每个任务只被一个人做：对每个 j，∑_i x_{ij} = 1 |
| 目标 | 最小化总工时：min ∑_i ∑_j (c_{ij} × x_{ij})，其中 c_{ij} 是工人 i 做任务 j 的耗时 |

**数学模型（逐行注释）**：

```
min  Σ_i Σ_j  c_{ij} · x_{ij}       ← 总耗时越小越好，c_{ij}是已知的每对耗时
s.t. Σ_j  x_{ij}  =  1   ∀ i        ← 每个工人 i 只能分配一个任务（∀=对所有）
     Σ_i  x_{ij}  =  1   ∀ j        ← 每个任务 j 只能由一个工人做
     x_{ij}  ∈  {0, 1}   ∀ i, j     ← 只能选「是」或「否」，不能用分数个人
```

**典型规模**：n = 100 个工人/任务（10,000 个变量），求解器通常几秒内出解。

---

### 模型二：背包问题

**难度：★☆☆☆☆**

**场景**：你要去露营。背包最多装 15 公斤。有 10 样东西可以带——帐篷（5kg，重要度 8）、睡袋（3kg，重要度 6）、炉头（1kg，重要度 3）……每样东西装或不装，不能装半个。你的目标：在不超过 15 公斤的前提下，让带的东西总「重要度」最高。

| 要素 | 内容 |
|------|------|
| 决策变量 | x_i ∈ {0,1}：第 i 样东西带不带 |
| 约束 | 总重量不能超过背包容量：∑_i (w_i × x_i) ≤ W，其中 w_i 是物品重量，W 是容量 |
| 目标 | 最大化总重要度（价值）：max ∑_i (v_i × x_i) |

```
max  Σ_i  v_i · x_i                   ← 总重要度越高越好，v_i 是每样东西的价值
s.t. Σ_i  w_i · x_i  ≤  W             ← 总重量 ≤ 背包容量，w_i 是重量，W 是上限
     x_i  ∈  {0, 1}   ∀ i             ← 要么带，要么不带，不存在半个帐篷
```

**典型规模**：n = 10,000 个物品，求解器秒解（背包有高效的动态规划算法）。

---

### 模型三：集合覆盖

**难度：★★☆☆☆**

**场景**：你要在 N 个候选位置建消防站。每个候选位置能覆盖周边一定范围内的居民区。你的目标是——用最少的消防站，确保每个居民区都至少被一个消防站覆盖。

| 要素 | 内容 |
|------|------|
| 决策变量 | x_j ∈ {0,1}：是否在位置 j 建消防站 |
| 约束 | 每个居民区 i 至少被一个建了的消防站覆盖：∑_{j: j 覆盖 i} x_j ≥ 1 |
| 目标 | 最小化建站数量：min ∑_j x_j |

```
min  Σ_j  x_j                         ← 消防站越少越好（或可加建站成本 c_j）
s.t. Σ_{j: j覆盖i}  x_j  ≥  1  ∀ i    ← 每个居民区 i 至少被一个覆盖
     x_j  ∈  {0, 1}   ∀ j             ← 建或不建，没有半个消防站
```

**典型规模**：候选位置数百个，居民区数千个。求解有一定难度但工业级求解器可处理。

---

### 模型四：设施选址

**难度：★★☆☆☆**

**场景**：你要在几个城市建仓库。每个候选城市建仓库有固定成本（租金+人力），从仓库到每个客户有运输成本。你的目标：选哪些城市建仓库、每个仓库服务哪些客户，使得总成本（建仓成本 + 运输成本）最小。

| 要素 | 内容 |
|------|------|
| 决策变量 | y_j ∈ {0,1}：是否在城市 j 建仓库；x_{ij} ∈ [0,1]：客户 i 从仓库 j 获得的比例 |
| 约束（1） | 每个客户的需求必须被满足：∑_j x_{ij} = 1 |
| 约束（2） | 只能从建了的仓库发货：x_{ij} ≤ y_j |
| 目标 | 最小化总成本：min ∑_j (f_j × y_j) + ∑_i ∑_j (c_{ij} × x_{ij}) |

```
min  Σ_j f_j·y_j  +  Σ_i Σ_j c_{ij}·x_{ij}    ← 建仓固定成本f_j + 运输成本c_{ij}
s.t. Σ_j x_{ij}  =  1   ∀ i                     ← 每个客户的需求100%满足
     x_{ij}  ≤  y_j   ∀ i, j                    ← 没建仓库就不能从那里发货
     y_j  ∈  {0, 1}                              ← 建或不建
     x_{ij}  ≥  0                                ← 发货比例不能是负数
```

**典型规模**：候选仓库 50 个，客户 1,000 个。这是一个经典的 *Big-M* 可能出现的地方（以后会讲为什么要注意）。

---

### 模型五：生产排程

**难度：★★★☆☆**

**场景**：你有一条生产线，产品切换时需要清洗设备（产生切换时间和成本）。未来一周有 5 种产品要生产，每种有各自的订单量和截止时间。你的目标：决定生产顺序和每种生产多少，使得总切换成本 + 库存成本最小，同时不超截止时间。

| 要素 | 内容 |
|------|------|
| 决策变量 | x_{pt} ≥ 0：产品 p 在第 t 时段的生产量；y_{pt} ∈ {0,1}：时段 t 是否在生产产品 p；z_{pqt} ∈ {0,1}：时段 t 是否从产品 p 切换到 q |
| 约束（1） | 每个时段只能生产一种产品 |
| 约束（2） | 每时段产量不超过设备产能 |
| 约束（3） | 库存 + 生产 - 需求 = 新库存（物料平衡） |
| 目标 | 最小化切换成本 + 库存持有成本 |

```
min  Σ切换成本  +  Σ库存持有成本
s.t. 产能约束：每次只能做一种，量不超过上限
     物料平衡：上期库存 + 本期生产 - 本期需求 = 本期库存
     截止时间：每产品的累计生产 ≥ 累计需求（在截止时间前）
     y 和 z 为 0/1，x ≥ 0
```

**典型规模**：5 种产品 × 7 天 = 35 个时段。中小规模，但逻辑比前几个复杂，是真实工厂问题的简化版。

---

### 模型六：网络流

**难度：★★☆☆☆**

**场景**：你管一个国家电网的输电线路。每个城市有的是发电端（供应节点），有的是用电端（需求节点），中间是传输线路（每条线有最大传输容量和单位传输成本）。你的目标：以最小成本把电从发电端送到用电端，满足所有需求，不超任何线路容量。

| 要素 | 内容 |
|------|------|
| 决策变量 | f_{ij} ≥ 0：从节点 i 到节点 j 的流量（传输了多少单位的电） |
| 约束（1） | 流量守恒：每个中间节点，流入 = 流出 |
| 约束（2） | 线路容量：f_{ij} ≤ cap_{ij} |
| 约束（3） | 供应节点总流出 ≤ 供应量；需求节点总流入 ≥ 需求量 |
| 目标 | 最小化总传输成本：min ∑_{ij} (cost_{ij} × f_{ij}) |

```
min  Σ_{ij} cost_{ij}·f_{ij}              ← 总传输成本最小
s.t. Σ_j f_{ij} - Σ_k f_{ki} = b_i  ∀ i   ← 流出-流入=净供应量b_i（b_i>0是供应，<0是需求）
     0 ≤ f_{ij} ≤ cap_{ij}   ∀ i,j        ← 流量不能为负，不能超线路容量
```

**典型规模**：数千个节点，数万条线路。网络流有特殊结构，即使规模很大也可以极高效求解——这是 MIP 中最「好说话」的一类问题。

---

> 🆘 **逃生通道**：你不需要记住全部六个模型。选**两个最贴近你工作场景的**仔细读一遍变量设定和目标，其他的扫一眼场景描述就行。关键是建立「这个问题好像长得像 XX 模型」的直觉——以后要用的时候，回来查模板。

## 1.5 建模心法：拿到新问题，五分钟写出第一个模型

这不是「每个问题长什么样」——那是 1.4 干的事。这一节教的是：**你面对一片空白的屏幕，从哪开始落笔。**

### 第一步：画图（最重要，花 2 分钟）

别急着写公式。拿一张白纸——或者打开 draw.io、Excalidraw——把你要优化的场景画出来。画什么？

- **实体**（方框）：卡车、仓库、客户、产品、工人、机器。画在纸上。
- **关系**（箭头）：卡车从哪里装货、送往哪里。工人分配到哪条产线。仓库覆盖哪些客户。
- **标注**（数字）：卡车载重 5t、客户 A 需要 2t 明天 10 点前到、从仓库 1 到客户 A 运费 30 元/公里。

**举例**——配送问题：

```
   [仓库] ──50km,30¥/km──▶ [客户A: 2t, 10:00前]
     │
     └──20km,25¥/km──▶ [客户B: 1.5t, 14:00前]
     │
     └──35km,20¥/km──▶ [客户C: 3t, 全天]
```

画完这张图，你再去看问题描述——是不是清晰了很多？如果你连图都画不出来，说明你还没理解业务。**回去找业务方聊清楚，别急着建模。**

### 第二步：找决策变量（口诀）

先问自己一个问题：「我最后要交给领导看的那张 Excel 表里，哪些单元格是需要填的？」

那些需要你「填」的单元格，就是你的决策变量。

还是配送的例子——你最后要给司机的是一张表：

| 司机 | 路线 | 出发时间 |
|------|------|----------|
| 老王 | 仓库→A→B→仓库 | 7:30 |
| 小李 | 仓库→C→D→仓库 | 8:00 |

你的决策变量就是：每个司机去不去（是/否）、走哪条路线（序号）、什么时间出发（时间）。

技巧：把所有变量分成两类——**离散的决策**（去不去、建不建、选不选）和**连续的量**（运多少、存多少、做多少）。离散变量用 0/1 或整数，连续变量用非负实数。

### 第三步：写约束清单（用自然语言）

不要一上来就 ∑ 和 ∀。先用大白话列出所有「不能」：

- 一辆车不能装超过 5 吨
- 一个客户不能同时被两辆车服务
- 一个司机一天不能开超过 8 小时
- 客户 A 必须在 10:00 前送到
- 仓库的出货量不能超过库存

写完以后，拿这张清单去问业务方：「还有没有遗漏的物理限制？」这比给他们看公式有效得多。

### 第四步：写目标（通常一句话）

目标通常是 4 种之一：
- 让总成本最小
- 让总利润最大
- 让完成时间最短
- 让资源利用率最均衡

只需要一句话：「我想让 X 最小化/最大化。」

**重要提醒**：最大化利润 和 最小化成本 **不是等价的**。如果你只最小化成本——最优解可能是「什么都不干，成本为零」。所以写目标的时候要想清楚：你到底想要什么结果。

### 第五步：检查边界

写完模型后，做三个极端测试：

- **零输入测试**：如果没有任何订单（0 个客户），模型应该返回空方案或者 infeasible（不可行），而不是报奇怪的错。
- **超大规模测试**：如果你有 1,000,000 个客户，模型应该报超时（这是正常的），而不是默默把内存吃光。
- **松弛测试**：把所有整数变量改成连续变量，求解 LP。得到的值应该 ≤ 整数最优值（如果是最小化问题）——如果 LP 的值比你的整数解还好很多，说明整数约束真的很「贵」，你可能需要考虑*启发式*方法。

### 五步法速查卡

```
Step 1 (2 min)  → 画实体、箭头、数字
Step 2 (1 min)  → 找 Excel 里需要填的格
Step 3 (1 min)  → 用大白话列「不能」
Step 4 (30 sec) → 用一句话说目标
Step 5 (30 sec) → 零输入/超大规模/松弛 三连测
```

> 🆘 **逃生通道**：如果你连步骤一都画不出来——你还没理解业务。回去找业务方聊清楚，别急着建模。一张画对的图，比一百行写错的公式有用。另外，步骤 5 的三个测试，在你第一次建模时经常能发现 80% 的 bug。
