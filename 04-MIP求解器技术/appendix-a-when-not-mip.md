# 附录A：什么时候不需要 MIP

MIP 很强，但不是银弹。以下场景你应该考虑其他方法。

## 决策流程图

```
你有一个优化问题
        │
        ▼
   ┌─ 问题规模有多大？
   │
   ├── < 10⁵ 个决策变量 ──→ 直接 MIP ✓
   │
   ├── 10⁵ ~ 10⁶ ──→ 试试 MIP + 调参
   │                   （MIPFocus=1, 加大 MIPGap, 加对称性破坏）
   │
   └── > 10⁶ ──→ 先分解再 MIP，或换方法
```

## 替代方案速查

| 情况 | 替代方案 | 为什么 |
|------|---------|--------|
| 规模 > 10⁷ 个变量 | 元启发式（遗传/退火/蚁群） | MIP 分支树过于庞大，预处理就卡死 |
| 只要「差不多就行」 | 贪心 + 局部搜索 | 10 行代码出结果 vs 等一天 |
| 纯约束满足（无目标） | CP-SAT (OR-Tools) | 在纯离散约束上可能更快几个数量级 |
| 线性 + 大规模 + 特殊结构 | 列生成 | 分解到 MIP 能处理的子问题 |
| 连续变量 + 凸目标 | LP / 内点法 | 没必要整 MIP，太慢 |
| 动态决策（时序） | 动态规划 / 强化学习 | MIP 是一次性静态决策 |
| 不确定参数（需求波动） | 随机规划 / 鲁棒优化 | 标准 MIP 假设所有参数已知 |

## 一句话

**MIP 适合「变量几千到几十万、必须精确、约束复杂」的场景。** 超出这个范围，先想分解，再想替代。
