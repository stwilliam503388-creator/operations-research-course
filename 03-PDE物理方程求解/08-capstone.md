
# 🏆 毕业项目：芯片封装多物理场模拟

> 用学到的 PDE 知识解决一个真实的工程问题：芯片在工作时发热 → 温度升高 → 材料热膨胀 → 产生热应力 → 可能导致封装开裂或芯片失效。你需要至少组合 2 种 PDE（热传导 + 结构力学），评估不同数值方法的优缺点。

---

## 问题描述

你是一家芯片封装公司的仿真工程师。老板拿过来一块新的封装设计：硅芯片（die）通过焊料和基板（substrate）连接，再封装到 PCB 上。芯片在满负荷工作时，一个 0.5mm × 0.5mm 的「热点」产生极高的热流密度（~1 MW/m²）。

老板问三个问题：
1. **芯片最高温度在哪？多少度？**
2. **热点附近的机械应力有多高？会不会导致封装开裂？**
3. **换一种散热方案（比如加个铜散热片），温度能降多少？**

这些问题无法单独用热传导或单独用结构力学回答——你必须把两个物理场耦合起来。

## 物理建模

### 物理场 1：热传导（抛物型 PDE）

$$
\rho c_p \frac{\partial T}{\partial t} = \nabla \cdot (k \nabla T) + Q
$$

- T(x,y,t) — 温度场 [°C]
- k — 导热系数 [W/(m·K)]（硅: 130, FR4: 0.3）
- ρc_p — 体积热容 [J/(m³·K)]
- Q(x,y) — 热源密度（芯片热点区域）

边界条件：
- 四边对流换热：-k ∂T/∂n = h(T - T_ambient)，h = 50 W/(m²·K)
- 初始条件：T(x,y,0) = T_ambient = 25°C

### 物理场 2：热弹性方程（椭圆型 PDE）

热膨胀产生的应变：

$$
\boldsymbol{\varepsilon} = \frac{1}{2}(\nabla \mathbf{u} + \nabla \mathbf{u}^T)
$$

本构关系（平面应力近似）：

$$
\boldsymbol{\sigma} = \frac{E}{1-\nu^2} \left[ \boldsymbol{\varepsilon} - \alpha \Delta T \cdot \mathbf{I} \right]
$$

- u = (u, v) — 位移场
- σ — 应力张量
- E — 弹性模量，ν — 泊松比，α — 热膨胀系数
- ΔT = T - T_ref — 温差

平衡方程（忽略惯性力）：

$$
\nabla \cdot \boldsymbol{\sigma} + \mathbf{f} = 0
$$

### 耦合方式

```
热源 Q → 热传导方程 → 温度场 T(x,y)
                           ↓ (单向耦合)
                    热膨胀 αΔT → 本构方程 → 热应力 σ(x,y)
```

**单向耦合（one-way coupling）**：温度影响应力，但应力不影响温度。这是最常见的工程简化。如果考虑裂纹导致导热系数变化、或变形导致的几何变化，就需要**双向耦合**。

## 分步实现

### Step 1：网格与参数设置

| 参数 | 值 | 说明 |
|------|-----|------|
| 封装尺寸 | 10mm × 10mm × 1mm | 平面应力近似 |
| 芯片导热系数 | 130 W/(m·K) | 硅 |
| 基板导热系数 | 0.3 W/(m·K) | FR4 |
| 热点尺寸 | ~1mm × 1mm | 芯片中心区域 |
| 热源密度 | 1×10⁹ W/m³ | ~10W 集中在热点 |
| 环境温度 | 25°C | — |
| 对流系数 | 50 W/(m²·K) | 自然对流 |

### Step 2：热传导求解

**方法选择对比**：

| 方法 | 优缺点 | 本实现选用 |
|------|--------|-----------|
| FDM 显式 | 简单但 Δt 受 CFL 限制（Δx²/α） | — |
| FDM 隐式 | 无条件稳定，每步解线性系统 | — |
| ADI | 隐式的变体，每步只解三对角系统（Thomas 算法） | ✅ |
| FEM | 适合复杂几何，但实现复杂 | — |

**本实现**：在 `code/python/capstone.py` 中同时提供了两个求解器：
- `solve_steady_state()` — Gauss-Seidel 迭代法
- `solve_transient()` — ADI（交替方向隐式）法

### Step 3：温度场可视化

运行后的温度场云图显示：热点中心温度可达 85~100°C（取决于散热条件），从热点到边缘温度梯度陡峭。

### Step 4：热应力计算

给定温度场后，用热弹性本构方程计算应力：
- 热点中心区域受压缩（温度高、膨胀被周围材料约束）
- 热点边缘区域出现拉伸应力峰值
- 最大 von Mises 应力通常在 30~80 MPa 范围

### Step 5：评估与对比

```
硅的抗拉强度 ≈ 150 MPa（取决于工艺和缺陷密度）
安全系数 = 150 / σ_max
```

如果安全系数 < 1.5，就需要改进：更大芯片面积、更低功耗、更好散热、或者用低 CTE（热膨胀系数）基板材料。

## 评估标准

衡量你的多物理场模拟做得好不好：

1. **正确性**：与已知解析解（如简单几何的一维热传导 + 热应力）对比。误差在 5% 以内为合格。

2. **收敛性**：网格加一倍，解的变化应减小。用不同的 N 做网格收敛性分析。

3. **效率**：你的代码能处理多大的网格？多大的时间步长？能用多长时间完成一次仿真？

4. **可扩展性**：从 2D 到 3D 需要改多少代码？从单向耦合到双向耦合呢？

5. **物理合理性**：温度分布合理吗？（没有负的绝对温度、最大值不超过材料熔点）应力分布合理吗？（约束处的应力集中区域与直觉一致）

## 扩展方向

这个骨架代码是让你「跑起来」的起点。如果你想认真做毕业设计，可以选以下方向之一深入：

### 方向 A：双向热-力耦合

- 裂纹导致局部导热系数下降 → 温度再分布 → 应力再分布
- 需要迭代求解直到收敛（固定点迭代或 Newton-Raphson）

### 方向 B：FEM 代替 FDM

- 用三角形网格处理圆形芯片/异形封装
- 用 `scikit-fem` 或 `FEniCS` 实现
- 对比 FDM 和 FEM 的精度与效率

### 方向 C：瞬态 + 材料非线性

- 材料参数（k, E, α）随温度变化
- 焊料的蠕变行为（Coffin-Manson 疲劳模型）

### 方向 D：PINN 尝试

- 用 Physics-Informed Neural Networks 求解同样的热传导方程
- 比较 PINN 与传统 FDM/FEM 的精度和训练时间

### 方向 E：3D 扩展

- 把 2D 温度场扩展为 3D
- 考虑芯片厚度方向的温度梯度
- 3D 热应力的完整张量计算
