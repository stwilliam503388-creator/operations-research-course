# 案例 1：新闻摊贩的随机版

> 难度 ★★☆☆☆ · 方法：随机规划/单阶段 · 核心概念：期望值模型、解析解 vs 仿真 · 代码：[code/case01_newsvendor.py](code/case01_newsvendor.py)

---

## 1 场景

你经营一个报刊亭。每天要决定进货多少份报纸。数字如下：

| 参数 | 值 |
|------|-----|
| 进货价 c | 0.5 元/份 |
| 零售价 p | 1.0 元/份 |
| 退货价 v | 0.1 元/份 |
| 需求 D ~ N(μ=100, σ=20) | 均值为 100，标准差为 20 的正态分布 |

**关键矛盾**：
- 进货太少 → 错失销售机会（underage cost = p - c = 0.5 元/份）
- 进货太多 → 卖不完折价退（overage cost = c - v = 0.4 元/份）

> 🆘 核心记住：所有随机优化问题都在这两种「错误」之间权衡。

### 先猜一个直觉方案

最自然的想法是：平均每天卖 100 份，那就进 100 份。这个猜法很有吸引力，因为“平均需求”看起来像最稳妥的数字。

但报童问题真正问的是：**多进一份亏多少，少进一份亏多少？** 如果少进一份损失更大，你就应该比平均需求多进一点；如果多进一份损失更大，你就应该少进一点。

> 停一下，想一想：如果卖不掉可以原价退货，你会不会进得比平均需求更多？如果卖不掉一文不值呢？

## 2 模型假设与现实边界

这个案例是单周期库存模型，适合用来理解“需求不确定时为什么不能只看平均值”。它依赖这些教学假设：

| 假设 | 教学中为什么合理 | 现实中什么时候会失效 |
|------|------------------|----------------------|
| 只决策一天的进货量 | 单周期问题能清楚展示过量/不足权衡 | 连续经营时会有库存滚动、补货提前期和季节性 |
| 需求分布已知且服从正态 | 方便用分位数解释解析解 | 真实需求可能偏态、截断、厚尾或受促销影响 |
| 未售出商品有固定残值 | 让 overage cost 清晰 | 生鲜、时效商品的残值可能随时间变化 |
| 不考虑缺货替代和客户流失 | 先聚焦利润公式 | 真实缺货会影响复购、平台排名和替代购买 |
| 目标是最大化期望利润 | 最适合入门 | 风险规避业务可能更关心 CVaR 或亏损概率 |

因此，本案例的“最优订货量”只是在当前分布、价格和单周期假设下最优。真实项目要先验证需求分布、估计误差、提前期、缺货成本和业务风险偏好。

## 3 建模：变量 → 约束 → 目标

**决策变量**：进货量 q（整数，≥ 0）

**目标**：最大化期望利润

```
max  E[ profit(q, D) ]
```

其中：
```
profit(q, D) = min(D, q) × p + max(q - D, 0) × v - q × c
```

**逐符号翻译**：
- `min(D, q) × p`：卖出去的部分 × 售价
- `max(q - D, 0) × v`：没卖出去的部分 × 退货价
- `q × c`：进货成本

---

## 4 方法详解

### 4.1 解析解（少数可以手算的随机优化模型）

**关键公式**：
```
F(q*) = (p - c) / (p - v) = critical ratio
```

其中 F 是需求 D 的累积分布函数（CDF），critical ratio = 临界比率。

**代入数字**：
- critical ratio = (1.0 - 0.5) / (1.0 - 0.1) = 0.5 / 0.9 ≈ 0.5556
- 查正态分布 CDF：F⁻¹(0.5556) ≈ 0.14（标准正态分位数）
- q* = μ + 0.14 × σ = 100 + 0.14 × 20 ≈ 102.8 → 取整 103 份

**含义**：当日需求累积分布达到 55.56% 时的需求量就是最优进货量。

### 错误直觉：最优订货量就是平均需求

- **为什么听起来合理**：平均需求代表“长期中心位置”。
- **为什么会错**：库存决策不是猜需求，而是在“缺货损失”和“滞销损失”之间权衡。两个损失不对称时，最优点就会偏离平均值。
- **公式怎么体现**：`(p - c) / (p - v)` 比较的正是少进一份和多进一份的相对代价。

### 4.2 仿真方法

当问题变复杂（多产品、非线性成本），解析解就不存在了。这时候用蒙特卡洛仿真：

1. 生成 N 个需求样本 D₁, ..., D_N ~ N(100, 20)
2. 对每个候选 q，计算平均利润
3. 搜索使期望利润最大的 q

## 9 关键洞察

**为什么新闻摊贩在随机规划中地位那么高？**
- 它是**少数有清晰解析解的随机优化模型之一**——让你能直观理解随机优化的核心权衡
- 临界比率公式可以推广到大部分单周期库存问题
- 几乎所有更复杂的模型（多产品、多阶段、风险规避）都是在这个模型上做加法

**过度订货 vs 订货不足的权衡线**：
- critical ratio > 0.5：进货偏多（因为少卖损失 > 多进货损失）
- critical ratio < 0.5：进货偏少（反之）
- critical ratio = 0.5：两种损失一样，进货量 = 需求中位数

### 如果你只记住一件事

随机库存不是“按平均需求订货”，而是看多订和少订哪个更痛。临界比率就是把这种痛感写成一个分位数。

## 10 延伸与变形

- **多产品新闻摊贩**：有预算上限时怎么分配——这是一个随机背包问题
- **风险规避版本**：用 CVaR 代替期望值，进货量更保守
- **需求预测更新**：先用先验分布，然后根据早期销售数据更新后验（贝叶斯新闻摊贩）
