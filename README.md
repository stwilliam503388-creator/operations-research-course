## 代码运行方式

### Python 案例

建议在仓库根目录统一安装依赖：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

然后进入对应课程目录运行案例，例如：

```bash
cd "05-运筹学基础与实战"
python code/python/case03_logistics.py
```

少数课程有局部依赖说明，例如 `04-MIP求解器技术/code/requirements.txt`。如果某个案例使用求解器或仿真库，请优先查看该课程 README 的环境准备章节。

### 全仓库快速检查

可以运行根目录 smoke check，快速确认 Python 脚本可编译、C++17 案例可编译，并执行第 10 门课的一键案例检查。它不是全仓运行时测试，不会执行所有依赖 Pyomo、SimPy、HiGHS 或绘图的案例：

```bash
make check
```

### C++ 案例

C++ 案例默认使用 **C++17**，优先只依赖标准库，统一放在各课程的 `code/cpp/` 目录。单文件案例可以直接编译：

```bash
cd "02-低复杂度算法设计"
g++ -std=c++17 -O2 code/cpp/case03_merge_sort.cpp -o /tmp/case03_merge_sort
/tmp/case03_merge_sort
```

C++ 案例不是对 Python 的机械翻译，而是用于补充三类教学价值：算法细节、性能差异、工程实现。

如果只想快速确认所有 C++ 对照案例能编译，也可以在根目录运行 `make cpp`。

## 内容质量标准

本课程系列采用“人话解释 + 严格边界 + 可运行案例”的写法。为了避免“听起来懂了但其实被误导”，后续新增或重写内容都遵循 [课程内容质量规范](COURSE_CONTENT_GUIDE.md)：

- 核心术语要同时说明直觉、严格含义、常见误解和案例用途。
- 案例要明确模型假设、现实边界和工业版还需要补什么。
- 数学模型要尽量给出“业务对象 → 数学符号 → Python 变量 → C++ 变量”的映射。
- 求解结果只能说明“当前模型内”的结论，不能直接等同于真实业务最优。

## 常见问题

**Q: 我需要按顺序学吗？**  
不必须。但概率论是随机规划和随机优化的基础，运筹学是多目标优化和 MIP 求解器的前提。如果你想跳着学，先确认前置知识够了。

**Q: 每门课要学多久？**  
设计目标都是「一个周末」（约 12-16 小时）。每门课 8 章 + 案例 + 附录，按你自己节奏来。

**Q: 代码能跑吗？**  
能。每门课都配有 Python 代码示例（位于 `code/python/`），依赖在根目录和各自 README 里说明。C++ 对照案例位于 `code/cpp/`，默认可用 `g++ -std=c++17 -O2` 编译。

**Q: 这个 monorepo 和原来的独立 repo 有什么关系？**  
原来的独立 repo（`probability-course`、`game-theory-course`、`low-complexity-algorithm-course` 等 9 个）已全部合并到此 monorepo，后续更新都在这里进行。第 10 门凸优化课程是在 monorepo 中新增的连续优化补位课程。`course-series` 仓库是早期的索引项目，本 monorepo 已完全替代其功能，可以归档。
