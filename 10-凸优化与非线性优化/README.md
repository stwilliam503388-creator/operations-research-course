# 凸优化与非线性优化基础课程

> 一个周末，从零到能判断问题是否凸、看懂 KKT 和梯度法，并用 Python/C++ 跑通连续优化案例。

---

## 内容质量标准

本课程遵循根目录的 [课程内容质量规范](../COURSE_CONTENT_GUIDE.md)：用人话解释核心直觉，同时明确模型假设、适用边界和可运行验证方式。

## 这份教程是什么

这是一份给**有基础微积分、线性代数和 Python 能力，但对连续优化只停留在“梯度下降会让 loss 下降”阶段**的技术人写的自学教程。它不把你拉进厚重定理，而是围绕一个问题展开：**什么时候一个优化问题好解，什么时候优化器只是给了你一个看起来不错的局部答案？**

读完以后，你一定会：

- 区分凸优化和非凸优化，知道“局部最优就是全局最优”什么时候成立
- 看懂梯度下降、牛顿法、投影梯度、KKT 条件在做什么
- 用 Python 求解最小二乘、投资组合 QP、逻辑回归和约束设计优化
- 用 C++ 实现一个轻量梯度下降，理解优化循环本身
- 知道什么时候不该强行用非线性优化

## 术语速查表

### 核心术语

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 优化问题 | Optimization Problem | 像在一堆可选方案里挑“最划算”的那个：成本最低、收益最高，或者误差最小 |
| 凸集 | Convex Set | 像一块没有裂缝的橡皮泥：任意两点之间拉一条直线，中间都还在橡皮泥里 |
| 凸函数 | Convex Function | 像一个光滑的碗：只要一路往低处走，最后不会被另一个更深的坑骗走 |
| 非凸函数 | Nonconvex Function | 像连绵山地：到处都有小山谷，从不同地方出发可能掉进不同坑 |
| 局部最优 | Local Optimum | 像小区里最低的地下车库入口：附近看最低，但全城可能还有更低的地方 |
| 全局最优 | Global Optimum | 像全城最低点：把所有合法方案都比过以后，真正最好的那个 |
| 梯度 | Gradient | 像地面最陡的上坡箭头；想让目标变小，通常就朝它反方向走 |
| Hessian | Hessian Matrix | 像告诉你地形弯不弯的“曲率雷达”：哪里像碗底，哪里像山脊，它都能提示 |
| 一阶条件 | First-Order Condition | 到了一个点，往任何很近的方向挪一下都占不到明显便宜 |
| 二阶条件 | Second-Order Condition | 不只看“斜不斜”，还看“弯不弯”，判断这是谷底、山顶还是马鞍 |
| 拉格朗日乘子 | Lagrange Multiplier | 像约束的价格标签：预算多放松一点，目标大概能改善多少 |
| KKT 条件 | KKT Conditions | 像约束优化的体检表：方案是否合法、哪些约束卡住了、有没有继续改进空间 |
| 对偶 | Duality | 换个角度算同一件事：不直接挑方案，而是给资源和约束标价 |
| 梯度下降 | Gradient Descent | 像蒙着眼下山：摸到最陡的上坡方向，然后小心往反方向迈一步 |
| 牛顿法 | Newton Method | 像带着地形图下山：不只知道哪边低，还估计谷底大概在哪里，所以可能走得更快 |
| 投影梯度 | Projected Gradient | 像先迈一步，再被安全绳拉回允许区域，保证方案不越界 |
| 二次规划 | Quadratic Programming, QP | 目标像“碗”，约束像几面墙：在墙围起来的区域里找碗底 |
| 条件数 | Condition Number | 像山谷又细又长：方向稍微不对就来回蹭墙，算法会走得很磨叽 |
| 鞍点 | Saddle Point | 像马鞍中间：前后看像谷底，左右看又像山顶，梯度为 0 也不代表最优 |
| 正则化 | Regularization | 像给模型上“防抖器”：别为了贴合训练数据，把参数调得太夸张 |

### 概念边界

凸优化的关键价值不是“公式更漂亮”，而是：**在满足条件时，局部最优就是全局最优**。非凸优化也不是不能用，而是你要承认初始点、算法选择和参数会影响最终结果。

## 学习验收标准

学完本课程后，建议用下面 6 条自检：

- 能判断简单函数和可行域是否凸，并说出为什么。
- 能解释梯度下降为什么可能慢，以及学习率太大会发生什么。
- 能用自己的话说明 KKT 条件中的可行性、互补松弛和拉格朗日乘子。
- 能运行最小二乘、QP、逻辑回归、约束设计和非凸陷阱案例，并解释核心输出。
- 能编译运行 C++ 梯度下降案例，并说明它对应 Python 最小二乘里的哪部分。
- 能指出一个教学优化模型在真实业务中还缺哪些数据、约束或风险检查。

## C++ 对照案例

C++ 案例只用标准库，强调“优化循环本身”：

```bash
g++ -std=c++17 -O2 code/cpp/case03_gradient_descent.cpp -o /tmp/case03_gradient_descent
/tmp/case03_gradient_descent
```

学习目标：理解梯度下降如何从初始参数出发，让损失逐步下降，并与 Python 的最小二乘结果对照。
