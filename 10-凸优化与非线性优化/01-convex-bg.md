# 01 凸优化背景：为什么“碗状问题”更好解

## 1. 故事开场

你在调一个模型参数。参数太小，预测偏低；参数太大，预测偏高。你想找一个误差最小的位置。

如果误差曲线像一个碗，你从哪里开始往下走，最后都会到同一个谷底。如果误差曲线像连绵山谷，你可能掉进第一个小坑，以为自己到底了。

这就是凸优化和非凸优化的核心区别。

> 如果你只记住一件事：凸优化的价值在于，满足条件时，局部最优就是全局最优。

## 2. 什么是优化问题

一个优化问题通常长这样：

```text
minimize    f(x)
subject to  x ∈ C
```

- `x` 是你能调整的变量
- `f(x)` 是你想变小的目标函数
- `C` 是可行域，也就是现实允许你做的选择

业务翻译：在所有可执行方案里，找成本最低、误差最小、风险最小或收益最大的那个。

## 3. 凸集：中间不会掉出去

小白版：如果两个方案都可行，把它们按比例混合出来的新方案也可行，这个可行域就是凸的。

专业版：集合 `C` 是凸集，当且仅当对任意 `x, y ∈ C` 和 `θ ∈ [0, 1]`，都有：

```text
θx + (1 - θ)y ∈ C
```

这行公式在说：从 `x` 到 `y` 的整条线段都留在可行域里。

## 4. 凸函数：连线在图像上方

小白版：凸函数像一个碗，没有多个互相隔开的低谷。

专业版：函数 `f` 是凸函数，当且仅当：

```text
f(θx + (1 - θ)y) ≤ θf(x) + (1 - θ)f(y)
```

这行公式在说：两个点之间的函数值，不会高到穿过连线之上。

## 5. 为什么凸优化重要

当目标函数凸、可行域凸时，局部最优就是全局最优。这意味着：

- 找到一个满足最优性条件的点，就不用担心还有另一个更深的坑
- 算法失败时更容易诊断，是学习率、条件数、约束还是数值问题
- 很多现实模型可以被设计成凸问题，例如最小二乘、QP、逻辑回归、部分鲁棒优化

## 6. 错误直觉：看起来像碗就一定凸吗？

- **为什么听起来合理**：二维图像上很多函数看起来只有一个谷底。
- **为什么会错**：高维空间里可能有弯曲、鞍点、约束断裂，肉眼很难判断。
- **正确理解**：凸性是数学性质，不是视觉印象。要检查函数、约束和变量域。

> 停一下，想一想：如果目标函数是凸的，但可行域被切成两个互不相连的区域，整个问题还是凸优化吗？

## 7. 和已有课程怎么连接

| 课程 | 连接点 |
|------|--------|
| 线性规划 | LP 是凸优化的特例 |
| MIP | LP/QP 松弛常用凸优化求解 |
| 投资组合 | 均值-方差模型是典型 QP |
| 机器学习 | 最小二乘、逻辑回归是常见凸优化 |
| 鲁棒优化 | 很多鲁棒对应模型可以转成凸优化 |
| PDE | 参数识别和反问题经常是连续优化 |

## 8. 学习验收

学完本章，你应该能：

- 用自己的话解释凸集和凸函数
- 说出凸优化为什么比一般非凸优化更可靠
- 判断“局部最优就是全局最优”需要哪些前提
- 举出一个非凸问题可能被初始点影响的例子
