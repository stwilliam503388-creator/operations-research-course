# 02 核心方法：从梯度下降到 KKT

## 1. 先猜一个直觉方案

如果你站在山坡上想下山，最自然的办法是：看哪边最陡，就往反方向走一步。梯度下降就是这个直觉的数学版本。

但连续优化的难点不只是“往哪走”，还有：

- 一步走多远？
- 如果走出可行域怎么办？
- 如果山谷很狭长怎么办？
- 如果有约束，谷底被墙挡住怎么办？

## 2. 梯度下降

更新公式：

```text
x_{t+1} = x_t - α ∇f(x_t)
```

这行公式在业务上表示：根据当前目标函数最陡上升方向 `∇f(x_t)`，往反方向调整决策变量。`α` 是学习率，也就是每次调整的步长。

### 错误直觉：梯度下降一定会下降

- **为什么听起来合理**：负梯度方向在局部确实是下降方向。
- **为什么会错**：学习率太大时，一步可能跨过谷底，甚至越走越远。
- **正确理解**：方向和步长都重要。方向对了，步子太大也会失败。

> 停一下，想一想：学习率太小会慢，太大会炸。你会先看 loss 曲线，还是先改模型？

## 3. 牛顿法

梯度下降只看坡度，牛顿法还看曲率：

```text
x_{t+1} = x_t - H^{-1} ∇f(x_t)
```

这里 `H` 是 Hessian。它告诉你每个方向弯得有多厉害。

- 优点：在二次函数附近收敛很快
- 缺点：要算 Hessian，还要解线性方程，大规模问题很贵

## 4. 投影梯度

如果变量必须满足约束，例如权重非负且总和为 1，梯度一步可能走出可行域。投影梯度做两件事：

1. 先按负梯度走一步
2. 再把结果拉回可行域

```text
x_{t+1} = Proj_C(x_t - α ∇f(x_t))
```

这行公式在业务上表示：先提出一个更优方向，再把方案修正到现实允许范围内。

## 5. 拉格朗日乘子和 KKT

带约束问题的最优点，常常不是目标函数自己最低的地方，而是被约束“顶住”的地方。

拉格朗日函数：

```text
L(x, λ) = f(x) + λ g(x)
```

小白版：`λ` 是约束的影子价格，表示这条约束有多“卡脖子”。

KKT 条件是一组检查清单：

- 原问题可行：方案满足原始约束
- 对偶变量可行：不等式约束的价格方向合理
- 互补松弛：没卡住的约束价格为 0
- 驻点条件：目标和约束的力达到平衡

## 6. 一阶方法还是二阶方法

| 方法 | 适合场景 | 风险 |
|------|----------|------|
| 梯度下降 | 大规模、简单、可微目标 | 学习率敏感，收敛可能慢 |
| 牛顿法 | 中小规模、曲率信息可得 | Hessian 昂贵或不稳定 |
| 投影梯度 | 简单约束，如盒约束、单纯形 | 投影本身可能复杂 |
| 通用求解器 | 复杂约束和工程问题 | 容易忽略模型假设和数值诊断 |

## 7. 如果你只记住一件事

优化算法不是魔法：梯度告诉你方向，学习率决定步长，约束决定边界，KKT 帮你检查“是不是被边界卡住的最优”。

## 8. 学习验收

学完本章，你应该能：

- 解释梯度下降每一项的含义
- 说出学习率太大和太小的表现
- 理解投影梯度为什么先走再拉回
- 用直觉解释拉格朗日乘子和 KKT 条件
