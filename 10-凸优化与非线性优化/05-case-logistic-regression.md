# 案例 3：逻辑回归与客户流失预测

> 难度 ★★★☆☆ · 方法：凸损失 / 梯度下降 / 正则化 · 代码：`code/python/case05_logistic_regression.py`

## 1. 故事开场

一家订阅制产品想预测哪些客户下个月可能流失。你手里有两个特征：最近登录次数和客服投诉次数。目标不是精确预测销量，而是判断“流失概率高不高”。

逻辑回归是二分类里最经典的凸优化案例。

## 2. 直觉猜法

你可能会写规则：投诉超过 3 次就判定会流失。但规则很硬，无法表达“投诉多但登录也多”的灰色地带。

逻辑回归做的是：把特征加权求和，再通过 sigmoid 转成概率。

## 3. 模型假设

| 假设 | 教学中为什么合理 | 现实中什么时候会失效 |
|------|------------------|----------------------|
| 特征和 log-odds 近似线性 | 便于得到凸损失 | 真实行为可能强非线性 |
| 样本独立同分布 | 入门训练简单 | 用户行为会受活动、季节、群体传播影响 |
| 标签可靠 | 便于解释 loss | 真实流失定义可能延迟或噪声很大 |
| 优化目标是 log loss | 数学性质好 | 业务可能更关注召回率、成本或 LTV |

## 4. 数学模型

```text
p_i = sigmoid(w^T x_i + b)
loss = -mean[y_i log p_i + (1-y_i) log(1-p_i)] + λ ||w||^2
```

这行公式在业务上表示：预测概率越接近真实标签，损失越小；正则化项防止参数过大。

## 5. 错误直觉

- **错误直觉**：loss 下降就代表业务效果一定变好。
- **为什么会错**：业务可能关心高风险客户召回率，而不是平均 log loss。
- **正确理解**：优化指标是训练目标，业务指标仍要单独验证。

## 6. 业务-公式-代码映射

| 业务对象 | 数学符号 | Python 变量 | 业务含义 |
|----------|----------|-------------|----------|
| 客户特征 | `x_i` | `X` | 登录、投诉等行为 |
| 是否流失 | `y_i` | `y` | 0/1 标签 |
| 权重 | `w` | `w` | 每个特征影响方向和强度 |
| 截距 | `b` | `b` | 基础流失倾向 |
| 正则化强度 | `λ` | `reg` | 控制参数大小 |

## 7. 如果你只记住一件事

逻辑回归的优化问题通常是凸的，但“数学上训练好”不等于“业务上决策好”，还要看阈值、成本和评估指标。

- 能运行 `code/python/case05_logistic_regression.py`，并看到 loss 下降。
