# Case 05: 生产调度多目标优化 ★★★☆☆

## 场景描述

一个车间有 3 台机器（M1、M2、M3），需要处理 8 个作业。每个作业依次经过这 3 台机器（流水车间，Flow Shop）。

作为车间主任，你有三个相互矛盾的目标：

1. **最小化最大完工时间（Makespan）**——最后一单完成的时间越早越好，客户等着呢
2. **最小化总延迟时间**——每个作业的截止时间各不相同，超期要赔钱
3. **最小化机器空闲时间**——机器闲着就是浪费，但赶工期又免不了有空档

问题来了：缩短 Makespan 通常意味着各机器并行度更高、缓冲区更满，但这样容易导致某些作业被"挤"到后面去而延迟交付。而为了减少延迟去插入加急订单，又会让 Makespan 变长。

三个目标互相拉扯，这就是生产调度的多目标优化困境。

### 问题要素

| 要素 | 说明 |
|------|------|
| **决策变量** | 各机器上的作业加工顺序（排列组合问题） |
| **目标 1** | 最小化最大完工时间 $ C_{\max} = \max_j C_j $ |
| **目标 2** | 最小化总延迟 $ \sum_j \max(0, C_j - d_j) $ |
| **目标 3** | 最小化总空闲时间 $ \sum_i (\text{机器 } i \text{ 的闲置时间}) $ |
| **约束** | 同一道工序在一台机器上独占，作业有固定工艺路线 |

## 建模

### 流水车间调度模型

有 $ n = 8 $ 个作业，每个作业需要依次经过 $ m = 3 $ 台机器。

每个作业 $ j $ 在机器 $ i $ 上的加工时间 $ p_{ji} $ 已知，截止时间 $ d_j $ 已知。

调度决策本质上是在**每台机器上确定一个作业排列顺序**，使得所有作业的完成时间满足约束。

### 完工时间计算

给定一个作业序列（所有作业的访问顺序），机器的完工时间递推公式：

$$
C_{j,i} = \max(C_{j-1,i}, C_{j,i-1}) + p_{ji}
$$

其中 $ C_{j,i} $ 表示第 $ j $ 个作业在第 $ i $ 台机器上的完工时间。初始条件：$ C_{0,i} = 0 $，$ C_{j,0} = 0 $。

这个递推关系意味着：作业只能在上一台机器加工完成 **且** 当前机器空闲时，才能开始加工。

### MILP 模型（扩展阅读）

对于更小的实例，我们可以建立混合整数线性规划：

决策变量：$ x_{jk} \in \{0,1\} $（作业 $ j $ 是否在第 $ k $ 个位置加工），$ s_{ji} $（作业 $ j $ 在机器 $ i $ 上的开始时间）。

约束包括作业排列的唯一性、加工优先级、时间连续性等。MILP 可以求得精确帕累托前沿，但计算量随作业数指数增长。

## 方法

### 1. 加权求和法 + 随机搜索

由于这是一个组合优化问题（排列问题），我们使用加权求和 + 随机采样的策略：

- 对每组权重 $ (w_1, w_2, w_3) $，生成大量随机排列
- 对每个排列计算三个目标值
- 用加权和选出该权重下的"最佳"调度
- 最后从所有候选解中提取帕累托非支配解

权重策略：$ w_1 $（Makespan 权重）从 0.5 到 1.0，$ w_2 $（延迟权重）从 0 到 0.5，$ w_3 = 1 - w_1 - w_2 \ge 0 $。

### 2. 帕累托过滤

对加权求和法产生的所有候选解，逐一检查是否被其他解支配（在三个目标上都更优），只保留非支配解。

### 3. 单目标对比

分别找到纯最小化 Makespan 和纯最小化延迟的调度方案，作为"边界点"验证。

## 洞察

1. **调度里的"不可能三角"**——Makespan、延迟、空闲时间三者之间，优化任意两个往往以牺牲第三个为代价。这与项目管理中的"时间-成本-质量"三角异曲同工。

2. **权重变化的效果**——当 $ w_1 $（Makespan 权重）从 0.5 增加到 1.0 时，调度方案从"少延迟"端移向"短 Makespan"端。这说明权重就是决策者的"价值观"——你更看重什么，算法就给你什么。

3. **组合优化的复杂性**——8 个作业的排列数有 8! = 40,320 种，我们只随机采样了 1,600 种（8×8×200）。这远非穷举，但已经能找到不错的帕累托前沿。对于更大规模的问题，需要用遗传算法或禁忌搜索。

4. **甘特图的直观力量**——看到机器上的彩色时间块，你立刻就能理解为什么 Makespan 和总延迟是冲突的：把作业往前提（减少延迟）会打乱整体流线，增加空闲和总时长。

5. **MILP 的可行性**——对于 8×3 的小规模问题，其实可以用 MILP 求解器（如 PuLP/Gurobi）得到精确前沿。这里用随机搜索是为了降低运行门槛，让所有读者都能跑起来。

## 延伸

1. **动态重调度**——机器突然故障，新订单插入，如何在线调整调度方案？需要反应式调度或鲁棒调度方法。

2. **绿色调度**——加入能耗目标（如每台机器的能耗），变成三目标调度。机器状态切换（开机/待机/关机）的能耗管理。

3. **分布式制造**——多个工厂协同排产，不同工厂有不同机器的产能和成本。变成了多工厂、多机器、多目标的超复杂问题。

4. **强化学习求解**——用 DQN 或 PPO 学习调度策略，输入是当前车间状态，输出是下一个作业的选择。在多目标场景下可以用多奖励架构。

---

*核心结论：生产调度是组合优化与多目标决策的完美交汇点。加权求和法帮助我们探索目标之间的权衡，甘特图让抽象的调度决策变得可视化。记住——不同的生产策略（交期优先 vs 效率优先）对应帕累托前沿上的不同区域，没有"绝对最优"的调度方案。*
