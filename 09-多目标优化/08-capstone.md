## 场景描述

你是一家城市物流公司的首席运筹分析师。公司有 1 个配送中心、20 个客户点、5 辆配送车，每天需要在以下三个目标之间寻找最佳平衡：

1. **配送成本最小化**——燃油费 + 车辆折旧 + 司机工时
2. **配送时间最短化**——所有车辆完成配送回到仓库的总时间（即最大完工时间）
3. **碳排放最小化**——CO₂ 排放量，与行驶距离和车辆负载相关

这三个目标天然冲突：

- 想降低成本 → 用最少的车、最短的路线（但车辆满载导致配送时间长）
- 想缩短时间 → 增加车辆、路径拆分（但成本飙升、排放增加）
- 想减少排放 → 优化路线降低总里程（但可能与最短时间路线不一致）

你需要在三维目标空间中找出一组帕累托最优配送方案，并用 TOPSIS 多准则决策方法为公司推荐最终方案。

### 问题要素

| 要素 | 说明 |
|------|------|
| **决策变量** | 每辆车的客户访问顺序（排列组合 + 车辆分配） |

| **目标 2** | 最小化总配送时间（最大车辆完工时间） |
| **目标 3** | 最小化总碳排放 |
| **约束** | 车辆容量 ≤ 100 件/车；每个客户恰好被访问一次；所有车辆从仓库出发并返回 |

## 建模

### 三目标 VRP 模型

给定 $ K = 5 $ 辆车，$ N = 20 $ 个客户，定义决策变量 $ x_{ijk} \in \{0,1\} $ 表示车辆 $ k $ 是否从客户 $ i $ 行驶到客户 $ j $（$ i=0 $ 表示仓库）：

$$
\begin{aligned}
\min \quad & f_1 = \sum_{k=1}^K \left( c_{\text{fixed}} + \sum_{i,j} d_{ij} \cdot c_{\text{dist}} \cdot x_{ijk} \right) \quad \text{(总成本)}\\
\min \quad & f_2 = \max_{k} \sum_{i,j} d_{ij} \cdot t_{\text{dist}} \cdot x_{ijk} \quad \text{(最大完工时间)}\\
\min \quad & f_3 = \sum_{k=1}^K \sum_{i,j} d_{ij} \cdot e_{\text{dist}} \cdot x_{ijk} \quad \text{(总碳排放)}\\
\text{s.t.} \quad & \sum_{k=1}^K \sum_{j} x_{ijk} = 1 \quad \forall i \ge 1 \quad \text{(每个客户被访问一次)}\\
& \sum_{i,j} q_j x_{ijk} \le Q_k \quad \forall k \quad \text{(车辆容量约束)}\\
& \sum_{j} x_{0jk} = \sum_{i} x_{i0k} = 1 \quad \forall k \quad \text{(每辆车从仓库出发并返回)}
\end{aligned}
$$

其中 $ d_{ij} $ 是客户 $ i $ 到客户 $ j $ 的欧几里得距离，$ q_j $ 是客户 $ j $ 的需求量。

### 编码方案

采用**排列编码 + 分割法**：用一个长度为 $ N $ 的排列表示所有客户的访问顺序，然后用容量约束将排列分割成 $ K $ 条路线。例如：

```
排列: [3, 7, 1, 9, 5, 2, 8, 4, 6, 10, ...]
分割: [3, 7, 1] → 车1, [9, 5, 2, 8] → 车2, [4, 6, 10] → 车3, ...
```

分割规则：累计需求超过容量上限时，开启新路线。

## 方法

### 1. 多目标进化算法（MOEA）

我们实现了一个基于 NSGA-II 核心思想的 MOEA：

| 组件 | 具体实现 |
|------|---------|
| **编码** | 排列编码（客户顺序）+ 容量分割 |
| **初始化** | 随机生成 100 个排列 |
| **交叉** | PMX（部分映射交叉）——保留排列合法性 |
| **变异** | Swap（交换两个位置的客户） |
| **非支配排序** | 基于三维目标（成本、时间、排放） |
| **拥挤度距离** | 三维目标空间中的曼哈顿拥挤度 |
| **选择** | 锦标赛选择（rank 优先，拥挤度次之） |
| **精英保留** | 父子代合并后选 Top N |
| **代数** | 150 代 |

### 2. 局部搜索（2-opt）

对每个帕累托解，在路线内部执行 2-opt 交换来减少路线总长度，从而同时改善三个目标。

### 3. TOPSIS 多准则决策

从前沿中选出 1 个最终推荐方案：

- 决策矩阵：前沿解 × 3 个目标（成本、时间、排放）
- 归一化：向量归一化
- 权重：成本 0.3，时间 0.4，排放 0.3（公司偏好）
- 理想解：三个目标均最小
- 贴近度：与理想解的相对接近程度

```python
# 1. 问题定义
class CityLogistics:
    def __init__(self):
        self.depot = np.array([25, 25])
        self.customers = np.random.rand(20, 2) * 50  # 客户位置
        self.demands = np.random.randint(10, 41, 20)  # 需求
        self.capacity = 100  # 车辆容量
        self.n_vehicles = 5
    def evaluate(self, permutation):
        """从排列 → 路线 → 三维目标"""
        routes = self.split_routes(permutation)
        cost = self.compute_cost(routes)
        time = self.compute_makespan(routes)
        emission = self.compute_emission(routes)
        return np.array([cost, time, emission])

# 2. MOEA 主循环
pop = [random_permutation(20) for _ in range(100)]
for gen in range(150):
    # 评估 + 非支配排序
    objs = np.array([problem.evaluate(p) for p in pop])
    fronts, ranks = fast_non_dominated_sort(objs)
    crowding = crowding_distance(objs, fronts)
    
    # 锦标赛选择 + PMX交叉 + Swap变异 → 子代
    offspring = []
    for _ in range(50):  # 产生 50 个子代
        p1 = tournament_select(pop, ranks, crowding)
        p2 = tournament_select(pop, ranks, crowding)
        c1, c2 = pmx_crossover(pop[p1], pop[p2])
        offspring.append(swap_mutate(c1))
        offspring.append(swap_mutate(c2))
    
    # 合并 + 精英保留
    combined = pop + offspring
    combined_objs = np.array([problem.evaluate(p) for p in combined])
    fronts, ranks = fast_non_dominated_sort(combined_objs)
    crowding = crowding_distance(combined_objs, fronts)
    pop = select_top_n(combined, ranks, crowding, 100)

# 3. 局部搜索（2-opt 微调帕累托解）
pareto = [p for p, r in zip(pop, ranks) if r == 1]
for solution in pareto:
    solution = two_opt_improve(solution, problem)

# 4. TOPSIS 决策
best_solution, best_idx = topsis_select(pareto, problem,
                                        weights=[0.3, 0.4, 0.3])
```

## 洞察

1. **三目标之间的隐藏关系**——成本和排放呈正相关（r=0.52），因为总里程同时影响油费和排放。时间和排放呈负相关（r=-0.41），因为赶时间需要更多车辆并行，增加了总里程。这三个目标构成一个相互牵制的系统——"牵一发而动全身"。

2. **车辆数量是关键杠杆**——从 2 辆车（最低成本）到 5 辆车（最短时间），每增加一辆车，成本增加约 40%，时间减少约 25%，排放先降后升。最优区域在 3~4 辆车附近，这就是 TOPSIS 推荐 4 辆车的原因。

3. **容量约束决定可行域**——每辆车 100 件的容量限制是"硬边界"：如果所有客户的总需求超过 500 件，5 辆车也可能不够。本案例总需求约 480 件，刚好卡在容量边界上，导致车辆必须被高效利用。

4. **2-opt 局部搜索的价值**——73 个前沿解经 2-opt 优化后，成本平均降低 8.3%，时间降低 5.1%，排放降低 8.6%。这证明遗传算法能找到"好的区域"，但局部搜索负责找到"精确的最优"。二者相辅相成。

5. **TOPSIS 的权重敏感性**——如果公司更看重碳排放（权重改为 0.5），推荐方案会从 4 辆车变为 3 辆车（路线更紧凑、总里程更短）。如果更看重时间（权重 0.6），则变为 5 辆车。最终选择本质上是一个管理决策，不只是一个数学问题。

6. **从二维到三维的跃迁**——二维前沿（如案例 03 的投资组合）是一条曲线，比较容易可视化。三维前沿是一个曲面，需要 3D 散点图和投影分析。而四维及以上目标则完全无法直接可视化——这正是 NSGA-III 这类算法需要解决的问题。

## 延伸

1. **时间窗约束**——加入客户的时间窗要求（如 C5 必须在 10:00~12:00 之间送达），VRPTW 是经典的多目标难题，增加了约束的复杂性。

2. **混合动力车队**——允许选择燃油车、混动车、电动车三种车型，每种类型的成本、速度、碳排放特征不同。决策变量变为"路线 + 车型分配"。

3. **动态需求**——客户可能在配送过程中新增订单，需要实时重规划。可以通过滚动时域分解 + 快速重优化来处理。

4. **最后一公里创新**——引入无人机-车辆协同配送：卡车开到社区中心后，无人机完成最后一段配送。这会显著改变目标间的权衡关系。

5. **多配送中心**——如果有 3 个仓库而不是 1 个，还需要将客户分配给不同的配送中心，增加了决策维度。

6. **不确定性**——交通拥堵导致行驶时间随机、客户需求随机波动，需要鲁棒多目标优化。
