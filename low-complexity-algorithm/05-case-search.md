# 案例5：有序空间快速搜索（二分答案+剪枝）★★☆☆☆

> 直接构造答案很难，但「猜一个答案再验证」却很简单。那就猜——用二分法猜得又快又准。

---

## 场景

你有一根长 10 米的钢管，要切成若干根长度相等的小段。你希望每段尽量长，但又要有足够段数来交差——比如至少切出 50 段。

你当然可以从 1 厘米开始试：1cm 行不行？2cm 行不行？3cm 行不行？……但这样试到可能的最大值要试 1000 次。如果有 10 万根钢管呢？

**二分答案**就是专门解决这类「猜一个数，然后判断它行不行」的问题。它把线性尝试从 O(n) 直接降到 O(log R)，其中 R 是答案的范围——1000 → 10 次，10 亿 → 30 次。

---

## 分析映射表

| 维度 | 说明 |
|------|------|
| **输入特征** | 答案是一个**连续数值**，且存在**单调性**（X 可行 ⇒ 所有 < X 或 > X 也都可行） |
| **输出特征** | 最大化/最小化某个数值（最大切割长度、最小耗时、最大边长） |
| **复杂度瓶颈** | 线性枚举答案范围太慢（O(R·f(n))），二分后降为 O(log R·f(n)) |
| **核心范式** | 二分查找（答案空间） + 贪心/简单模拟（判定函数） |
| **能否并行** | 理论上可以（每次二分只依赖判定函数，各次判定之间无关） |
| **思维难度** | ★★☆☆☆ —— 模板非常固定，难的是「识别出这是一个二分答案问题」 |

---

## 核心思路

### 直觉：你玩过「猜数字」吗？

一个人心里想一个 1~100 的数字，你每猜一次他告诉你「大了」还是「小了」。

- 菜鸟：1, 2, 3, 4, … 平均 50 次猜中
- 高手：50 → 25 → 37 → 43 → … 最多 7 次猜中

**二分答案就是把「猜数字游戏」用到算法问题上**——只是这里不猜具体的数，而是猜「答案的值」。你说「每段长 X cm」，然后问自己：「这个 X 可行吗？」——可行就试试更大的 X，不可行就试试更小的。

### 二分答案的三要素

```
┌─────────────────────────────────────────────────────────┐
│  二分答案 = 答案范围 + 判定函数 + 二分循环（6 行模板）  │
│                                                        │
│  1. 答案范围 [lo, hi]：答案不会小于 lo，不会大于 hi    │
│  2. 判定函数 can_do(X)：回答「X 作为答案行不行？」      │
│  3. 二分循环：根据 can_do(mid) 缩小 lo 和 hi            │
└─────────────────────────────────────────────────────────┘
```

### 三个场景的判定函数设计

**场景A：木材切割**

> 问题：N 根钢管，每根长 L[i]。要切出至少 K 段长度相等的小段，求每段的最大可能长度。

判定函数 `can_cut(length)`：
```
每段长 length 时，能切出多少段？
对所有钢管：总段数 += 钢管长度 // length
返回总段数 >= K
```

单调性：length 越小，段数越多 → 如果 length 可行，所有更短的 length 也一定可行。

**场景B：书籍抄写**

> 问题：M 本书，每本书页数 p[i]，K 个抄写员，每人连续抄一段（抄完一本才能下一本），求完成全部抄写的最短时间。

判定函数 `can_copy(limit)`：
```
每个抄写员最多抄 limit 页
贪心分配：从头开始，每本书加给当前抄写员
如果当前抄写员加上这本书会超过 limit，换下一个抄写员
如果需要的抄写员数 <= K，返回 True
```

单调性：limit 越大，需要的抄写员越少 → 如果 limit 可行，所有更大的 limit 也一定可行。

**场景C：分巧克力**

> 问题：N 块巧克力，每块尺寸 H[i] × W[i]。要切出至少 K 块正方形巧克力（只能沿格线切，不能拼接），求正方形最大边长。

判定函数 `can_cut_choco(side)`：
```
边长为 side 时，能切出多少块正方形？
对每块巧克力：(H[i] // side) * (W[i] // side)
返回总块数 >= K
```

单调性：side 越小，能切的块数越多 → 如果 side 可行，所有更小的 side 也一定可行。

```python

    ...

lo, hi = lower_bound, upper_bound
while lo < hi:
    mid = (lo + hi + 1) // 2  # 注意 +1 避免死循环
    if can_do(mid):
        lo = mid        # mid 可行，尝试更大的值
    else:
        hi = mid - 1    # mid 不可行，尝试更小的值
return lo
```

> ⚠️ 二分模板有多种变体（左闭右开、左闭右闭、找最小可行/最大可行）。上面的模板是「找最大可行值」的标准写法。关键是 `mid` 的计算和 `lo/hi` 的更新要保持一致。

## 关键洞察

### 洞察一：判定函数是二分答案的灵魂

**二分循环几乎永远是那 6 行代码。真正的工程在判定函数里。**

- 判定函数决定了你能不能二分——如果判定本身是 O(N²)，那总复杂度会变成 O(N² log R)
- 判定函数通常是贪心 + 简单模拟，复杂度尽量做到 O(N) 或 O(N log N)
- 优化判定函数能直接带来整体加速

### 洞察二：二分答案可以和 DP 配合

不是所有判定函数都是贪心。有些问题的判定函数本身就是一个 DP。

> 场景：你有 N 个任务，每个任务有 deadline 和 duration，你想知道在时间限制 T 内最多能完成多少个任务？
> 判定函数需要 DP 或贪心（按 deadline 排序 + 优先队列）来回答「T 单位时间内最多完成多少」。

二分答案 + DP 是一种强大的组合——外层二分猜答案，内层 DP 判定可行性。

### 洞察三：单调性不是二分的充分条件，是必要条件

**有单调性不一定能用二分答案**——你还需要判定函数本身是高效的。

反例：判断「是否存在一个子集，其和等于 X」——这是子集和问题（NP完全），判定函数本身是指数复杂度。二分范围再小也没用。

## 延伸与变形

| 变形 | 说明 |
|------|------|
| **浮点数二分** | 答案要求是浮点数时，用 `while hi - lo > eps` 代替 `lo < hi` |
| **三分查找** | 当函数是先增后减（凸函数）时用三分，不是二分 |
| **答案空间不是数值** | 离散空间也可以二分，比如在排序数组中找目标值的插入位置 |
| **二分+贪心（经典）** | 这是二分答案最常见的组合——木材切割、书籍抄写都是 |
| **二分+DP（进阶）** | 外层二分答案，内层 DP 判定，适合资源分配类问题 |
| **整体二分** | 一次性对多个查询二分，复用判定结果，适合离线场景 |
