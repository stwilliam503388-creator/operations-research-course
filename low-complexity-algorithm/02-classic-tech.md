# 第二部分：五大算法设计范式详解

> 不推导，不炫技。只讲「它解决什么问题 → 怎么想通 → 怎么在代码里用」。

## 2.1 一张图看懂算法设计范式的协作关系

不同的范式不是互相排斥的——很多时候你会组合使用。

```
                    ┌──────────────────────┐
                    │    原始问题 (n 规模)    │
                    └──────────┬───────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                 ▼
       ┌──────────┐     ┌──────────┐     ┌──────────────┐
       │ 可拆？   │     │ 有重叠？  │     │ 答案有边界？ │
       │ 子问题   │     │ 子问题   │     │ 单调可行？   │
       │ 独立？   │     │ 最优子   │     │              │
       │          │     │ 结构？   │     │              │
       └────┬─────┘     └────┬─────┘     └──────┬───────┘
            │                │                   │
            ▼                ▼                   ▼
     ┌──────────┐     ┌──────────┐     ┌──────────────┐
     │  分治    │     │   DP    │     │  二分答案     │
     │  O(n log n)   │  O(n·state)   │  O(log R·f(n))│
     └──────────┘     └──────────┘     └──────────────┘
            │                │
            └────────────────┼────────────────┐
                             ▼                ▼
                      ┌──────────┐     ┌──────────────┐
                      │  贪心    │     │  回溯+剪枝   │
                      │  局部=全局  │  │  穷举+提前停  │
                      └──────────┘     └──────────────┘
```

**核心思想**：大多数「看起来新」的问题，本质上是对这五个范式的组合或变体。学会识别问题 → 范式映射，就学会了 80% 的算法设计。

## 2.2 分治的深度剖析

### 分治 vs 动态规划：分界线在哪？

这是初学者最容易混淆的一对。我们用一张表说清楚：

| 维度 | 分治 | 动态规划 |
|------|------|---------|
| 子问题关系 | 互不重叠 | 重叠（不同路径到达相同状态）|
| 典型结构 | 二叉树——从上往下拆，从下往上合 | 有向无环图——状态之间有转移 |
| 重复计算 | 无（子问题独立） | 有（需要记忆化） |
| 能否用递归 | 天然适合 | 可以，但容易爆栈，推荐递推 |
| 典型复杂度 | O(n log n) | O(n·state) 通常更大 |

**什么时候分治自然演变成 DP？** 当你发现合并子问题时，需要用到同一个子问题的很多副本——这就是重叠子问题的信号，可以考虑 DP 了。

### 分治三连问

| 问题 | 答案 |
|------|------|
| 能不能拆？ | 输入是否能切割为互不相交的子集？ |
| 能不能合？ | 子问题的解能不能在 O(合并复杂度) 内生成原问题解？ |
| 能不能更快？ | 如果拆分成 k 份，合并复杂度是 O(k) 还是 O(k²)？ |

### 归并排序：分治的完美标本

归并排序的三个步骤无比清晰：

```
Step 1 — 拆：把数组从中间切两半
Step 2 — 解：递归地将两半各自排序
Step 3 — 合：两个已排序的数组合并成一个（双指针 O(n)）
```

```
输入: [38, 27, 43, 3, 9, 82, 10]

拆:
               [38, 27, 43, 3, 9, 82, 10]
               /                        \
        [38, 27, 43]                 [3, 9, 82, 10]
         /        \                  /           \
      [38]      [27, 43]          [3, 9]       [82, 10]
                /    \             /    \        /    \
              [27]   [43]        [3]   [9]    [82]   [10]

合 (合并两个已排序数组):
              [27, 38, 43]               [3, 9, 10, 82]
                    \                     /
                [3, 9, 10, 27, 38, 43, 82]
```

每一次合并操作：扫描两个子数组各一次 → O(n)。一共 log₂n 层 → 总复杂度 **O(n log n)**。

## 2.3 DP 的核心：状态设计是艺术

### 状态设计的三个层次

```
Level 1: 一维 DP → dp[i]
  语义：前 i 个元素 / 以 i 结尾的 ...
  典型题：最大子序和、最长递增子序列、爬楼梯
  复杂度：O(n) 或 O(n²)

Level 2: 二维 DP → dp[i][j]
  语义：前 i 个元素在「第二维约束 j」下的答案
  典型题：0/1 背包、LCS、编辑距离
  复杂度：O(n·m) 或 O(n·W)

Level 3: 状态压缩 DP → dp[mask]
  语义：mask 的二进制为 1 的位表示「已选」
  典型题：TSP DP、集合划分
  复杂度：O(n·2ⁿ) — n ≤ 20 时可用
```

### 经典案例：最长递增子序列 (LIS)

**问题**：给定数组 [10, 9, 2, 5, 3, 7, 101, 18]，求最长严格递增子序列的长度。

**Step 1：定义状态**
```
dp[i] = 以 nums[i] 结尾的最长递增子序列的长度
```

**Step 2：转移方程**
```
dp[i] = max(1, max(dp[j] + 1  for j < i and nums[j] < nums[i]))
```
含义：看 i 前面所有小于 nums[i] 的数 j，以 j 结尾的 LIS 加上 i 本人，就是候选值。取所有候选的最大值。

**Step 3：初始化**
```
dp[i] = 1    每个元素至少自己能构成一个长度为 1 的 LIS
```

**Step 4：遍历顺序**
```
从左到右：for i in range(n):
            for j in range(i):
                # 更新 dp[i]
```

**Step 5：返回结果**
```
max(dp)
```

**手动模拟 n=8**：

| i | nums[i] | dp[i] 计算过程 | dp[i] |
|---|---------|---------------|-------|
| 0 | 10 | 前面无元素，初始值 1 | 1 |
| 1 | 9 | j=0: 10>9, 不转移 | 1 |
| 2 | 2 | j=0:10>2, j=1:9>2 | 1 |
| 3 | 5 | j=2: 2<5, dp[2]+1=2 | **2** |
| 4 | 3 | j=2: 2<3, dp[2]+1=2 | 2 |
| 5 | 7 | j=3:5<7, dp[3]+1=3; j=4:3<7, dp[4]+1=3 | **3** |
| 6 | 101 | j=5:7<101, dp[5]+1=4; 还有很多 | **4** |
| 7 | 18 | j=5:7<18, dp[5]+1=4 | **4** |

答案：**dp[6] = 4**（子序列 [2, 5, 7, 101] 或 [2, 3, 7, 101]）

**复杂度**：O(n²) 时间，O(n) 空间。可优化至 O(n log n)（用二分维护 tails 数组）。

## 2.4 贪心：最难的不是写，是证明

### 贪心正确性的两个支柱

```
1. 贪心选择性质：局部最优选择最终一定是全局最优的一部分
   → 证明手段：交换论证（Exchange Argument）

2. 最优子结构：做出贪心选择后，剩下的子问题的最优解 = 原问题在剩余部分的最优解
   → 证明手段：反证法
```

### 交换论证模板

```
假设存在一个全局最优解 OPT。
如果 OPT 的第一步选择 ≠ 贪心选择：
  Step 1: 交换 OPT 中的「第一步」和「贪心选择」，得到新解 OPT'
  Step 2: 证明 OPT' 不比 OPT 差（目标函数值相等或更好）
  Step 3: 因此 OPT' 也是最优解
  Step 4: 归纳——对所有后续步骤重复，贪心选择构造出最优解
```

### 经典案例：活动安排

**问题**：给定 N 个活动，每个有开始时间 s_i 和结束时间 f_i。同一时间只能参加一个活动。求最多能参加几个。

**贪心策略**：每次选结束时间最早的活动。

**反证法正确性**：
如果最优解 OPT 的第一个活动不是结束时间最早的（假设是 a），而贪心选的是 b（最早结束的）。由于 b 结束时间 ≤ a 结束时间，把 OPT 中的 a 换成 b 后，剩下的活动不会减少（因为 b 结束得更早，给后面的活动留了更多时间）。所以存在一个包含 b 的最优解。归纳下去，贪心成立。

### 什么时候做「贪心错了」的快速测试

构造一个 n=3 或 n=4 的小输入，手动执行贪心策略，再用暴力穷举验证结果是否一致。

## 2.5 回溯+剪枝：暴力打不过就加条件

### 什么情况下回溯比 DP 更好？

| 情况 | 该用回溯还是 DP |
|------|---------------|
| 只需要一个解（而非最优值） | 回溯（DFS 找到即停） |
| 所有解都要列举 | 回溯 |
| n 很小且 DP 状态空间太大 | 回溯+强剪枝 |
| 需要最优值 | DP |
| 状态维数 ≤ 3 | DP（通常更快） |

### 剪枝的三个层次

```
Level 1 (基础)：可行性剪枝
  当前 partial 已不可能满足约束 → return
  例：N 皇后中，当前位置已被攻击

Level 2 (进阶)：最优性剪枝（用于求最优解）
  当前 partial 已不可能优于当前 best 值 → return
  例：已用成本 > 当前最优成本

Level 3 (高阶)：对称性剪枝
  两个分支本质等价 → 只探索一个
  例：组合选择中，选 {1,2} 和 {2,1} 是同一个组合
```

### 经典案例：N 皇后

**问题**：在 N×N 棋盘上放 N 个皇后，任何两个不能在同一行、同一列、同一条对角线上。

**搜索剪枝思路**：
- 按行放置皇后（每行放一个，自动满足「不同行」）
- 用 `cols`, `diag1`, `diag2` 三个集合记录已占用的列和两个方向的对角线
- 放皇后前检查位置是否冲突 → 冲突则剪枝

## 2.6 二分答案：很多人想不到它

### 识别二分答案的三个特征（全部满足才用）

1. **求最值**：最小化最大值 / 最大化最小值 / 求满足条件的最大/最小值
2. **有单调性**：如果 X 可行，所有大于（或小于）X 的值也一定可行
3. **判定函数简单**：给定一个 X，能在 O(n) 或 O(n log n) 内验证它是否可行

### 二分答案 vs 二分查找

| | 二分查找 | 二分答案 |
|--|---------|---------|
| 搜索对象 | 数组中的值 | 答案空间 |
| 前提 | 数组已排序 | 答案具有单调性 |
| 判定 | 比较大小 | 运行一个判定函数 |
| 复杂度 | O(log n) | O(log R · f(n)) |

**R = 答案范围，f(n) = 单次判定的时间复杂度。**

### 经典案例：木材切割

**问题**：有 N 根木材，长度分别是 L₁, L₂, ..., L_N。需要切出 K 段长度相同的小段（可保留余料）。求每小段的最大可能长度。

**判定函数**：
```
def can_cut(X):
    total = 0
    for L in woods:
        total += L // X     # 每根木材能切出 floor(L/X) 段
    return total >= K
```

**二分框架**：lo=1, hi=max(L)，找最大的 X 使得 can_cut(X)=True。

**复杂度**：O(N log max(L))——相比线性尝试的 O(N·max(L))，指数级加速。

## 2.7 进阶技术：滑动窗口与单调栈

### 滑动窗口（Sliding Window）

**直觉**：维护一个动态的「窗口」，在扫描数组时窗口内容不断变化，你实时维护窗口内的统计量。

**判断准则**：
- 处理的是**连续区间**（子数组、子串）
- 窗口的收缩条件与展开条件清晰

**模板骨架**：
```
left = 0
for right in range(n):       # right 是窗口右边界
    add(arr[right])          # 将右边界元素加入窗口
    while not_ok():          # 窗口不满足条件时收缩左边界
        remove(arr[left])
        left += 1
    update_answer()          # 此时窗口满足条件，更新答案
```

**典型问题**：长度最小的子数组（≥ target）、无重复字符的最长子串

### 单调栈（Monotonic Stack）

**直觉**：维护一个栈，栈内元素保持单调（递增或递减）。当你需要「快速找出数组中每个元素的下一个更大（或更小）元素」时——用单调栈。

**典型问题**：每日温度（下一个更大）、接雨水、柱状图中最大矩形

| 算法 | 时间复杂度 | 空间复杂度 | 稳定排序？ |
|------|-----------|-----------|-----------|
| 冒泡排序 | O(n²) | O(1) | ✅ |

## 2.9 选型决策树：怎么为新问题选范式

```
                    新问题
                       |
                  ┌────┴────┐
                  ▼         ▼
            n ≤ 20?      n > 20?
               │            │
               ▼            ▼
        暴力/回溯可行   ┌────┴────┐
         → 直接暴力     ▼         ▼
                   求最值?     求存在/计数?
                       │            │
                  ┌────┴────┐      ┌┴──────┐
                  ▼         ▼      ▼       ▼
            有重叠?    无重叠?   二分?  不用二分
               │         │        │       │
               ▼         ▼        ▼       ▼
              DP       分治     二分答案  线性扫描
                       │        或哈希
                  ┌────┴────┐
                  ▼         ▼
            局部=全局?   局部≠全局?
               │            │
               ▼            ▼
             贪心         回到 DP
                          或回溯+剪枝
```

> **使用这张决策树的关键**：如果你按路径走到叶子节点仍然没有明确的答案，没关系——你至少排除了 3-4 个不可能的方向。剩下的，试就完了。
