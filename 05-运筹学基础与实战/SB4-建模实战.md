# SB4：建模实战 — 产品组合优化

> 难度 ★★☆☆☆ · LP 建模 · 影子价格 · 灵敏度分析 · 代码：`code/python/case05_product_mix.py` · 预计阅读：25min

---

## 1 场景

你是工厂经理。工厂能生产三种产品（A、B、C），受原料、人工、机器三种资源限制。

**业务方的原话**："这个月原料库存 2000kg，工人满负荷 3000 小时，机器最多开 1500 小时。A 利润 120 一件、B 利润 80 一件、C 利润 150 一件。你帮我算算各产多少最赚钱？"

**隐含问题**（业务方没问但你必须回答的）：
- "如果原料不够了，多买 1kg 能多赚多少钱？" → 影子价格
- "如果工人要求加薪 10%，方案要不要变？" → 灵敏度分析
- "如果研发出了新产品 D，要不要上？" → 新产品评估

---

## 2 业务→模型映射表

| 业务要素 | 模型对应 | 数据 |
|---------|---------|------|
| A 产多少 | 决策变量 x_A | ≥ 0 |
| B 产多少 | 决策变量 x_B | ≥ 0 |
| C 产多少 | 决策变量 x_C | ≥ 0 |
| 原料 ≤ 2000kg | 约束：2x_A + 1x_B + 4x_C ≤ 2000 |
| 人工 ≤ 3000h | 约束：3x_A + 2x_B + 5x_C ≤ 3000 |
| 机器 ≤ 1500h | 约束：1x_A + 2x_B + 1x_C ≤ 1500 |
| 最大化利润 | 目标：120x_A + 80x_B + 150x_C |

---

## 3 数学模型（逐行注释式）

```
max  120·x_A + 80·x_B + 150·x_C        ← 总利润最大化
s.t.   2·x_A +  1·x_B +  4·x_C ≤ 2000  ← 原料限制（kg）
       3·x_A +  2·x_B +  5·x_C ≤ 3000  ← 人工限制（小时）
       1·x_A +  2·x_B +  1·x_C ≤ 1500  ← 机器限制（小时）
       x_A, x_B, x_C ≥ 0                ← 不能生产负数
```

**量纲检查**（建模第五步——最重要的 30 秒）：
```
目标：元 = 元/件 × 件  ✅
原料：kg = kg/件 × 件  ✅
人工：h  = h/件 × 件   ✅
机器：h  = h/件 × 件   ✅
```

---

## 4 求解方法选择

**算法决策树**（来自 OR 课程 2.2 节）：
- 线性？✅ 所有约束和目标都是线性的
- 连续？✅ 没有整数变量
- → **LP，单纯形法/内点法**

3 个变量、3 个约束——对求解器来说就是一瞬间。

## 6 运行结果

### 最优生产方案

```
产品 A: 生产 1000 件, 利润 120 元/件
产品 B: 生产 0 件
产品 C: 生产 0 件
总利润: 120,000 元
```

**结论**：全部资源用来生产 A。单件利润 A 居中，但原料/人工消耗最低，综合性价比最高。

### 资源利用率

```
原料: 2000/2000 (100%) ████████████████████  ← 瓶颈！
人工: 3000/3000 (100%) ████████████████████  ← 瓶颈！
机器: 1000/1500 (67%)  █████████████░░░░░░░  ← 产能过剩
```

### 灵敏度分析

| 场景 | 新利润 | Δ | 解读 |
|------|--------|---|------|
| 人工 +10% | 126,667 | +6,667 | 影子价格 ≈ 22.2 元/小时 |
| 原料 -10% | 116,000 | -4,000 | 影子价格 ≈ 20.0 元/kg |
| 机器 -10% | 120,000 | 0 | 非瓶颈，不影响 |

## ✅ 验证标准

1. 所有资源用量 ≤ 可用上限
2. 资源增加 → 利润增加（影子价格为正）
3. 非瓶颈资源减少 → 利润不变
4. 新产品引入 → 利润变化方向合理

## 7 关键洞察

### 洞察 1：影子价格 = 瓶颈的货币化

```
瓶颈资源（利用率 100%）→ 有正影子价格
非瓶颈资源（利用率 < 100%）→ 影子价格 = 0
```

本案例中：**原料值 20 元/kg，人工值 22.2 元/小时，机器值 0 元/小时。** 如果你只能投钱扩一项资源——扩人工。

### 洞察 2：不要相信单一的"最优方案"

方案说"只产 A，0 件 B 和 C"——但你不会真的只产 A。实际排产需要考虑：客户关系、市场风险、设备保养…**优化模型给出"数学最优"，业务决策在此基础上做"综合最优"**。

### 洞察 3：灵敏度分析回答最值钱的问题

"如果…那…"——这是业务方每天在问的。灵敏度分析就是系统性地回答这些"如果"。**一个模型如果没有灵敏度分析，它只回答了一半的问题。**
