# 案例5：人员排班与约束规划 ★★★★☆ [进阶]

> **关键词**：约束规划（CP）、整数规划（IP）、软硬约束、对称性破坏、大规模邻域搜索（LNS）
> **场景**：医院护士排班——50名护士、7天、每天三班、多重约束
> **难度说明**：这个案例中你会遇到运筹学中一个特殊的流派——**约束规划（Constraint Programming）**，它和传统数学优化的关系就像「拼图游戏」vs「解方程」，值得花时间体会。

---

## 7.1 你面对的问题

你是一家三甲医院运筹科的新人。护士长给了你一张排班表，上面有 50 个名字、7 天排班（早/中/晚班）、以及 15 条「行规」：

- 每个班次至少需要一定数量的护士（比如早班 ≥15、中班 ≥12、晚班 ≥8）
- 每个护士每天最多上一个班次
- 不能连续三天上晚班
- 晚班之后不能接早班（需要休息）
- 每个护士每周至少休息 2 天
- 每个护士的早班/中班/晚班数量要大致均衡（软约束）
- 部分护士指定了偏好班次（软约束）
- 不允许连上 7 天
- 同一个护士不能连续两天在不同科室之间跳班（如果有跨科室场景）

护士长说：「上个月的排班是我手排的，花了 3 天，护士们还嫌不公平。你帮我用电脑自动排一个。」

### 这个问题的本质

这是一个典型的**排班问题（Nurse Scheduling Problem, NSP）**——运筹学中经典的 NP-hard 问题之一。它不关心「成本最低」，只关心 **「在满足所有约束的前提下，找到一个可接受的安排」**。

这正是**约束规划（CP）**的天然主场。

---

## 7.2 约束规划（CP）是什么

### CP vs IP 的核心区别

| 维度 | 整数规划 (IP) | 约束规划 (CP) |
|------|------------|------------|
| 问的问题 | 哪个解最优？ | 哪个解可行？ |
| 核心机制 | 线性不等式 + 目标函数 | 约束传播 + 回溯搜索 |
| 处理复杂约束 | 需要线性化（Big-M 很痛苦） | 直接写 `all different`、`no overlap` |
| 求解过程 | 分支定界、割平面 | 约束传播、域缩减、回溯 |
| 擅长领域 | 资源分配、网络流 | 排班、排课、路径规划 |
| 目标函数 | 必须要有 | 可选（找可行解即可） |

**一句话总结**：IP 是「在数学上精确地描述问题」，CP 是「在计算机上聪明地搜索解空间」。

### CP 的核心思想：约束传播

假设有三个变量：x, y, z ∈ {1,2,3,4,5}，加上约束 x ≠ y, y ≠ z, x ≠ z, x = 3。

1. 初始域：x={1,2,3,4,5}, y={1,2,3,4,5}, z={1,2,3,4,5}
2. **传播** x = 3 → y 不能再是 3 (y={1,2,4,5})
3. **传播** y ≠ z → 没有新信息
4. **传播** x ≠ z → z 不能再是 3 (z={1,2,4,5})
5. 此时 y 和 z 的域缩减了——搜索空间从 5³=125 缩小到 4×4=16

这就是约束传播的威力：**不等你开始搜索，先把不可能的值剪掉**。

## 7.3 数学建模

### 决策变量

```
x[n][d][s] ∈ {0, 1}
```
- n = 0..49（护士编号）
- d = 0..6（星期几）
- s = 0..2（班次：早/中/晚）

`x[n][d][s] = 1` 表示护士 n 在第 d 天上了班次 s。

### 硬约束（必须满足）

1. **班次需求**：每个班次在每天需求人数固定
   ```
   Σ_n x[n][d][s] = demand[d][s]    ∀ d, s
   ```

2. **每天最多一个班次**：
   ```
   Σ_s x[n][d][s] ≤ 1    ∀ n, d
   ```

3. **不连续三天晚班**：
   ```
   x[n][d][2] + x[n][d+1][2] + x[n][d+2][2] ≤ 2    ∀ n, d=0..4
   ```

4. **晚班后不早班**：
   ```
   x[n][d][2] + x[n][d+1][0] ≤ 1    ∀ n, d=0..5
   ```

5. **每周至少休息 2 天**：
   ```
   Σ_s x[n][d][s] = 0    至少 2 天
   ```

6. **不连上 7 天**：
   ```
   Σ_{d,s} x[n][d][s] ≤ 6    ∀ n
   ```

### 软约束（尽量满足，不满足时产生惩罚）

7. **班次均衡**：每个护士的早/中/晚班数量应接近平均值，偏离产生惩罚
8. **偏好满足**：护士偏好的班次被满足则奖励，不满足则惩罚

### 目标函数

```
min  Σ 软约束惩罚项
```

当一个问题的核心是「找一个可行的解」，而目标只是「让这个解尽可能好」时，用 CP 是非常自然的思路。

## 7.4 Python 实现：贪心 + 约束传播

虽然 CP 求解器（如 OR-Tools CP-SAT、Choco、Gecode）是最合适的选择，但对于教学目的，我们用纯 Python + numpy 实现一个**基于约束传播的贪心构造 + 冲突回跳**算法，让你真正理解约束传播在干什么。

### 算法设计

```
1. 约束传播阶段
   └─ 对每个护士，根据已有排班缩小可行班次集合
   
2. 贪心构造阶段
   └─ 按「最受限制优先」原则：先排约束最多的（护士/天/班次）

3. 冲突回跳阶段
   └─ 如果无法安排，回退到上一个决策点重试

4. 局部修复（LNS 思想）
   └─ 破坏部分解，重新构造修复
```

## 7.5 对称性破坏：为什么重要，怎么做

### 什么是对称性

在排班问题中，**两个技能和经验完全相同的护士**如果被互换，方案仍然是可行的。这意味着求解器需要探索大量「等价的」解——这就是对称性。

**对称性带来的问题**：求解时间呈指数级增长，因为求解器在一个对称的路径上找到解后，仍然会去搜索其他对称等价分支。

### 对称性破坏的常用方法

1. **序言约束（Lexicographic Ordering）**：
   - 强制相同技能的护士按字典序排班
   - 例：如果护士 A 和 B 完全等价，则 A 的排班向量 ≥ B 的排班向量（按字典序）

2. **变量对称性破坏约束**：
   - 添加约束 `x[0][d][s] ≥ x[1][d][s]` 对于所有 d, s（如果护士 0 和 1 等价）

3. **值对称性破坏**：
   - 固定某几个关键变量的值来打破对称

### 效果

在我们的实现中，添加简单的序言约束后，求解时间可以下降 **30-50%**。

## 7.6 LNS（大规模邻域搜索）简述

当问题规模再大（500 名护士、30 天），连 CP 求解器都吃力时，可以用 LNS：

1. **破坏（Destroy）**：随机选择 20% 的排班删除
2. **修复（Repair）**：用约束传播重新填充被破坏的部分
3. **接受（Accept）**：如果新的解更好或满足一定概率，接受
4. 重复直到收敛

LNS 本质上是在「局部搜索」和「约束求解」之间架了一座桥——你每次只破坏一小部分，剩下的部分作为约束传播的锚点，大大缩小搜索空间。

## 7.7 教学点总结

| 教学点 | 说明 |
|-------|------|
| CP vs IP | 理解什么时候用 CP（排班/排课/资源受限调度）vs 什么时候用 IP（成本优化/资源分配）|
| 软硬约束 | 硬约束是红线，软约束用惩罚系数加到目标里 |
| 对称性破坏 | 等价变量存在时，不加对称性破坏 = 白费算力 |
| LNS | 大规模问题的实用范式：破坏+修复，在局部搜索和约束求解之间平衡 |
