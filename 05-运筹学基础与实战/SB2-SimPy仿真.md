# SB2：SimPy 仿真 — 当数学不够用时

> 难度 ★★★☆☆ · 离散事件仿真 · 排队论 · 代码：`code/python/case04b_simpy_queue.py` · 预计阅读：25min

---

## 1 场景

你已经用 M/M/c 排队论公式算过急诊科等待时间。但业务方说：

> **"我们的病人到达率白天和晚上不一样。危重病人插队。周末医生少一个。"**

排队论的解析公式在"到达率随时间变化 + 多优先级 + 非齐次服务率"面前直接失效。你不是数学不够好——是**数学在这个复杂度上已经不够用了**。

你需要仿真。仿真不问你能不能写出闭式解——它直接跑给你看。

---

## 2 分析：仿真 vs 解析排队论

| 层面 | 解析排队论 (M/M/c) | 离散事件仿真 (SimPy) |
|------|-------------------|--------------------|
| 到达模式 | 必须齐次 Poisson | 任意（时变、批量、相关）|
| 服务时间 | 必须指数分布 | 任意分布 |
| 优先级 | 不支持（或极其复杂）| `PriorityResource` 一行搞定 |
| 结果精度 | 精确（在假设成立时）| 统计（需足够多次运行）|
| 开发时间 | 公式代入，10 分钟 | 写仿真模型，1-2 小时 |
| 运行时间 | 毫秒 | 秒~分钟 |

**选择原则**（来自 OR 课程 2.2 节）：
- 解析解可用 → 用解析解（快、精确）
- 解析解不可用 → 用仿真（灵活、可验证）
- 两者都做 → 先用解析解验证仿真，再用仿真做解析解做不了的场景

## 3 核心思路

### 3.1 SimPy 的三个基本概念

```
Environment（环境）     → 仿真时钟
Process（进程/生成器）   → 自然语言描述的行为
Resource（资源）        → 有限的共享资源（医生、收银台、服务器）
```

**生活类比**：SimPy 像拍电影。
- `Environment` = 摄影棚的时钟（第 1 天、第 2 天…）
- `Process` = 剧本（病人到达→排队→看病→离开）
- `Resource` = 道具（医生 X 个，一次只能看一个人）

### 3.2 最简单的 SimPy 模型

```python
import simpy
import random

def patient(env, name, doctor, service_time):
    """一个病人的完整流程"""
    arrive = env.now                        # 记录到达时间
    with doctor.request() as req:           # 排队等医生（自动排队！）
        yield req                           # 拿到医生
        wait = env.now - arrive             # 等待时间
        yield env.timeout(service_time)     # 看病

def setup(env, doctor, lam):
    """病人到达生成器"""
    i = 0
    while True:
        yield env.timeout(random.expovariate(lam))  # 间隔到达
        env.process(patient(env, f'P{i}', doctor, random.expovariate(1/30)))

env = simpy.Environment()
doctor = simpy.Resource(env, capacity=2)     # 2 个医生
env.process(setup(env, doctor, lam=0.5))     # 每小时 0.5 人到达
env.run(until=24*7)                          # 跑一周
```

### 3.3 M/M/1 验证（核心验证手段）

仿真做对了没有？先用解析解验证：

```python
# 解析解
L = rho / (1 - rho)    # L = λ/(μ-λ)
W = 1 / (mu - lambda)  # 平均等待时间

# 仿真统计
W_sim = np.mean(waits)       # 从仿真收集
L_sim = lambda * W_sim       # Little 定律 L = λW

# 验证：偏差 < 10%
assert abs(L_sim - L) / L < 0.10
```

## ✅ 验证标准

1. M/M/1 特殊情况：仿真结果与解析解偏差 < 10%（排除模型实现错误）
2. Little 定律验证：L = λW（L/W = λ 误差 < 5%）
3. 增加医生后等待时间下降（方向正确）
4. 优先级越高等待越短（危重 < 紧急 < 常规）

## 6 关键洞察

### 洞察 1：先验证，再仿真

永远先用一个解析可解的特例验证你的仿真代码。不经过验证的仿真结果是数字垃圾。这是 OR 课程附录 B 的"TOP 5 必踩坑"第 0 条。

### 洞察 2：仿真不是用来"算答案"的，是用来"比方案"的

仿真给不出"最优解"。它告诉你：方案 A 的平均等待 60 分钟，方案 B 是 45 分钟。**哪个更好你判断，仿真只给你数字。**

### 洞察 3：仿真时间要够长

高利用率排队（ρ > 0.8）需要极长仿真才能达到稳态。案例中用了 50 万时间单位来验证 M/M/1。**生产仿真跑 1 分钟是不够的。**
