# 案例 3：牛鞭效应仿真 ★★★☆☆

> 难度：★★★☆☆ · 模型类型：系统仿真 / 多级供应链 · 求解方法：离散事件仿真 + 统计比较 · 代码路径：`code/case05_bullwhip.py`

---

## 1 场景

你运营一个 4 级供应链：**零售商 → 批发商 → 分销商 → 工厂**。每一级都使用 (s, S) 库存策略管理库存。零售端的终端消费者需求服从独立同分布的正态分布 N(100, 20)。你的目标是观察和量化"牛鞭效应"——需求信息沿供应链向上游传递时被逐级放大的现象。

**教学点**：
- 牛鞭效应的产生机制——每级只能看到下级订单，看不到真实需求
- 信息共享如何缓解牛鞭效应（POS 数据共享）
- 提前期和批量参数对放大系数的影响
- 上游订单方差 > 下游订单方差的量化验证

---

## 2 博弈建模（参与者 / 策略 / 收益）

| 要素 | 说明 |
|------|------|
| **参与者** | 4 级节点：零售商(R)、批发商(W)、分销商(D)、工厂(F) |
| **每级策略** | (s, S) 库存策略：当库存位置 IP ≤ s 时订货至 S；否则不订货 |
| **信息结构** | 每级只能看到下级发来的订单（非共享模式）或也能看到终端需求（共享模式） |
| **每级收益** | 成本最小化（持有成本 + 缺货成本 + 订货成本），但每级独立决策 |
| **博弈性质** | 多级顺序博弈，每级做最优决策，但信息延迟和批量订货导致整体效率下降。这是"局部最优→整体次优"的经典例证 |

牛鞭效应不是恶意博弈的结果——每级都在独立做合理的库存决策，但由于信息结构的限制，整体表现偏离最优。

## 3 数学模型（逐行注释式）

### 业务→模型映射表

| 业务要素 | 模型符号 | 说明 |
|---------|---------|------|
| 供应链层级 | 4 级 | 零售商→批发商→分销商→工厂 |
| 每级库存策略 | (s, S) | 当库存位置 IP ≤ s 时订货至 S |
| 零售端需求 | D ~ N(100, 20) | 独立同分布，每周期 |
| 提前期（级间） | L = 2 周期 | 从下单到到货的周期数 |
| 订货点 | s = μ×L + z×σ×√L | 基于提前期需求的安全库存计算 |
| 目标库存 | S = s + Q | Q 为基础批量 |
| 放大系数 | σ(上级订单) / σ(终端需求) | 衡量牛鞭效应严重程度 |

### 每级库存仿真逻辑

```
每周期 t 执行：
  1. 到货处理：从在途队列中取出提前期满的货物，加入库存
  2. 接收需求（或下级订单）：来自下一级的订货量
  3. 发货：取 min(当前库存, 收到的需求)
  4. 计算缺货：需求 - 发货，累计缺货量
  5. 库存位置 IP = 当前库存 + 在途库存 - 缺货量
  6. 决策：如果 IP ≤ s，订货量 = S - IP；否则订货量 = 0
  7. 订单发出：放入在途队列（lead_time 后到货）
  8. 记录：记录本周期发出的订单量
```

### 牛鞭效应量化

```
无信息共享模式：
  放大系数(工厂) = σ(工厂接收订单) / σ(零售端需求)
  → 理论预期 > 1.5

信息共享模式：
  上游可以看到零售端的真实 POS 数据
  在做预测时直接用真实需求而非被放大的订单
  → 放大系数可降至 1.0~1.2
```

| 总库存成本 | 基准 | -35% | — |

信息共享越往上游改善越显著——因为上游受信息扭曲的累积效应最大。

### 提前期灵敏度

| 提前期 L | 工厂σ | 放大系数 |
|----------|-------|---------|
| 1 | 45.7 | 2.30 |
| 2 | 72.3 | 3.64 |
| 3 | 98.1 | 4.94 |

提前期越长，牛鞭效应越严重。缩短提前期是缓解牛鞭效应的有效手段。

## 5 代码实现概览

```python
import numpy as np

class SupplyChainNode:
    """供应链节点：维护库存、处理订单、记录数据"""
    def __init__(self, name, s, S, lead_time):
        self.name = name
        self.s = s
        self.S = S
        self.inventory = S
        self.lead_time = lead_time
        self.pipeline = [0] * lead_time  # 在途队列
        self.backlog = 0
        self.orders_history = []
        self.shipments_history = []
    
    def step(self, demand, shared_demand=None):
        """一个周期的运营"""
        # 1. 到货
        self.inventory += self.pipeline.pop(0)
        self.pipeline.append(0)
        
        # 2. 发货
        shipment = min(self.inventory, demand)
        self.inventory -= shipment
        self.backlog += (demand - shipment)
        self.shipments_history.append(shipment)
        
        # 3. 订货决策
        ip = self.inventory + sum(self.pipeline) - self.backlog
        if ip <= self.s:
            order = self.S - ip
        else:
            order = 0
        
        self.pipeline[-1] = order
        self.orders_history.append(order)
        return order

def run_simulation(nodes, n_periods=200, warmup=50):
    """运行多级供应链仿真"""
    # 初始化
    for _ in range(n_periods + warmup):
        demand = np.random.normal(100, 20)
        demand = max(0, int(demand))
        # 传递订单
        d = demand
        for node in nodes:
            d = node.step(d)
    # 返回各节点订单历史
```

## 7 关键洞察

- **牛鞭效应不是"非理性决策"所致**：即使每级都使用最优的 (s, S) 策略，牛鞭效应依然存在。它是信息延迟和批量订货的系统性结果。
- **信息共享是最有效的缓解手段**：让上游直接看到真实需求数据（POS 共享），而不是从被放大的订单中"反推"需求。改善幅度可达 50%+。
- **促销人为制造牛鞭效应**：促销导致需求尖峰，经过供应链放大后波及整个系统。促销带来的增量利润可能远小于其引发的供应链额外成本。
- **牛鞭效应的代价远不止库存成本**：产能波动、紧急运输、加班费、人员流动——所有成本都被放大了。这也是为什么"需求平稳"对供应链如此重要。

## 8 延伸与变形

- **啤酒游戏**——经典课堂仿真，强烈推荐实际玩一次，亲身体验牛鞭效应
- **加入促销的价格波动仿真**——促销制造需求尖峰，观察放大效应变化
- **VMI（供应商管理库存）**——供应商直接获取零售端销售数据，进一步降低牛鞭效应
- **不同预测方法的比较**：移动平均 vs 指数平滑 vs 完美信息下的订货波动
- **集中式需求信息（CPFR）**——协同预测、补货与计划，供应链整体最优
