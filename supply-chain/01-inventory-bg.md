# 第一部分：库存管理概览

> 在这一章，你不会看到一行代码。你会看到：库存管理是怎么从一个仓库记账工具变成今天供应链核心决策引擎的？一个补货问题怎么被拆解成「成本结构 / 需求模式 / 服务水平」这三个要素？以及——四大经典库存模型分别解决什么问题？

---

## 1.1 没有库存管理之前，世界是什么样子的

先想象三个场景。

**场景 A：缺货**。双十一当天，你的爆款商品在开售 2 小时后卖光了。系统显示「库存为 0」，但实际需求是库存的 5 倍。你失去了 500 万的销售额，还让用户跑去竞争对手那里。客服被骂了一天。供应链总监问：「为什么不多备一些？」采购经理说：「多备了万一卖不完呢？」

**场景 B：积压**。去年你多备了 3 个月的安全库存——结果竞品出了个更好的产品，你的库存全部变成滞滞品。仓储费每月 20 万，财务说「这批货的持有成本已经超过了货值本身」。你不得不打折清仓——亏了 800 万。

**场景 C：牛鞭效应**。零售端需求只波动了 5%。零售商为了安全多订了 10%。批发商看到订单涨了 10%，又加了一点保险——向分销商订了 20% 更多。分销商看到趋势，又订了 30% 更多。最后工厂看到的订单波动是 50%。工厂不得不开两班倒 + 紧急采购——然后下个月订单突然回落，工厂产能过剩。

这三个场景的共同主题：**库存是必要的恶——太少会缺货、太多会积压、错配会放大。**

### 从 EOQ 到智能供应链：一段简史

| 年份 | 人物/事件 | 贡献 |
|------|----------|------|
| 1913 | Ford Harris | 发明 EOQ 公式——库存管理的起点 |
| 1934 | Wilson | 独立推导 EOQ（EOQ 也被称为 Wilson 公式） |
| 1951 | Arrow, Harris, Marschak | 报童问题的数学奠基——不确定需求下的最优订货 |
| 1958 | Magee | (s,S) 策略的直觉——定期盘点+订货点 |
| 1960s | Clark, Scarf | 多级库存系统的最优策略——串行供应链的分析基础 |
| 1985 | Lee, Padmanabhan, Whang | 牛鞭效应的正式理论分析——解释了信息扭曲的放大的机制 |
| 1990s~ | 实践爆发 | VMI、CPFR、供应链契约（回购/收益共享）大规模应用 |
| 2010s~ | AI+供应链 | 需求预测 ML + 库存优化 + 端到端可视化 |

### 一个简单的思想实验

假设你开了一家小卖部，每天卖大约 10 瓶可乐。进货要 3 天才能到。你每天观察销量，觉得「差不多该进货了」就去进货。这种做法叫**凭经验补货**——在 99% 的小店里仍然在用。

问题是：
- 周一卖得好，你多进了一些。周二卖得少，库存积压了。
- 你不知道「最优」是多少——全凭感觉。
- 你无法回答这个问题：「我应该保持多少安全库存？」

库存模型就是回答这些问题的。它不神奇——它只是把「凭经验」变成了「算一下」而已。

## 1.2 库存管理的四个核心矛盾

| 矛盾 | 描述 | 你面对的取舍 |
|------|------|------------|
| 持有成本 vs 缺货成本 | 存货越多持有成本越高，越少缺货风险越大 | 最优解在两者平衡点 |
| 规模经济 vs 灵活度 | 一次订越多单件成本越低，但灵活性降低 | EOQ vs JIT |
| 服务水平 vs 库存成本 | 99% 服务水平需要的库存是 95% 的 2 倍以上 | 边际递减——最后 1% 最贵 |
| 局部最优 vs 全局最优 | 每个节点优化自己 vs 供应链整体最优 | 牛鞭效应的根源 |

**先记住一句话**：库存管理的本质不是「减少库存」，而是「在正确的服务水平下最小化总成本」。

这四个矛盾是贯穿整门课程的主线。每当你觉得某个库存决策很纠结，往回看这四对矛盾，大概率能帮你理清思路。

## 1.3 四大经典库存模型

### 1. EOQ — 确定需求下的最优批量

最简单的起点。假设需求已知且恒定，每次订货有固定成本 K，每单位每周期有持有成本 h。

```
总成本 = 年订货次数 × K + 平均库存 × h
       = (D/Q) × K + (Q/2) × h

最优订货量 Q* = sqrt(2·D·K / h)
```

**核心直觉**：订货多→订货次数少（省固定成本）→但平均库存高（持有成本高）。最优解在两者相等处。

**灵敏度**：Q* 的误差对总成本影响很小——即使 Q 偏差 30%，总成本也只增加约 5%。所以 EOQ 是一个「容错率高」的模型。

### 2. 报童问题 — 单周期不确定需求

一次性的订货决策（季节性商品、生鲜、报纸）。需求不确定，已知分布。订多了浪费，订少了亏。

```
Cu = 缺货成本（少卖一件的利润损失）
Co = 超量成本（多订一件的损失）

最优服务水平 = Cu / (Cu + Co)   ← 关键公式
最优订货量 = F^{-1}(Cu / (Cu + Co))   ← 需求分布的该分位数
```

**核心直觉**：缺货成本越高，你该订越多；超量成本越高，你该订越少。比率 Cu/(Cu+Co) 决定最优分位数。

**实际意义**：如果一件货缺货的损失是超量损失的 4 倍（Cu=4, Co=1），最优服务水平是 80%——你该订到需求分布 80% 分位数的量。

### 3. (s,S) 策略 — 多周期不确定需求

每周期检查库存。当库存降到 s（订货点）时，订货到 S（目标库存水平）。

```
s = 订货点 = 提前期内的期望需求 + 安全库存
S = 目标库存水平 = s + Q*（或通过 DP 求解）
安全库存 = z × σ × sqrt(L)   ← z 由服务水平决定，L = 提前期
```

**核心直觉**：s 负责防止缺货，S-s = Q 负责平衡订货与持有成本。

**实际应用**：这是现实中用得最多的模型。绝大多数 ERP 系统的补货逻辑都基于 (s,S) 或其变体。

### 4. Base Stock 策略 — 单周期订货

每周期检查库存，订到基准库存水平 S。如果当前库存 < S，订 S - 当前库存这么多。

**核心直觉**：适合短提前期、高服务要求的场景。供应链中常作为协同基准。

| 模型 | 需求 | 周期 | 关键参数 | 适用场景 |
|------|------|------|---------|---------|
| EOQ | 确定 | 多周期 | Q* (批量) | 稳定需求的常规补货 |
| 报童 | 不确定 | 单周期 | 服务水平分位数 | 季末商品、生鲜、报纸 |
| (s,S) | 不确定 | 多周期 | s (订货点), S (目标库存) | 常规物品的再补货 |
| Base Stock | 不确定 | 多周期 | S (基准库存) | 短提前期、高周转物品 |

### 进阶模型（先了解，用到再深学）

**多级库存 (Multi-Echelon)** — 你的库存不只是你的库存

```
工厂 → 区域仓库 → 零售店
```

各级库存是耦合的。区域仓库的库存策略直接影响零售店的缺货率。Clark-Scarf 模型是理论基础。现实世界中，大部分企业只优化自己这一级——这就是为什么供应链整体效率往往很差。

**联合补货 (Joint Replenishment)** — 多个产品一起订

如果你有 1000 个 SKU，每次无论订哪种产品都有固定成本 K0，每种产品还有各自的固定成本 Ki。最优策略比单产品 EOQ 复杂得多——需要平衡补货频率和批次。实际中常用启发式算法（如 Roundy 的 56% 近似策略）。

## 1.4 供应链场景建模五步法

### 第一步：画供应链结构（2 分钟）

实体（方框）+ 流向（箭头）+ 关键参数。谁向谁订货？运输时间多长？库存放在哪？

**示例**：
```
[供应商] --(提前期 5天)--> [仓库] --(提前期 2天)--> [零售店]
                            ↑                        ↑
                         持有成本 h=0.3/件/月    缺货成本 Cu=15/件
```

画完这张图，你就能回答 80% 的结构问题。

### 第二步：确认需求模式（1 分钟）

- 需求是确定的还是随机的？
- 如果是随机的，分布已知吗？均值？方差？
- 单周期还是多周期？
- 需求有趋势/季节性吗？

**注意**：需求模式的判断直接影响模型选择。把随机需求当成确定需求来建模，是初学者最容易犯的错误之一（详见附录B）。

### 第三步：列成本项（1 分钟）

| 成本类型 | 符号 | 说明 |
|---------|------|------|
| 固定订货成本 | K | 每次下单的行政/运输固定费用 |
| 持有成本 | h | 每单位每月的仓储/资金占用 |
| 缺货成本 | Cu | 每缺一件的损失（利润损失 + 信誉损失）|
| 超量成本 | Co | 每剩一件的损失（进货成本 - 残值）|

**Tip**：缺货成本最难量化。一个实用的方法是：用服务水平倒推——管理层说「我们要 95% 服务水平」，你就能反算出他们心里认定的缺货成本大约是多少。

### 第四步：选模型（30 秒）

```
确定需求 + 多周期 → EOQ
不确定 + 单周期 → 报童
不确定 + 多周期 + 连续盘点 → (s,S)
不确定 + 多周期 + 定期盘点 → Base Stock
不确定 + 多级 + 多产品 → 进阶模型
```

### 第五步：验证直觉（30 秒）

做灵敏度分析：改变一个参数，看结果是否符合直觉。如果不符合，要么模型错了，要么你的直觉需要更新。

- 固定成本 K 增大 → 批量增大（✓）
- 持有成本 h 增大 → 批量减小（✓）
- 服务水平要求提高 → 安全库存增加（✓）
- 提前期变长 → 安全库存增加（✓）

**一个反直觉的例子**：需求波动 σ 增大 → 安全库存增加（✓ 直觉正确）。但如果在多级供应链中，某个节点只看到被放大过的订单波动 → 该节点对安全库存的需求也会被放大（牛鞭效应）。这就是为什么「你做对了自己这一步，但整个系统仍然糟糕」。
