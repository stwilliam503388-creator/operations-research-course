## 1 项目背景

你是一家消费品公司的供应链总监，负责设计并优化一条三级供应链：**工厂 → 配送中心(DC) → 零售店**。这是一个综合性的毕业项目，要求你运用课程中学到的所有核心知识——EOQ 批量决策、报童模型与服务水平、设施选址优化、库存策略设计，以及供应链契约协调——来解决一个接近真实行业场景的复杂问题。

## 4 分步建模与分析

### Step 1：设施选址（战略层）

**问题**：从 3 个候选位置中选择建几个 DC、在哪建，最小化建设成本 + 运输成本。

**MIP 模型**：
```
决策变量：
  y_j ∈ {0,1}      # 在候选 j 建 DC
  x_ij ∈ {0,1}     # 将零售店 i 分配给 DC j

目标：min Σ_j f_j·y_j + Σ_i Σ_j t_ij·d_i·x_ij

约束：
  Σ_j x_ij = 1           ∀i     # 单分配
  x_ij ≤ y_j             ∀i,j   # 只有建了才能分配
  Σ_i d_i·x_ij ≤ cap_j  ∀j     # 容量约束
  Σ_j y_j ≥ 1                   # 至少建1个
```

**验证**：选址方案满足容量约束，每个零售店有唯一分配的 DC。

### Step 2：库存策略设计（战术层）

**每个零售店（报童模型）**：
```
Cu = p - w  （缺货成本 = 售价 - 进货价）
Co = w - s  （超量成本 = 进货价 - 残值）
最优服务水平 SL* = Cu / (Cu + Co)
安全库存 SS = z(SL*) · σ · √L
订货点 s = μ·L + SS
EOQ = √(2·D·K / h)
目标库存 S = s + EOQ
```

**DC（风险分担）**：
```
DC 的安全库存考虑合并效应：
  SS_DC = z(0.98) · √(Σσ_i²) · √(L_DC)
DC 的 (s, S) 参数基于下级零售店的总需求
```

**验证**：仿真中零售店缺货率 < 5%，DC 缺货率 < 2%。

### Step 3：联合优化与仿真评估（运营层）

将选址和库存策略组合，运行 500 个周期的离散事件仿真。

**仿真逻辑**：
```
每周期 t：
  for 每个零售店 i:
    1. 生成需求 d_it ~ N(100, 20)
    2. 从本地库存发货（min(库存, 需求)）
    3. 检查是否需要向 DC 订货（(s,S) 策略）
    4. 记录销量、缺货、库存水平
  
  for 每个 DC j:
    1. 接收零售店的订单汇总
    2. 从本地库存发货
    3. 检查是否需要向工厂订货
    4. 记录库存水平、缺货情况
  
  工厂：按 DC 订单生产（受产能约束）
```

### Step 4：灵敏度分析

| 参数变化 | 预期影响 |
|---------|---------|
| 服务水平 95%→99% | 安全库存大幅增加，总成本 ↑20-40% |
| 需求波动 σ=10→σ=40 | 风险分担效应增强，更倾向于集中化（少建 DC） |
| 持有成本 ↑50% | EOQ 减小，安全库存减少，总成本上升 |
| DC 固定成本 ↑100% | 更倾向于合并 DC（少建） |

# ========== Step 2: 库存策略 ==========

def newsvendor_ss(mu, sigma, Cu, Co, lead_time):
    """基于报童模型计算安全库存"""
    sl = Cu / (Cu + Co)
    z = stats.norm.ppf(sl)
    ss = z * sigma * np.sqrt(lead_time)
    return ss, sl

def eoq_batch(D, K, h):
    """EOQ 批量"""
    return np.sqrt(2 * D * K / h)

def design_inventory_policy(assignments, stores, ...):
    """为每个 DC 和零售店设计 (s, S) 策略"""
    policies = {}
    for store_id, dc_id in assignments.items():
        store = stores[store_id]
        ss, _ = newsvendor_ss(store['mu'], store['sigma'], 
                              Cu=6, Co=3, lead_time=1)
        q = eoq_batch(store['mu'] * 365, K=100, h=store['h'])
        s = int(store['mu'] * 1 + ss)  # 订货点
        S = int(s + q)                   # 目标库存
        policies[store_id] = (s, S)
    return policies

## 7 关键洞察

- **三级供应链的核心挑战是多级协调**：每个层级都有自己的库存策略和目标，但整体最优要求各级决策对齐。牛鞭效应和双重边际效应在三级结构中同时存在。
- **设施选址是战略决策，库存策略是战术决策**：选址一旦确定难以更改，而库存参数可以灵活调整。所以选址阶段必须充分考虑后续库存成本的影响。
- **服务水平的边际成本是递增的**：从 95% 到 99%（提高 4%），安全库存需要翻倍以上，总成本增加 20-40%。服务水平目标需要从财务角度评估其合理性。
- **风险分担在三级供应链中的体现**：DC 合并了多个零售店的需求波动，安全库存按 √(Σσ²) 计算而非 Σσ，因此集中 DC 的总安全库存低于分散 DC。
- **这个项目是真实世界的缩影**：实际工作中，供应链规划师正是通过"选址→策略→仿真→调参"这个循环不断优化供应链的。
