## 这份教程是什么

这是一份给**有基础数学和 Python 能力、但对库存与供应链管理完全陌生**的技术人/供应链分析师/运营管理者写的自学教程。它不讲学院派的定理证明——每一章都围绕「这个库存模型解决什么业务问题」和「这个供应链协调机制为什么在这里好使」展开。

读完以后，你不一定会成为供应链总监，但你一定会：

## 术语速查表

✅ **核心术语**（先记住这 15 个，其余的以后查）

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 经济订货批量 | EOQ | 在确定需求下，每次订多少让持有成本+订货成本总和最小 |
| 安全库存 | Safety Stock | 需求不确定时多备的那部分，用来防止缺货 |
| 牛鞭效应 | Bullwhip Effect | 越往供应链上游，需求波动越大——零售商波动10%，工厂可能波动50% |
| 报童问题 | Newsvendor Problem | 单周期不确定需求下，订多了浪费、订少了亏——最优解等于需求分布的某个分位数 |
| 服务水平 | Service Level | 不缺货的概率——95%服务水平=100次里95次不缺货 |
| 订货点 | Reorder Point (ROP) | 库存降到这个数就下单补货 |
| (s,S) 策略 | (s,S) Policy | 库存降到s时订货到S——s是订货点，S是目标库存 |
| 批量折扣 | Quantity Discount | 买得越多单价越便宜——但持有成本也会上升 |
| 联合补货 | Joint Replenishment | 多个产品一次下单分摊固定成本——但可能多屯了不需要的货 |
| 持有成本 | Holding Cost | 库存占用的资金、仓储费、损耗——一般按货值的20-30%/年算 |
| 缺货成本 | Stockout Cost | 缺货导致的损失——丢单、罚款、信誉受损 |
| 提前期 | Lead Time | 从下单到到货的时间——提前期越长，安全库存越多 |
| 供应商管理库存 | VMI | 供应商替你看库存——你告诉销售数据，供应商决定补多少 |
| CPFR | Collaborative Planning, Forecasting & Replenishment | 供需双方共享预测和补货计划——比VMI更进一步 |
| 回购契约 | Buyback Contract | 供应商承诺买回你卖不掉的货——你更有动力多订 |

🔧 **进阶术语**（用到时回来查）

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 收益共享契约 | Revenue Sharing Contract | 供应商低价卖给你，但拿走你一部分销售收入 |
| 弹性订货契约 | Quantity Flexibility Contract | 你可以根据实际需求调整订货量——不超过某个百分比 |
| 数量折扣契约 | Volume Discount Contract | 总订货量越大，折扣越多 |
| 可选契约 | Option Contract | 你先买一个「订货权」，到时候可以按约定价格执行 |
| 多级库存 | Multi-Echelon Inventory | 不止一级（如工厂→区域仓→零售店），各级库存互相影响 |
| 风险分担 | Risk Pooling | 把多个地点的需求合在一起管理——方差降低，安全库存减少 |
| 延迟策略 | Postponement | 产品最后一道工序延迟到收到订单再做——通用件阶段不区分 |

# 第一部分：库存管理概览

> 在这一章，你不会看到一行代码。你会看到：库存管理是怎么从一个仓库记账工具变成今天供应链核心决策引擎的？一个补货问题怎么被拆解成「成本结构 / 需求模式 / 服务水平」这三个要素？以及——四大经典库存模型分别解决什么问题？

---

## 1.1 没有库存管理之前，世界是什么样子的

先想象三个场景。

**场景 A：缺货**。双十一当天，你的爆款商品在开售 2 小时后卖光了。系统显示「库存为 0」，但实际需求是库存的 5 倍。你失去了 500 万的销售额，还让用户跑去竞争对手那里。客服被骂了一天。供应链总监问：「为什么不多备一些？」采购经理说：「多备了万一卖不完呢？」

**场景 B：积压**。去年你多备了 3 个月的安全库存——结果竞品出了个更好的产品，你的库存全部变成滞销品。仓储费每月 20 万，财务说「这批货的持有成本已经超过了货值本身」。你不得不打折清仓——亏了 800 万。

**场景 C：牛鞭效应**。零售端需求只波动了 5%。零售商为了安全多订了 10%。批发商看到订单涨了 10%，又加了一点保险——向分销商订了 20% 更多。分销商看到趋势，又订了 30% 更多。最后工厂看到的订单波动是 50%。工厂不得不开两班倒 + 紧急采购——然后下个月订单突然回落，工厂产能过剩。

这三个场景的共同主题：**库存是必要的恶——太少会缺货、太多会积压、错配会放大。**

### 从 EOQ 到智能供应链：一段简史

| 年份 | 人物/事件 | 贡献 |
|------|----------|------|
| 1913 | Ford Harris | 发明 EOQ 公式——库存管理的起点 |
| 1934 | Wilson | 独立推导 EOQ（EOQ 也被称为 Wilson 公式） |
| 1951 | Arrow, Harris, Marschak | 报童问题的数学奠基——不确定需求下的最优订货 |
| 1958 | Magee | (s,S) 策略的直觉——定期盘点+订货点 |
| 1960s | Clark, Scarf | 多级库存系统的最优策略——串行供应链的分析 |
| 1985 | Lee, Padmanabhan, Whang | 牛鞭效应的正式理论分析 |
| 1990s~ | 实践爆发 | VMI、CPFR、供应链契约（回购/收益共享）大规模应用 |
| 2010s~ | AI+供应链 | 需求预测 ML + 库存优化 + 端到端可视化 |

## 1.2 库存管理的四个核心矛盾

| 矛盾 | 描述 | 你面对的取舍 |
|------|------|------------|
| 持有成本 vs 缺货成本 | 存货越多持有成本越高，越少缺货风险越大 | 最优解在两者平衡点 |
| 规模经济 vs 灵活度 | 一次订越多单件成本越低，但灵活性降低 | EOQ vs JIT |
| 服务水平 vs 库存成本 | 99% 服务水平需要的库存是 95% 的 2 倍以上 | 边际递减——最后 1% 最贵 |
| 局部最优 vs 全局最优 | 每个节点优化自己 vs 供应链整体最优 | 牛鞭效应的根源 |

**先记住一句话**：库存管理的本质不是「减少库存」，而是「在正确的服务水平下最小化总成本」。

---

## 1.3 四大经典库存模型

### 1. EOQ — 确定需求下的最优批量

最简单的起点。假设需求已知且恒定，每次订货有固定成本 K，每单位每周期有持有成本 h。

```
总成本 = 年订货次数 × K + 平均库存 × h
       = (D/Q) × K + (Q/2) × h

最优订货量 Q* = sqrt(2·D·K / h)
```

**核心直觉**：订货多→订货次数少（省固定成本）→但平均库存高（持有成本高）。最优解在两者相等处。

### 2. 报童问题 — 单周期不确定需求

一次性的订货决策（季节性商品、生鲜、报纸）。需求不确定，已知分布。订多了浪费，订少了亏。

```
Cu = 缺货成本（少卖一件的利润损失）
Co = 超量成本（多订一件的损失）

最优服务水平 = Cu / (Cu + Co)   ← 关键公式
最优订货量 = F^{-1}(Cu / (Cu + Co))   ← 需求分布的该分位数
```

**核心直觉**：缺货成本越高，你该订越多；超量成本越高，你该订越少。比率 Cu/(Cu+Co) 决定最优分位数。

### 3. (s,S) 策略 — 多周期连续盘点

每周期检查库存。当库存降到 s（订货点）时，订货至 S（目标库存水平）。

```
s = 订货点 = 提前期内的期望需求 + 安全库存
S = 目标库存水平 = s + Q*（或通过 DP 求解）
安全库存 = z × σ × sqrt(L)   ← z 由服务水平决定，L = 提前期
```

**核心直觉**：s 负责防止缺货，S-s = Q 负责平衡订货与持有成本。

### 4. Base Stock 策略 — 单周期订货

每周期检查库存，订到基准库存水平 S。如果当前库存 < S，订 S - 当前库存这么多。

**核心直觉**：适合短提前期、高服务要求的场景。供应链中常作为协同基准。

| 模型 | 需求 | 周期 | 关键参数 | 适用场景 |
|------|------|------|---------|---------|
| EOQ | 确定 | 多周期 | Q* (批量) | 稳定需求的常规补货 |
| 报童 | 不确定 | 单周期 | 服务水平分位数 | 季末商品、生鲜、报纸 |
| (s,S) | 不确定 | 多周期 | s (订货点), S (目标库存) | 常规物品的再补货 |
| Base Stock | 不确定 | 多周期 | S (基准库存) | 短提前期、高周转物品 |

### 进阶模型（先了解，用到再深学）

**🔹 多级库存 (Multi-Echelon)** — 你的库存不只是你的库存

```
工厂 → 区域仓库 → 零售店
```

各级库存是耦合的。区域仓库的库存策略直接影响零售店的缺货率。Clark-Scarf 模型是理论基础。

**🔹 联合补货 (Joint Replenishment)** — 多个产品一起订

如果你有 1000 个 SKU，每次无论订哪种产品都有固定成本 K0，每种产品还有各自的固定成本 Ki。最优策略比单产品 EOQ 复杂得多——需要平衡补货频率和批次。

## 1.4 库存建模五步法（针对供应链场景）

### 第一步：画供应链结构（2 分钟）

实体（方框）+ 流向（箭头）+ 关键参数。谁向谁订货？运输时间多长？

### 第二步：确认需求模式（1 分钟）

- 需求是确定的还是随机的？
- 如果是随机的，分布已知吗？均值？方差？
- 单周期还是多周期？

### 第三步：列成本项（1 分钟）

| 成本类型 | 符号 | 说明 |
|---------|------|------|
| 固定订货成本 | K | 每次下单的行政/运输固定费用 |
| 持有成本 | h | 每单位每月的仓储/资金占用 |
| 缺货成本 | Cu | 每缺一件的损失 |
| 超量成本 | Co | 每剩一件的损失 |

### 第四步：选模型（30 秒）

```
确定需求 + 多周期 → EOQ
不确定 + 单周期 → 报童
不确定 + 多周期 + 连续盘点 → (s,S)
不确定 + 多周期 + 定期盘点 → Base Stock
不确定 + 多级 + 多产品 → 进阶模型
```

### 第五步：验证直觉（30 秒）

- 固定成本 K 增大 → 批量增大
- 持有成本 h 增大 → 批量减小
- 服务水平要求提高 → 安全库存增加
- 提前期变长 → 安全库存增加

# 第二部分：供应链管理核心

> 从单个仓库的补货决策，升级到整条供应链的协调。什么是牛鞭效应？为什么信息共享能缓解它？契约怎么设计才能让大家劲往一处使？

---

## 2.1 供应链结构：推式 vs 拉式

```
推式（Push）：
  生产 → 库存 → 销售
  「我猜你会买，所以我先生产」
  
拉式（Pull）：
  订单 → 生产 → 交付
  「你买了，我才生产」
```

| 维度 | 推式 | 拉式 |
|------|------|------|
| 需求不确定性 | 低（可以预测） | 高（难以预测） |
| 提前期 | 长 | 短 |
| 规模经济 | 好（批量生产） | 差（按单制造） |
| 库存风险 | 高（可能积压） | 低（零库存） |

**现实**：大部分供应链是「推拉结合」——前端按库存生产（推），后端按订单组装（拉）。例如 Dell 的直销模式：零件按预测备货（推），最终组装按订单（拉）。

---

## 2.2 牛鞭效应的数学直觉

### 为什么需求波动会被放大？

假设 4 级供应链：零售商 → 批发商 → 分销商 → 工厂。

每级在做各自的库存决策。每一级都试图从历史订单中「推断」未来需求——但每个推断都加入了安全库存。

**数学直觉**：
- 零售端需求是 \( D_t = \mu + \epsilon_t \)（均值 + 随机波动）
- 零售商使用移动平均预测，并加安全库存 → 订单波动 > 需求波动
- 批发商看到零售商的订单（已放大），再次预测、再次加安全库存 → 进一步放大
- 到工厂级别的订单波动可能是原始需求的 5-10 倍

### 牛鞭效应的量化指标

```
放大系数 ≈ 工厂端订单标准差 / 零售端需求标准差
```

实验中可以观察到，即使需求是简单的独立同分布，放大系数也 > 1。

### 缓解牛鞭效应的四个策略

1. **信息共享**：零售商实时共享销售数据（POS 数据），避免上下游仅靠订单推断
2. **减少提前期**：提前期越短，安全库存需求越小，放大效应越低
3. **批量缩减**：EOQ 越大的企业越容易放大波动——考虑小批量多批次
4. **价格稳定**：促销和批量折扣会人为制造需求波动

## 2.3 信息共享与协同

### VMI（供应商管理库存）

供应商替零售商管理库存——供应商能直接看到零售商的库存水平和销售数据。

```
传统模式：零售商→下订单→供应商发货
VMI 模式：零售商→共享数据→供应商决定补货量和时间
```

**优势**：
- 供应商有全局视野，减少牛鞭效应
- 零售商省去采购管理成本
- 供应商可以优化自己的生产计划

**风险**：
- 供应商可能过度补货（多卖货）
- 零售商需要信任供应商的数据使用权

### CPFR（协同规划、预测与补货）

比 VMI 更进一步——供需双方共享预测和补货计划，联合制定。

```
步骤 1：联合需求预测
步骤 2：联合产能/库存规划
步骤 3：联合补货执行
步骤 4：联合偏差评估
```

**适用场景**：零售-供应商长期合作关系，双方都有 IT 能力。

## 2.4 供应链契约设计

供应链契约的目标：让每个节点在追求自身利益最大化的同时，实现供应链整体最优。

### 双重边际效应（Double Marginalization）

```
制造商成本 = c → 出厂价 w → 零售商售价 p → 市场需求 D(p)
```

如果制造商和零售商各自最大化自己的利润，总利润 < 整合供应链的最优利润。这就是双重边际效应——两层都在加价，结果总利润更低。

### 四种核心契约

| 契约类型 | 核心机制 | 协调效果 |
|---------|---------|---------|
| **批发价契约** | 零售商按 w 进货，承担所有风险 | 不协调——双重边际效应存在 |
| **回购契约** | 供应商以 b < w 回购未售商品 | ✅ 协调——零售商风险降低，订更多 |
| **收益共享契约** | 供应商以低价 w' 供货，分享φ比例的收入 | ✅ 协调——等价于回购契约 |
| **数量弹性契约** | 零售商可调增/减订货量一定比例 | ✅ 部分协调——需求不确定性分担 |

**关键结论**：好的契约设计可以让零售商的订货决策等价于整合供应链下的最优决策。

## 2.5 供应链网络设计（选址-库存-运输联合优化）

### 三要素权衡

```
选址决策（建几个仓、在哪建）
   → 影响运输成本（离客户近 = 运费低）
   → 影响库存成本（仓越多 = 总安全库存越多）
   → 影响服务水平（仓越多 = 响应越快）

运输决策（配送路线、频率）
   → 影响库存（快运减少安全库存）
   → 影响成本（快运更贵）

库存决策（各级库存策略）
   → 影响选址（集中库存 vs 分散库存）
   → 影响运输（批量 vs 频率）
```

### 典型建模框架

```python
# 概念框架（非可运行代码）
决策变量:
  y_j ∈ {0,1}      # 是否在位置 j 建仓库
  x_ij ≥ 0         # 从仓库 j 运到客户 i 的量
  S_j ≥ 0          # 仓库 j 的安全库存水平

目标:
  min  运输成本(x) + 固定成本(y) + 库存成本(S)

约束:
  - 每个客户被分配到一个仓库
  - 仓库产能限制
  - 库存平衡
  - 服务水平约束（安全库存 ≥ z·σ·sqrt(L)）
```

这种联合优化在 OR 中是一个 NP-hard 的 MIP，实际中常采用启发式（先定选址，再定库存，再调运输）或 Benders 分解。

## 案例 1：EOQ 与批量决策 ★★☆☆☆

**模型类型**：确定型库存模型 / 闭式解
**求解方法**：一阶导数求极值 + 灵敏度分析
**代码**：`code/case01_eoq.py`

**场景**：你管理一家连锁超市的纸巾采购。纸巾年需求量 D = 10,000 箱，每次订货固定成本 K = 100 元（运输+行政），每箱年持有成本 h = 5 元/年。问：一次订多少箱最划算？

**教学点**：
- EOQ 公式的推导——总成本 = 订货成本 + 持有成本
- 最优解的本质：平衡两种成本的边际变化
- 灵敏度分析——如果 K 或 h 变了，批量应该怎么调
- EOQ 的局限性——实际中需求不是恒定的

**✅ 验证标准**：
1. EOQ 公式计算结果 = sqrt(2*10000*100/5) = 632.46 ≈ 632 箱
2. 在该批量下，年订货成本 = 年持有成本（验证平衡条件）
3. K 增大 20% 时批量增加 ≈ 9.5%（sqrt 关系）
4. h 增大 20% 时批量减少 ≈ 9.5%（对称性）

**关键洞察**：
- EOQ 对参数误差不敏感：K 或 h 偏离 20%，总成本只增加 < 2%
- 对于定性决策（一周订一次 vs 一月订一次），EOQ 已经够用
- EOQ 不是「正确数量」，而是「最佳近似」

**延伸**：
- 加入批量折扣（全量折扣 vs 增量折扣）
- 有限生产率模型（EPQ）——一边生产一边消耗
- 考虑缺货允许（backorder）的 EOQ 变体

## 案例 2：报童与服务水平 ★★★☆☆

**模型类型**：随机库存模型 / 单周期 / 概率决策
**求解方法**：分位数求解 + 蒙特卡洛仿真验证
**代码**：`code/case02_newsvendor.py`

**场景**：你是某生鲜电商的采购经理。周日要进一批草莓，周一卖。每盒成本 20 元，售价 50 元，卖不掉的只能以 5 元处理。需求服从正态分布，均值 μ = 500 盒，标准差 σ = 100 盒。问：应该进多少盒？

**教学点**：
- 报童问题的最优条件：临界分位数
- Cu 和 Co 的计算——缺一件亏多少、多一件亏多少
- 服务水平的概念——最优服务水平 = Cu/(Cu+Co)
- 从单周期扩展到多周期（安全库存）

**✅ 验证标准**：
1. Cu = 50-20 = 30，Co = 20-5 = 15
2. 最优服务水平 = 30/(30+15) = 0.667
3. 最优订货量 = 500 + 0.43×100 = 543 盒（z=0.43 对应 66.7% 分位数）
4. 蒙特卡洛仿真验证：该订货量的期望利润 > 其他试错方案的利润

**关键洞察**：
- 当缺货成本 >> 超量成本时（如救命药），最优服务水平接近 100%
- 当超量成本 >> 缺货成本时（如生鲜），最优服务水平降低
- 服务水平每提高 1%，库存非线性增加——要 99% 还是 95%？

**延伸**：
- 需求分布不是正态分布怎么办（非参数方法）
- 多产品一次性订货（预算约束下分配订货量）
- 风险规避的报童（期望效用替代期望利润）

## 案例 3：牛鞭效应仿真 ★★★☆☆

**模型类型**：系统仿真 / 多级供应链
**求解方法**：离散事件仿真 + 统计比较
**代码**：`code/case03_bullwhip.py`

**场景**：4 级供应链——零售商 → 批发商 → 分销商 → 工厂。每级使用简单的 (s,S) 策略。零售端需求是 i.i.d. 正态分布。观察各级订单的标准差变化。

**教学点**：
- 牛鞭效应的产生机制——每级只看到订单，看不到真实需求
- 信息共享如何缓解牛鞭效应
- 订货策略参数对放大的影响
- 不同批量策略的波动性比较

**✅ 验证标准**：
1. 工厂端订单标准差 > 零售端需求标准差（放大系数 > 1）
2. 信息共享（POS 数据共享）时放大系数下降
3. 减少批量/缩短提前期时放大系数下降
4. 图形显示：越往上游，订单曲线越陡峭

**关键洞察**：
- 即使每级都使用最优的 (s,S) 策略，牛鞭效应依然存在
- 信息共享是最有效的缓解手段之一
- 促销和折扣会人为制造牛鞭效应——平稳需求是最好的

**延伸**：
- 啤酒游戏——经典课堂仿真，强烈推荐实际玩一次
- 加入价格波动（促销）后的牛鞭效应
- 引入 VMI 机制的仿真对比

## 案例 4：供应链契约设计 ★★★★☆

**模型类型**：博弈论 + 库存理论 / 契约协调
**求解方法**：数学推导 + 数值仿真
**代码**：`code/case04_contract.py`

**场景**：一个制造商 M 和一个零售商 R。M 的生产成本 c = 10，以批发价 w = 20 卖给 R，R 以零售价 p = 40 卖给消费者。市场需求 D(p) = 100 - p（确定需求）。问：批发价契约下各自利润是多少？如何用回购契约消除双重边际效应？

**教学点**：
- 双重边际效应的数学推导——分散决策 vs 集中决策的利润差
- 回购契约的协调条件——回购价 b 应使零售商的最优订货量等于供应链最优订货量
- 收益共享契约——等价于回购契约的另一种形式
- 利润分配 vs 效率——协调契约不改变总利润吗？

**✅ 验证标准**：
1. 集中决策下最优订货量 > 分散决策（批发价）的订货量
2. 集中决策总利润 > 分散决策总利润
3. 存在回购价 b 使得零售商选择集中决策的最优订货量
4. 回购价格越高，零售商订货越多，但制造商利润越低（矛盾）

**关键洞察**：
- 好的契约让「零售商的最优 = 供应链的最优」——不再有冲突
- 契约协调了利益，但如何分配利润仍然取决于批发价和回购价的组合
- 实际中，契约条款的谈判比契约本身更难

**延伸**：
- 需求不确定下回购契约的协调条件
- 收益共享契约 vs 回购契约的等价性证明
- 三阶段供应链（制造商→批发商→零售商）的契约设计

## 案例 5：供应链网络设计 ★★★★☆ [进阶]

**模型类型**：混合整数规划 / 设施选址 + 库存 + 运输
**求解方法**：MIP（分支定界）+ 灵敏度分析
**代码**：`code/case05_network.py`

**场景**：你需要在 10 个候选城市中选择 3-5 个建仓库。有 50 个客户城市，需求已知。每个仓库有建设固定成本、运营变动成本和单位运输成本。需求不确定（各城市需求独立正态分布）。目标：最小化总成本（建设+运输+库存）。

**教学点**：
- 设施选址问题的 MIP 建模（经典单分配模型）
- 库存-运输权衡——仓越多运输成本越低，但库存成本越高
- 服务水平约束转化为安全库存约束
- 风险分担——合并仓库可以降低总安全库存

**✅ 验证标准**：
1. 求解结果满足每个客户只分配到一个仓库（单分配约束）
2. 总建设成本 + 运输成本 + 库存成本 = 模型目标值
3. 仓库数量增加时运输成本下降、库存成本上升——趋势正确
4. 需求波动 σ 增大时，安全库存成本增加，集中化趋势更明显

**关键洞察**：
- 库存成本 ≈ 安全库存成本 + 周转库存成本——安全库存与 sqrt(仓库数) 成正比
- 不确定性越大，集中化越有利（风险分担效果更显著）
- 网络设计是战略层决策——一旦建了仓库，后面几年都要用

**延伸**：
- 多产品多梯级网络（工厂→配送中心→零售店）
- 考虑碳排放约束的绿色供应链网络
- 动态网络设计——需求随时间变化时如何调整选址

## 毕业项目（08-capstone.md）

### 项目主题：多级供应链优化

**问题**：你管理一个三级供应链——工厂 → 配送中心(DC) → 零售店。

**已知参数**：
- 1 个工厂（产能有限）
- 3 个候选 DC 位置（建一个固定成本 100 万/年，每个 DC 有容量限制）
- 10 个零售店，每个零售店日需求 ~ N(100, 20)，提前期工厂→DC=3天，DC→零售店=1天
- 持有成本：DC 为货值 15%/年，零售店为货值 25%/年
- 服务水平要求：零售店 95%，DC 98%
- 运输成本：工厂→DC 和 DC→零售店均为 0.5 元/单位/公里

**要求**：
1. **Step 1**：决定建几个 DC、在哪建（设施选址 MIP，不考虑库存成本）
2. **Step 2**：为选定的 DC 和零售店设计库存策略（(s,S) 策略参数 + 安全库存）
3. **Step 3**：考虑需求不确定性下的联合优化——用仿真评估整体表现
4. **Step 4**：做灵敏度分析——服务水平要求从 95% 提到 99%，总成本增加多少？

**✅ 验证标准**：
1. Step 1 选址结果合理（运输成本最低的选址方案）
2. Step 2 库存策略满足服务水平约束（仿真验证缺货率 < 1% 或 5%）
3. Step 3 总成本 = 建设成本 + 运输成本 + 库存成本（持有+订货）
4. Step 4 中 95%→99% 的服务水平提升导致总成本增加 20-40%（合理范围）
5. 集中化（少建 DC）时运输成本上升但库存成本下降——权衡趋势正确

**扩展方向**：
- 加入季节性需求
- 加入多个产品品类（共享补货资源）
- 加入应急运输（空运 vs 海运的权衡）

## 附录A：什么时候不需要库存模型

| 场景 | 理由 |
|------|------|
| 你的库存决策只有 2-3 个选项 | 穷举比建模快 |
| 需求完全不可预测（连分布都不知道） | 先做需求预测，库存模型建立在预测之上 |
| 供应链极度不稳定（每周换供应商） | 优化一个短期会变的东西没意义 |
| 库存占运营成本不到 1% | 不值得花精力优化 |
| 业务方只接受「凭经验」 | 库存模型的解释成本可能高于收益 |

## 附录B：TOP 5 必踩坑

1. **假设需求恒定** — EOQ 很好用，但前提是需求稳定。季节性、促销期、新品上市不能用简单 EOQ
2. **忽略提前期** — 「今天订明天到」和「今天订三个月后到」的安全库存天差地别
3. **服务水平设太高** — 95% 到 99% 听起来只差 4%，但库存可能要多 50%
4. **独立优化各级库存** — 零售商优化自己 → 批发商优化自己 → 加起来就是牛鞭效应
5. **忘记数据质量** — 「输入垃圾，输出垃圾」——库存模型的精度不会超过需求预测的精度

## 附录C：库存与供应链 × ML 交叉点

| 结合方式 | 说明 | 典型应用 |
|---------|------|---------|
| ML 需求预测 → 库存优化 | 用 LSTM/Prophet 预测需求，输入 (s,S) 模型 | 所有消费品零售 |
| 强化学习补货 | 将补货策略训练为 RL policy | 大规模高维库存（千级 SKU） |
| 聚类 + 差异化策略 | 用 ML 对 SKU 聚类，不同类用不同库存模型 | 多 SKU 库存管理 |
| 异常检测 | 检测供应链中的异常事件（缺货前兆、品质问题） | 实时供应链监控 |
| 端到端决策优化 | 预测层+决策层联合训练 | 定价+库存联合优化 |
