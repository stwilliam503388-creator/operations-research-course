# 附录C：多目标优化 × ML 的交叉点

> 多目标优化和机器学习的结合越来越紧密。这不仅是两个工具的拼凑——在多个方向上都产生了 1+1 > 2 的效果。以下是五个主要的交叉方向，以及一个正在兴起的方向。

---

## 1. ML 预测 → MOO 输入

**模式**：用 ML 模型预测不确定的目标参数，作为多目标模型的输入。

**典型应用**：需求预测 → 供应链多目标优化

```
历史数据 → LSTM/Prophet → 需求预测 → 多目标库存优化
                                     ↓
                              成本↓ 服务↑ 碳排放↓
```

**关键挑战**：ML 预测的不确定性需要传递到 MOO 中。如果只是点估计（单值预测）→ 结果忽略了波动性。推荐做法：
- **场景法**：生成多个预测场景（P10 / P50 / P90），每个场景跑一次 MOO
- **机会约束**：把「需求满足率 ≥ 95%」作为约束，而非固定需求值
- **鲁棒优化**：最小化最坏情况下的成本

---

## 2. MOO 生成训练数据 → ML 学习帕累托前沿

**模式**：帕累托前沿的解作为 ML 的训练标签，训练神经网络学习从问题参数到帕累托前沿的映射。

**典型应用**：在线决策场景下的实时帕累托前沿推理

```
训练阶段：NSGA-II → 生成帕累托前沿 → 训练 NN
推理阶段：新问题参数 → NN → 直接输出帕累托前沿（0.1ms vs 10s）
```

**为什么需要这个**：传统的 NSGA-II 求解一次可能需要几秒到几分钟。在实时场景（如自动驾驶的路径规划、电商的实时定价）中，这个延迟不可接受。用 NN 近似帕累托前沿可以实现毫秒级推理。

**注意事项**：
- 训练数据的覆盖性很重要——参数空间的边界区域容易被忽略，但可能带来最大的收益
- NN 输出的是近似前沿，需要用少量真实评估验证精度
- 前沿不连续时（如 ZDT3），NN 可能产生平滑化效应——需要特殊损失函数

---

## 3. ML 作为代理模型加速 MOO

**模式**：用高斯过程（GP）、神经网络或随机森林近似目标函数，替代昂贵的真实评估。

**典型应用**：昂贵的仿真优化（CFD 流体仿真、FEA 有限元分析）+ NSGA-II

```text
真实目标函数（2 小时/次）
       ↓ 高斯过程拟合
代理模型（0.01 秒/次）
       ↓
用代理模型跑 90% 的代 + 少量真实评估验证
```

**关键点**：
- **主动学习**：选择「不确定性最大」或「最有希望改进」的点做真实评估——最大化每次评估的信息增益
- **模型更新**：每 N 代用真实评估结果重新拟合代理模型
- **常用的代理模型**：高斯过程（自带不确定性估计）、Random Forest（适合高维离散输入）、贝叶斯神经网络

---

## 4. 多目标超参数调优

**模式**：模型的多个性能指标同时优化——精度 vs 速度 vs 模型大小。

**典型应用**：AutoML、模型压缩、NAS（神经网络架构搜索）

| 目标 | 说明 |
|------|------|
| 精度 | validation accuracy（最大化）|
| 推理速度 | latency（最小化）|
| 模型大小 | parameter count（最小化）|

**推荐方法**：Optuna 内置了多目标 TPE + NSGA-II 两种后端。pymoo 也可以与 scikit-learn pipeline 结合。

```python
import optuna

def objective(trial):
    n_layers = trial.suggest_int('n_layers', 2, 10)
    lr = trial.suggest_float('lr', 1e-5, 1e-2, log=True)
    # 训练模型...
    return accuracy, inference_time, model_size

study = optuna.create_study(
    directions=['maximize', 'minimize', 'minimize'],  # 三目标
    sampler=optuna.samplers.NSGAIISampler()
)
study.optimize(objective, n_trials=100)
```

**经验**：三目标超参数优化的帕累托前沿通常呈「精度-速度-大小」的 Pareto surface——树模型（LightGBM/XGBoost）往往比深度学习模型在速度和大小上有绝对优势。

---

## 5. 公平性多目标

**模式**：模型精度 vs 群体公平性 vs 个体公平性。

**典型应用**：信用评分、招聘筛选、医疗诊断

```text
目标1：模型预测准确率（最大化）
目标2：群体公平性差异（最小化，如 Demographic Parity）
目标3：个体公平性（最小化，类似条件下应得相同预测）
```

**帕累托前沿的解读**：前沿上的每个点对应一种「精度-公平性」的折衷。前沿的斜率告诉你「为了提升 1% 的公平性，需要牺牲多少精度」。如果前沿很陡峭——说明公平性约束的边际成本很高。如果前沿很平缓——说明你可以做到公平而不损失多少精度。

**实际案例**：某银行的信用评分模型，在 Demographic Parity 差距从 0.3 降到 0.05 时，AUC 从 0.92 降到 0.88——这个 4% 的精度损失换来了显著更公平的模型，业务上可以接受。

---

## 6. 正在兴起的方向：ML for MOO 算法设计

**模式**：用强化学习或元学习自动设计 MOO 算法的交叉/变异算子或参数调节策略。

这个方向还在学术前沿阶段，但值得关注——未来的多目标求解器可能会用图神经网络选择算子、用 GFlowNet 生成初始种群。

## 交叉点速查表

| 结合方式 | 难度 | 成熟度 | 推荐工具 |
|---------|------|-------|---------|
| ML→MOO | ★★☆ | 成熟 | scikit-learn + pymoo |
| MOO→ML | ★★★ | 发展中 | PyTorch + pymoo |
| 代理模型 | ★★★★ | 成熟 | GPyOpt / SMAC |
| 超参数调优 | ★☆☆ | 非常成熟 | Optuna / Hyperopt |
| 公平性 | ★★★ | 发展中 | Fairlearn + pymoo |
| ML for MOO 设计 | ★★★★★ | 前沿 | 无成熟工具 |
