## 2.1 帕累托支配关系

### 直觉

> 解 A 支配解 B ⇔ A 在所有目标上不比 B 差，且至少在一个目标上比 B 好。

**生活类比**：你和朋友各自找到了一款电脑。

```
你的电脑：  5000元, 16GB内存, 512GB硬盘
朋友的电脑：6000元, 8GB内存,  512GB硬盘
```

你的电脑更便宜（更好）、内存更大（更好）、硬盘一样——你的电脑**支配**朋友的电脑。朋友不会考虑他那台了。

```
你的电脑：  5000元, 16GB内存, 512GB硬盘
朋友的电脑：4500元,  8GB内存, 1TB硬盘
```

你的更贵但内存大，朋友的更便宜但硬盘大——**互不支配**。这两个都是候选方案。

### 形式定义

对于最小化问题，解 A 支配解 B（记为 A ≺ B）当且仅当：

```
∀i ∈ {1,...,m}: fᵢ(A) ≤ fᵢ(B)   (A 在所有目标上不差于 B)
且 ∃j ∈ {1,...,m}: fⱼ(A) < fⱼ(B)   (至少一个目标严格更好)
```

三个类别：
- **强支配**：A 在所有目标上严格比 B 好
- **弱支配**：A 在所有目标上不差于 B（至少一个相等）
- **互不支配**：A 在某些目标上更好，B 在另一些上更好

### 算法实现（Python 伪代码）

```python
def dominates(a, b):
    """返回 True 当 a 支配 b（两个目标向量都是越小越好）"""
    return all(a <= b) and any(a < b)

# 使用示例
A = np.array([30, 2000])   # 续航30min, 成本2000
B = np.array([45, 3500])   # 续航45min, 成本3500
C = np.array([20, 1800])   # 续航20min, 成本1800
D = np.array([40, 4000])   # 续航40min, 成本4000

print(dominates(B, D))  # True: B 支配 D
print(dominates(A, C))  # False: 互不支配
print(dominates(C, A))  # False: 互不支配
```

## 2.2 加权求和法

### 直觉

> 给每个目标一个权重，把它们加起来变成一个目标——然后你就回到了熟悉的单目标优化世界。

```
min  F(x) = w₁·f₁(x) + w₂·f₂(x) + ... + wₘ·fₘ(x)
s.t. 原始约束
     Σ wᵢ = 1, wᵢ ≥ 0
```

**生活类比**：你要买手机，你内心觉得「性能对我来说占 60%，拍照占 30%，外观占 10%」——你给每个维度打了权重，然后算总分。

### 优点

- 极其简单——任何单目标求解器都可以用
- 计算效率高——跑一次得到一个解
- 直观易解释——「你的偏好权重是多少」

### 陷阱：非凸前沿

加权求和法有一个致命的陷阱：**它找不到非凸区域的帕累托最优解**。

```
        │
        │    ╱╲
        │   ╱  ╲        ← 帕累托前沿
        │  ╱    ╲
        │ ╱      ╲
        │╱        ╲
        └─────────────────
            f₁
```

加权求和等价于用一条直线（权重法线）去切可行域，找切点。如果前沿是非凸的——直线切不到凹进去的部分，那些解永远得不到。

**你的权重不可能覆盖所有非支配解，无论你怎么调权重。**

### 什么时候用

| 适用场景 | 不适用场景 |
|---------|-----------|
| 你清楚权重 | 你不知道权重 |
| 前沿近似凸 | 前沿可能是非凸的 |
| 你只需要一个解（不是整个前沿） | 你需要整个帕累托前沿 |
| 快速原型验证 | 正式决策支持 |

## 2.3 ε-约束法

### 直觉

> 选一个目标作为目标函数，把所有其他目标转为约束——给它们设上限 εᵢ。

```
min  f₁(x)
s.t. f₂(x) ≤ ε₂
     f₃(x) ≤ ε₃
     ...
     fₘ(x) ≤ εₘ
     原始约束
```

**生活类比**：你买房子。「最看重价格」——先把价格最小化；但增加了约束「面积 ≥ 100㎡」、「离地铁 ≤ 500m」、「房龄 ≤ 10年」。这些约束是你对其他目标的最低要求。

### 怎么得到整个前沿

通过改变 ε₂, ε₃, ..., εₘ 的取值，反复求解，得到不同的最优解——这些解就构成了帕累托前沿的近似。

```
for ε₂ in [min_f₂, max_f₂]:
    for ε₃ in [min_f₃, max_f₃]:
        求解 min f₁ s.t. f₂≤ε₂, f₃≤ε₃...
```

### 三步确定 ε 范围

1. 分别求出每个目标的单目标最优值（理想点）
2. 用各目标的最差值作为 ε 的上限
3. 在 [min, max] 之间均匀取 N 个点

```
# 示例：双目标
# 1. 分别求单目标最优
f1_min = min f1(x)          # 得到 f1* = 100
f2_min = min f2(x)          # 得到 f2* = 50

# 2. 求对方目标值
f1_at_f2min = f1(x*) where x* = argmin f2    # ≈ 300
f2_at_f1min = f2(x*) where x* = argmin f1    # ≈ 200

# 3. ε 范围: [f2_min, f2_at_f1min] = [50, 200]
epsilons = np.linspace(50, 200, 30)
```

### 优点 vs 缺点

| 优点 | 缺点 |
|------|------|
| 可以找到非凸前沿（克服加权求和的致命缺陷）| 计算量大——每个 ε 组合都要跑一次优化 |
| 每个解有明确含义——「在 f₂ 不超过 ε₂ 时 f₁ 的最小值」| ε 的取值范围需要知道各目标的 min/max |
| 约束式表达符合人类直觉 | 多个 ε 同时变时，组合爆炸 |

### 什么时候用

| 适用场景 | 不适用场景 |
|---------|-----------|
| 目标数量少（2~3 个）| 目标数量多（4+ 个）|
| 你能给其他目标设定合理的 ε 值 | ε 值完全不知道从哪设 |
| 需要展示不同约束水平下的表现 | 需要一次跑出整个前沿（太慢）|

## 2.4 目标规划

### 直觉

> 给每个目标设一个理想值（目标值），然后最小化所有偏离的加权和（或最大偏离）。

```
min  Σ wᵢ · |fᵢ(x) - Tᵢ|   （加权和型）
或  min  max { wᵢ · |fᵢ(x) - Tᵢ| }  （极小化最大型）
s.t. 原始约束
```

其中 Tᵢ 是目标 i 的理想值（target）。

**生活类比**：你去买车，心里有三个目标：「价格 15 万以内」、「百公里油耗 6L 以下」、「后备箱至少 500L」。没有一辆车全满足——你选一辆偏离最小的。

### 两种常见形式

| 形式 | 数学 | 适用场景 |
|------|------|---------|
| 加权和型 | min Σ wᵢ·dᵢ⁺ + wᵢ·dᵢ⁻ | 希望平均偏离小 |
| 极小化最大型（切比雪夫型）| min max(wᵢ·dᵢ) | 希望最差的那个目标也不至于太差 |

其中 dᵢ⁺ = max(0, fᵢ - Tᵢ)（超过目标的偏离），dᵢ⁻ = max(0, Tᵢ - fᵢ)（不足的偏离）。

### 什么时候用

| 适用场景 | 不适用场景 |
|---------|-----------|
| 决策者能给出明确的理想值 | 理想值完全没概念 |
| 目标有明确的「满意水平」| 目标是最大化（没有上限）|
| 需要惩罚不同方向的偏离 | 真正的帕累托前沿才是你需要的 |

## 2.5 多目标进化算法（NSGA-II）

### 为什么需要进化算法

加权求和法和ε-约束法每次求解只得到一个解。要得到整个帕累托前沿，需要反复求解。当问题非线性、非凸、非连续时，反复求解代价极高。进化算法可以**一次运行生成整个帕累托前沿**。

### NSGA-II 核心流程

```
初始化 → 非支配排序 → 拥挤距离 → 锦标赛选择 → 交叉变异 → 合并种群 → ... → 终止
```

#### Step 1：非支配排序（Non-dominated Sorting）

把种群中的解分成多个前沿层级（Front）：

```
前沿 1 (Front 1)：不被任何其他解支配的解
前沿 2 (Front 2)：只被前沿 1 中的解支配
前沿 3 (Front 3)：只被前沿 1 或 2 中的解支配
...
```

**直觉**：这就像剥洋葱——最外层是"最好的"（不被任何人支配），剥掉之后下一层出现。

```
              Front 1 (最外层)
           ╱──────┬──────╲
          │  Front 2       │
          │  ┌───┬───┐     │
          │  │ F3 │   │     │
          │  └───┴───┘     │
           ╲──────┴──────╱
```

#### Step 2：拥挤距离（Crowding Distance）

同一前沿内的解，越稀疏越好——我们要保留多样性，防止扎堆。

```
拥挤距离 = 解在每个目标方向上与相邻解的距离之和
```

**直觉**：在帕累托前沿上，一个解的两边邻居离它越远，它的拥挤距离越大——说明它处在"人少"的区域，值得保留。

```
     f₂ ↑
        │   ● ← 拥挤距离大（稀疏区域）
        │ ●   ●
        │  ● ← 拥挤距离小（密集区域）
        │   ●
        └──────────→ f₁
```

#### Step 3：锦标赛选择 + 精英保留

选择标准：
1. **前沿层级越低越好**（优先选 Front 1 的）
2. **同一层级内，拥挤距离越大越好**（优先选稀疏区域的）

**精英保留**：父代和子代合并后，按 1-2 标准选 N 个进入下一代——保证不丢失最优解。

### 伪代码

```
P = 随机初始种群(N)
对每一代 t = 1, 2, ..., T:
    Q = 从 P 通过选择+交叉+变异生成子代(N)
    R = P ∪ Q                    # 合并 2N
    fronts = 非支配排序(R)        # 分成多个前沿
    P_new = []
    for front in fronts:
        if len(P_new) + len(front) ≤ N:
            P_new ∪= front       # 全部加入
        else:
            计算拥挤距离(front)
            按拥挤距离降序选满剩余名额
    P = P_new
输出 P 中所有 Front 1 的解 = 帕累托前沿
```

### NSGA-II 的优点

| 优点 | 说明 |
|------|------|
| 一次运行得到整个前沿 | 不需要反复求解 |
| 不需要提前设权重 | 适合后验偏好 |
| 全局搜索能力 | 能跳出局部最优（交叉变异的随机性）|
| 拥挤距离保证多样性 | 前沿上的解均匀分布 |
| 精英保留 | 不丢失已找到的优解 |

### 局限性

| 局限 | 说明 |
|------|------|
| 不保证全局最优 | 进化算法是启发式 |
| 参数敏感 | 种群大小、交叉率、变异率需要调参 |
| 高维目标退化 | 3+ 个目标时，非支配比例爆炸，选择压力变弱 |
| 需要多次运行 | 随机性导致每次结果不同 |

## 2.6 性能指标

你跑出了一个帕累托前沿。它好还是不好？需要三个维度的评价：

### 1. 收敛性（Convergence）——离真正的帕累托前沿有多近

| 指标 | 定义 | 越小越好的方向 |
|------|------|---------------|
| GD (Generational Distance) | 找到的解到真实前沿的平均距离 | ↓ 越小越好 |
| IGD (Inverted GD) | 真实前沿上的点到找到的前沿的平均距离 | ↓ 越小越好 |
| ER (Error Ratio) | 找到的解中不在真实前沿上的比例 | ↓ 越小越好 |

### 2. 多样性（Diversity）——解在前沿上分布得多均匀、多广

| 指标 | 定义 | 方向 |
|------|------|------|
| Δ (Spread) | 相邻解之间的最大距离偏差 | ↓ 越小越好 |
| 前沿宽度 | 各目标方向上的范围 | ↑ 越宽越好（但要在前沿上）|
| 拥挤距离平均值 | 同一前沿内解的平均间距 | 适中为好 |

### 3. 综合指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 超体积 (Hypervolume, HV) | 帕累托前沿和参考点围成的超立方体体积 | ↑ 越大越好——同时衡量收敛性和多样性 |

**HV 为什么好**：它不要求你知道真实帕累托前沿（GD/IGD 需要参考真实前沿）。你只需要设定一个参考点（通常是各目标最差值的组合）。HV 越大→前沿越靠近理想点且覆盖越广。不过计算 HV 的复杂度随目标数指数增长。

### 计算示例（Python 伪代码）

```python
def hypervolume(front, reference_point):
    """简单 2D 超体积计算（矩形划分法）"""
    # 按 f1 排序
    sorted_front = sorted(front, key=lambda x: x[0])
    hv = 0.0
    prev_f1 = reference_point[0]
    for f1, f2 in sorted_front:
        if f1 < reference_point[0] and f2 < reference_point[1]:
            hv += (prev_f1 - f1) * (reference_point[1] - f2)
            prev_f1 = f1
    return hv
```

## 方法选择速查表

| 你的情况 | 推荐方法 |
|---------|---------|
| 知道权重，只需要一个解 | 加权求和法 |
| 知道权重，要整个前沿 | 加权求和法（多组权重）|
| 不知道权重，目标 ≤ 3 | ε-约束法 |
| 不知道权重，非线性/离散问题 | NSGA-II / 进化算法 |
| 有明确的理想值 | 目标规划 |
| 决策者有时间参与 | 交互式方法 |
