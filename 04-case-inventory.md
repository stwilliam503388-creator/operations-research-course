# 案例2：库存策略优化 — 报童问题 / MIP / 动态规划

> 难度：★★★☆☆ | 前置知识：概率基础 | ⏱ 阅读时间：40 分钟

---

## 1. 场景：隔壁老王的面包店

先讲个故事。

老王开了家面包店。每天早上 6 点，他得决定——**今天烤多少面包？**

- 烤多了：剩下的面包晚上就得扔，每个亏 2 元（持有成本 $h=2$）
- 烤少了：来了客人买不到，每个损失 8 元（缺货惩罚 $p=8$）
- 而且每次生火烤面包，不管烤多少，**固定费用 50 元**（$K=50$）

老王的店后面有个储藏室，最多放 30 个面包（容量 $C=30$）。每天的需求大概在 5~20 个之间，均匀分布。

老王现在靠经验拍脑袋——"今天周六，烤 15 个吧"。他想知道：**有没有一个科学的办法，告诉我每天该烤多少？**

这就是库存管理的核心问题。而它的答案，藏在**报童模型**和**动态规划**里。

---

## 2. 业务问题 → 数学模型

### 2.1 先看最简单的情况：只做一天生意

如果老王只做一天生意（或者他明天的面包必须今天烤，今天的面包明天卖不掉），这就是经典的**报童问题（Newsvendor Problem）**：

| 要素 | 含义 |
|------|------|
| **决策** | 订多少货（烤多少面包）$q$ |
| **不确定性** | 需求 $D$ 是随机变量（5~20 均匀分布） |
| **成本** | 订多了：持有成本 $h \cdot (q - D)$ |
| | 订少了：缺货惩罚 $p \cdot (D - q)$ |

**最优解**：找到 $q^*$ 使得累计概率刚好超过 $\frac{p}{p+h}$

$$
\Pr(D \leq q^*) \geq \frac{p}{p+h}
$$

这个 $\frac{p}{p+h}$ 叫**临界比率（Critical Ratio）**。老王的情况：
- $p=8, h=2 \rightarrow$ 临界比率 $= \frac{8}{10} = 0.80 = 80\%$
- 需求均匀分布 5~20（共 16 个值）
- $q^* =$ 第 $80\%$ 分位数 = **17 个面包**

**直觉**：缺货惩罚（8）比持有成本（2）大得多，所以老王应该多备一点货。宁可剩，不能缺。

### 2.2 多周期的情况：20 天连锁决策

现实世界比报童问题复杂——面包店不是只开一天。

老王要做的是**连锁决策**：
1. 今天结束时的库存 → 明天开始时的库存
2. 明天的订货决策 → 影响后天的库存
3. ...以此类推

这就变成了**多周期库存问题**：

#### 状态变量

$I_t$ = 第 $t$ 周期初的库存水平（$0 \leq I_t \leq C$）

#### 决策变量

$q_t$ = 第 $t$ 周期的订货量（$0 \leq q_t \leq C - I_t$）

#### 转移方程

$$
I_{t+1} = \max(0, \; I_t + q_t - D_t)
$$

> 如果库存不够卖，就缺货。多出来的库存留到下一周期。

#### 目标函数

$$
\min \; \mathbb{E} \left[ \sum_{t=1}^{T} \big( K \cdot \mathbb{1}_{[q_t > 0]} + h \cdot \max(0, I_t + q_t - D_t) + p \cdot \max(0, D_t - (I_t + q_t)) \big) \right]
$$

三项成本分别是：
1. **固定订货成本**：只要订货就收 $K$ 元（生火费）
2. **持有成本**：库存积压 $h$ 元/单位（面包过期）
3. **缺货惩罚**：库存不足 $p$ 元/单位（客人空手而归）

## 3. 数学模型（完整版）

### 集合与索引

| 符号 | 含义 |
|------|------|
| $t \in \{1, 2, ..., T\}$ | 周期索引 |
| $D_t$ | 第 $t$ 周期的随机需求（i.i.d.） |

### 参数

| 符号 | 含义 | 默认值 |
|------|------|--------|
| $T$ | 计划周期数 | 20 |
| $K$ | 固定订货成本 | 50 元 |
| $h$ | 单位持有成本/周期 | 2 元 |
| $p$ | 单位缺货惩罚 | 8 元 |
| $C$ | 仓库容量 | 30 单位 |
| $D \sim U[5, 20]$ | 需求分布 | 均匀分布 |

### 动态规划（DP）方程

**值函数**（value function）：

$$
V_t(I) = \min_{0 \leq q \leq C-I} \Bigg[
K \cdot \mathbb{1}_{[q > 0]} + \mathbb{E}_{D} \big[
  h \cdot \max(0, I + q - D) + p \cdot \max(0, D - (I + q)) + V_{t+1}\big(\max(0, \min(C, I+q-D))\big)
\big] \Bigg]
$$

**终端条件**：$V_{T+1}(I) = 0$（计划期结束后无成本）

### 逐行注释

| 部分 | 在说什么 | 业务含义 |
|------|---------|---------|
| $I_t$ | 周期初的库存 | 今天早上架子上的面包数 |
| $q_t$ | 今天的订货量 | 今天要烤多少面包 |
| $I_t + q_t - D_t$ | 卖完之后剩多少 | 正→有剩；负→缺货 |
| $K \cdot \mathbb{1}_{[q>0]}$ | 只要订货就收固定费 | 烤箱生火是要钱的 |
| $h \cdot \max(0, ...)$ | 剩一个亏 h 元 | 过期面包要扔掉 |
| $p \cdot \max(0, ...)$ | 缺一个亏 p 元 | 跑单的客人不满 |
| $V_{t+1}$ | 未来的期望成本 | 今天的决策影响明天 |

## 4. 求解方法选择

| 方法 | 能否用 | 原因 |
|------|--------|------|
| **动态规划（DP）** | ✅ 最优 | 状态数少（0~30），精确可解 |
| **报童公式** | ✅ 单周期 | 解析解，O(1) 计算 |
| **(s, S) 策略** | ✅ 可推导 | DP 的解天然是 (s, S) 形式 |
| 仿真 | ✅ 可验证 | 用 DP 求出的策略做仿真检验 |
| MIP 直接建模 | ❌ 冗余 | 随机需求下一般不用 MIP 直接建模 |
| 强化学习 | ❌ 杀鸡用牛刀 | 问题规模小，DP 就够了 |

**为什么选 DP？**

1. **状态少**：库存就 0~30 共 31 个状态，DP 的复杂度是 $O(T \times C^2)$，秒出
2. **全局最优**：穷举了所有可能性，保证找到最优策略
3. **可解释**：能从中提取出 (s, S) 策略的形式
4. **教学价值**：让你理解 Bellman 方程长什么样

> **粗俗理解**：DP 就是从最后一天往前倒着推——"如果今天是最后一天，库存 I 个，该订多少货？"然后往前一天，再往前一天...

## 5. 代码概览

完整的代码在 `code/case04_inventory.py`，核心结构：

```
main()
  ├── newsvendor_analytical()         → 报童问题的解析解
  ├── dp_inventory()                  → 多周期 DP 主函数
  │     ├── 倒推循环 (t=T-1 → 0)      → 从后往前求解每个状态
  │     │     └── 遍历所有可能订货量 q → 找到最优 q
  │     └── 前向仿真                  → 用 DP 策略跑样本路径
  ├── compute_s_s_policy()            → 从 DP 策略提取 (s, S)
  ├── check_newsvendor_vs_dp()        → 验证 1：报童 vs DP
  ├── check_capacity_violation()      → 验证 2：容量约束
  └── check_ordering_frequency()      → 验证 3：K vs 频率
```

### 核心算法：DP 倒推

```python
# 从最后一个周期往前倒推
for t in range(T - 1, -1, -1):
    for I in range(C + 1):          # 遍历所有库存水平
        best_cost = ∞
        for q in range(0, C - I + 1):  # 尝试所有可能的订货量
            cost = 固定成本(K) + 期望持有/缺货成本 + 未来期望成本
            if cost < best_cost:
                best_cost = cost, best_q = q
        V[t][I] = best_cost
        policy[t][I] = best_q
```

三个嵌套循环，简单粗暴。但如果 $C=1000$，一次计算需要 $20 \times 1001 \times 1001 \approx 2000$ 万次——依然可以秒出。

## 6. 运行结果解读

### 6.1 报童问题：单周期最优解

```
临界比率：p/(p+h) = 8/10 = 0.8000
最优订货量 Q* = 17
期望成本 = 12.75
```

**解释**：缺货惩罚（8元）远大于持有成本（2元），所以老王应该冒险多订。订 17 个面包时，有 80% 的概率需求被满足，20% 的概率会剩。

### 6.2 多周期 DP：仿真路径

```
周期 | 期初库存 | 订货量 | 实际需求 | 期末库存 | 固定成本 | 持有成本 | 缺货成本
-----|---------|-------|---------|---------|---------|---------|--------
  0  |    0    |  30   |   12    |   18    |   50    |  36.0   |   0.0
  1  |   18    |   0   |   16    |    2    |   0.0   |   4.0   |   0.0
  2  |    2    |  28   |    9    |   21    |   50    |  42.0   |   0.0
  3  |   21    |   0   |   19    |    2    |   0.0   |   4.0   |   0.0
  ...
```

注意看：
- 周期 0：老王开始创业，库存 0 → 一口气订满 30 个（到仓库容量上限）
- 周期 1：还有 18 个库存，够用 → **不订货**，省了 50 元固定成本
- 周期 2：库存降到 2 个 → **触发订货**，订 28 个补满
- 周期 3：库存 21 个 → 不开火

这就是 **(s, S) 策略**的直观表现！

### 6.3 (s, S) 策略

从 DP 策略中提取出的结果：

```
周期 | 再订货点 s | 订货目标 S
-----|-----------|-----------
  0  |     0     |    30
  1  |     0     |    30
  2  |     0     |    30
  ...
```

在这个案例里，$s=0, S=30$ 是一个非常清晰的 (s, S) 策略：
- **当库存降到 0 时**，订货到 30
- **当库存 > 0 时**，不订货

为什么 s=0？因为需求下限是 5，固定成本 50 相当高——老王宁愿冒着缺货风险也不愿频繁开火。只有当库存见底时，才值得花 50 元烤一次。

### 6.4 固定成本 vs 订货频率

```
K=   0: 订货次数=20/20, 平均库存= 5.14, 总成本=  280.00
K=  20: 订货次数=15/20, 平均库存= 5.29, 总成本=  546.00
K=  50: 订货次数= 8/20, 平均库存= 9.71, 总成本=  984.00
K= 100: 订货次数= 7/20, 平均库存= 8.67, 总成本= 1312.00
K= 200: 订货次数= 0/20, 平均库存= 0.00, 总成本= 2104.00
```

- **K=0**：每周期都订（第二天就把昨天剩的用完了再补），订货频率最高
- **K=50**：只订了 8 次，平均库存上升到 9.71——因为一次订得多
- **K=200**：干脆不订了——固定成本太高，订货一次亏一次

这个趋势完美验证了**固定成本越高，订货频率越低**的经济学直觉。

## 8. 📖 洞察

### 报童问题告诉我们什么

报童问题是**库存管理的入门心法**。它的核心洞察只有一句话：

> **备货量 = 缺货成本 vs 持有成本的权衡**

临界比率 $\frac{p}{p+h}$ 给了你一个量化的"安全感"指标：
- 如果缺货惩罚是持有成本的 4 倍（如本例 $p=8, h=2$），备到 80% 分位数
- 如果两者相等（$p=h$），备到 50% 分位数（中位数）
- 如果缺货惩罚是持有成本的 1/4（$p=2, h=8$），只备到 20% 分位数

**这是你在任何库存管理系统中都应该先算的一个数。**

### (s, S) 策略为什么自然出现

多周期库存问题的最优策略，在绝大多数情况下是 **(s, S) 策略**：
- $s$ = **再订货点**：库存降到这个水平就触发订货
- $S$ = **订货目标**：订货到库存水平为 $S$

为什么不是"每周期订固定数量"？因为**固定成本**的存在改变了激励：
- 订货一次 50 元 → 希望能一次管好几天的
- 所以要么不订，要订就订多点

你会在实际中看到的 (s, S) 策略变体：
- 零售业：$s=7$ 天的安全库存，$S=14$ 天的目标库存
- 制造业：$s=$ 原料用到还剩 3 天量，$S=$ 补到 30 天量
- 电商：$s=$ 前置时间 x 日均需求 + 安全库存，$S=$ 一个月的预计销量

### 动态规划的核心思想

DP 的精髓是**倒序求解**——从终点往前推：
1. 先想：最后一天，我该怎么做？（这个问题最简单）
2. 然后想：倒数第二天，我该怎么做？（需要考虑最后一天的决定）
3. 以此类推...

这对应了 Bellman 方程的核心：

> **最优策略 = 今天的最佳决策 + 未来的最优决策**

这个思想不仅仅适用于库存——路径规划、资源分配、最优控制，都是同一套数学模型。

### 为什么 DP 不一定能用到实际中

DP 虽然优雅，但它有个致命弱点——**维数灾难**：
- 1 个产品、1 个仓库、容量 1000 → 1001 个状态 → ✅ 可解
- 10 个产品、1 个仓库 → $1001^{10}$ 个状态 → ❌ 不可解

实际中库存有成千上万种 SKU，DP 根本算不动。这时候就需要：
- **近似动态规划（ADP）**：用神经网络近似值函数
- **强化学习**：通过采样学习最优策略
- **启发式策略**：比如固定 (s, S) 参数，用仿真调优

## 9. 延伸思考

1. **提前期（Lead Time）**：如果订货不是当天到，而是 3 天后才到
   - 模型需要扩展状态：在途库存 + 现有库存
   - 这就是**多级库存系统**的基础

2. **批量折扣（Quantity Discount）**：如果订得越多，单价越便宜
   - 目标函数不再是线性的
   - 变成凸优化问题

3. **多级供应链（Multi-Echelon）**：如果考虑供应商 → 制造商 → 零售商
   - 这就是**牛鞭效应**的数学来源
   - 需要协同多个节点的库存决策

4. **需求预测与库存的集成**：如果需求分布不是已知而是需要预测
   - 这就是**预测 + 决策**的端到端问题
   - 用 ML 做需求预测 + OR 做库存优化

5. **鲁棒库存策略**：如果你只知道需求的范围 [5, 20]，不知道具体分布
   - 不再用"期望值"，而是"最坏情况"
   - 这就是鲁棒优化的领域
