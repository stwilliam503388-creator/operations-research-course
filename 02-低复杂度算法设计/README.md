# 低复杂度算法设计与优化技术基础课程

> 一个周末，从零到能识别问题复杂度、设计低复杂度算法解决实际问题。

---

## 这份教程是什么

这是一份给**有编程基础（能写 Python/C++）、对数据结构有基本了解（数组、链表、树）、但算法复杂度分析停留在「背 Big-O」阶段**的技术人写的自学教程。它不讲虚的——每一章都围绕「为什么这个算法快/慢」和「这个问题到底长得像哪种经典模式」展开。读完以后，你不会成为 ACM 金牌选手，但你一定会：

- 拿到一个新问题，10 分钟内判断出「这题的复杂度下限在哪」
- 把「暴力解」系统地改进到「低复杂度解」
- 看懂 LeetCode 竞赛题解区说的「DP + 状态压缩」、「分治优化」、「单调队列加速」是什么意思，并且自己也能写出来

## 术语速查表

下面的表是你翻任何一章时会反复回来查的东西。每个术语只有一句话解释——够你「认出来」就行，不需要背。

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 时间复杂度 | Time Complexity | 输入规模翻倍后，你的程序慢多少倍——O(n) 是翻倍后慢一倍，O(n²) 是翻倍后慢四倍 |
| 空间复杂度 | Space Complexity | 你的程序用了多少额外内存——包括输入的、偷的、借的所有空间 |
| 大O记号 | Big-O Notation | 只关心增长趋势，不关心常数——就像只看增长率不看起跑线 |
| 分治 | Divide & Conquer | 看不懂的大问题切成几个看得懂的小问题，分别解完再合并 |
| 动态规划 | Dynamic Programming | 把问题的答案记在小本本上，下次遇到同样的问题直接抄答案，不用重算 |
| 贪心 | Greedy | 每一步选当前最好的，赌全局也是最好——短视但有时候是好事 |
| 状态空间 | State Space | 一个问题所有可能的「情况」——你站在每个状态时面临的选择 |
| 状态转移方程 | State Transition | 从一个状态到下一个状态的「规则说明书」 |
| 最优子结构 | Optimal Substructure | 问题的最优解里藏着子问题的最优解——先做对每一小步，最后才能走对整条路 |
| 无后效性 | No Aftereffect | 过去已经发生了的事，不会改变你未来的决策逻辑——翻篇就是翻篇 |
| 记忆化 | Memoization | 把算过的结果存起来，下次要查不重算——跟缓存一个道理 |
| 递推 | Recurrence | 从最小的问题开始往上推——你知道了第1天和第2天，就能推出第3天 |
| 递归 | Recursion | 函数调自己——把大问题拆到不能再拆为止，再一层层返回答案 |
| 剪枝 | Pruning | 提前发现「这条路走的再多也不可能是最好的」，掉头 |
| 状态压缩 | State Compression | 用整数（bitmask）来代表一个「已选了哪些元素」的状态——节省空间，方便位运算 |
| 滑动窗口 | Sliding Window | 用一个窗口扫过数组，窗口内维护你需要的统计信息——O(n) 的典型代表 |
| 双指针 | Two Pointers | 两个指针对着一维数组从头尾或同向移动，配合缩小搜索范围 |
| 单调栈/队列 | Monotonic Stack/Queue | 栈或队列里的元素永远保持递增/递减——帮你快速找到下一个更大/更小的元素 |
| 回溯 | Backtracking | 试错——走不通就退一步换一条路。穷举但不傻举 |
| 二分答案 | Binary Search on Answer | 不知道答案具体是多少，但是猜一个值然后问「比这个值可行吗？」——像猜数字游戏 |
| 摊还分析 | Amortized Analysis | 单次操作偶尔很慢但平均下来很快——就像每年双十一抢购很堵，但平均到全年还是很通畅 |
| NP完全 | NP-Complete | 计算机科学家还没找到快速解法的问题——如果你能证明你的问题跟它等价，就别追求完美解了 |

> 读到任何一个术语忘了什么意思，回到这里看「一句话人话」列，3 秒搞定。

## 环境准备

本课程所有案例用 **Python 3.9+** 实现，代码独立可运行，无外部依赖（仅标准库）。

```bash
# 1. 创建虚拟环境
python3 -m venv algo-env
source algo-env/bin/activate

# 2. 进入课程目录
cd low-complexity-algo-course

# 3. 确认 Python 版本
python --version  # >= 3.9
```

可选工具（增强调试体验）：
- **matplotlib**：用于可视化算法执行过程（案例 03, 06）
- **timeit / pytest-benchmark**：用于性能对比测试

> ⚠️ 本教程所有案例均可用纯 Python 跑通。绘图和基准测试部分均标注了可选依赖。

## 你需要什么预备知识

- **编程基础**：能写函数、循环、列表/字典。熟悉「函数调函数」
- **基本数据结构**：知道数组、链表、栈、队列、哈希表是什么——但不用精通
- **高中数学**：懂对数（log₂）、指数、求和符号 Σ——够了
- **不需要任何算法竞赛经验**——本教程从零定义每一个概念
