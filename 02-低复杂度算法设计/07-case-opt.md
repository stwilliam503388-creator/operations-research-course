# 案例7：实时数据流优化（滑动窗口 + 单调队列）

> ★★★★☆ [进阶] 一个数据流，两种优化技巧，三组复杂度对比。

---

## 场景

你是一家金融科技公司的后端工程师。系统需要实时处理股票价格流——每秒钟涌入成千上万个价格点。产品经理给你提了两个需求：

**需求 A（窗口最值）**：给定一个长度为 n 的价格数组和一个窗口大小 k，对于每个窗口位置，输出窗口内的最大值——你需要实时计算「过去 k 秒的最高价」。

**需求 B（下一个更大值）**：对于数组中的每一个价格，找出「下一个比它更高的价格」出现在多少秒之后——用于自动止盈信号。

两个需求都基于同一个输入数据流。产品经理希望做到 **亚秒级响应**，而数据规模是 n = 10^5 甚至 10^6。

你一开始的直觉：「这不就是两个 for 循环吗？」

---

## 分析

### 暴力解的问题

需求 A（滑动窗口最大值）：
```
对于每个 i 从 0 到 n-k:
    扫描 [i, i+k-1] 找最大值
```
复杂度：O(n·k)。当 n=10^5, k=10^4 时，操作数 ≈ 10^9——在 Python 中需要几十秒。

需求 B（下一个更大元素）：
```
对于每个 i 从 0 到 n-1:
    从 i+1 往后找第一个大于 arr[i] 的值
```
复杂度：O(n²)。当 n=10^5 时，操作数 ≈ 5×10^9——不可能实时。

### 核心洞察

两个问题的共同点：**信息在相邻窗口/元素之间有重复利用价值**。

| 问题 | 暴力在算什么 | 可复用信息 |
|------|-------------|-----------|
| 窗口最大值 | 每个窗口独立扫描 | 窗口 [i, i+k-1] 和 [i+1, i+k] 有 k-1 个元素重叠 |
| 下一个更大值 | 每个元素往后找 | 一旦 A > B，B 就不可能是 A 左边任何元素的「下一个更大值」 |

---

## 核心思路

### 需求 A：滑动窗口最大值 → 单调队列（双端队列）

维护一个**单调递减队列**，队列里存的是**数组下标**（不是值）：

```
窗口滑动规则：
1. 右边界进入：新元素从队尾入队，把队尾比它小的全部弹出
   → 保证队列从头到尾是递减的
2. 左边界离开：如果队头元素正好是左边界要离开的元素，弹出队头
   → 保证队头始终在窗口内
3. 队头就是当前窗口的最大值
```

**每个元素入队一次、出队一次** → 摊还 O(n)。

### 需求 B：下一个更大值 → 单调栈（递增栈）

从右往左扫描（或从左往右），维护一个**单调递减栈**（栈里从底到顶递减）：

```
扫描规则（从右往左）：
1. 当前元素入栈前，把所有 ≤ 当前元素的栈顶弹出
2. 弹出后，栈顶（如果存在）就是「下一个更大的元素」
3. 当前元素入栈
```

**每个元素入栈一次、出栈一次** → 摊还 O(n)。

## 代码概览

完整代码见 `code/case07_opt.py`。以下是核心函数骨架：

```python
# 滑动窗口最大值（单调队列 O(n)）
def sliding_max_deque(arr, k):
    from collections import deque
    dq = deque()  # 存下标，值单调递减
    result = []
    for i, v in enumerate(arr):
        # 入队：弹出所有比 v 小的队尾
        while dq and arr[dq[-1]] <= v:
            dq.pop()
        dq.append(i)
        # 出队：队头超出窗口左边界
        if dq[0] == i - k:
            dq.popleft()
        # 记录：窗口形成后取队头
        if i >= k - 1:
            result.append(arr[dq[0]])
    return result
```

```python
# 下一个更大值（单调栈 O(n)）
def next_greater(arr):
    n = len(arr)
    res = [-1] * n
    stack = []  # 存下标，值单调递减
    for i in range(n - 1, -1, -1):
        while stack and arr[stack[-1]] <= arr[i]:
            stack.pop()
        if stack:
            res[i] = arr[stack[-1]]
        else:
            res[i] = -1
        stack.append(i)
    return res
```

```python
# 暴力参考（用于对比）
def sliding_max_brute(arr, k):
    return [max(arr[i:i+k]) for i in range(len(arr)-k+1)]

def next_greater_brute(arr):
    n = len(arr)
    return [next((arr[j] for j in range(i+1, n) if arr[j] > arr[i]), -1) for i in range(n)]
```

## 洞察：摊还分析的直觉

单调队列和单调栈最让人困惑的地方是：「明明代码里有个 while 循环，为什么不是 O(n²)？」

答案在**摊还分析（Amortized Analysis）**：

```
每个元素「入栈/入队」恰好 1 次
每个元素「出栈/出队」最多 1 次（一旦弹出，永不回来）
所以：所有 while 循环的总迭代次数 ≤ n
     总操作次数 = O(n)
```

**精确类比**：你在超市购物，每件商品拿进购物车（入队）一次，结账时拿出（出队）一次。虽然收银台前可能排了长队（while 循环很长），但每个人只排一次队。

### 对比表

| 问题 | 暴力复杂度 | 优化复杂度 | 核心数据结构 | 摊还分析关键 |
|------|-----------|-----------|-------------|-------------|
| 滑动窗口最大值 | O(n·k) | O(n) | 双端队列 (deque) | 每个元素入队/出队各一次 |
| 下一个更大元素 | O(n²) | O(n) | 单调栈 | 每个元素入栈/出栈各一次 |

## 延伸

### 滑动窗口的变种

| 变种 | 窗口统计量 | 维护方式 | 复杂度 |
|------|-----------|---------|-------|
| 最小值 | 跟最大值镜像 | 单调递增队列 | O(n) |
| 中位数 | 难（需要两个堆） | 对顶堆 | O(n log k) |
| 平均值 | 简单 | 前缀和或累计和 | O(n) |
| 众数 | 较难 | 哈希表 + 计数 | O(n) 平均 |
| 第 K 大 | 适中 | 堆 + 延迟删除 | O(n log k) |

### 单调栈的经典应用

- **接雨水**：对每个位置，用单调栈找左右第一个比它高的柱子
- **柱状图中最大矩形**：对每个高度，用单调栈找左右第一个比它矮的柱子
- **每日温度**：下一个更大元素的标准问题
- **股票价格跨度**：上一个更大元素

### 当数据流是「无限流」时

如果数据源源不断地来（真正的流式场景）：
- 滑动窗口最大值 → **Segment Tree** 或 **Sparse Table**（支持实时查询，但更新有代价）
- 下一个更大值 → 只能维护到当前时间点
