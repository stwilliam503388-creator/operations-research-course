# 附录D：推荐阅读

> 从入门到深入，按照阅读顺序排列。每本书附带一句话评价——帮你判断「这本值不值得我花时间」。

---

## 必读（读完一本就值了）

### 《算法导论》（CLRS）— Thomas H. Cormen 等

- **评价**：计算机算法的「圣经」。如果你只能买一本算法书，买这本。
- **优点**：数学证明严谨，覆盖了所有经典范式的理论根基
- **缺点**：厚（1,300 页），不适合通勤读
- **建议阅读策略**：不要从头读到尾。需要时当字典查——比如你写 DP 时读第 15 章，写贪心时读第 16 章
- **本课程对应**：第 15 章（DP）、第 16 章（贪心）、第 4 章（分治与主定理）、第 17 章（摊还分析）

### 《编程珠玑》（Programming Pearls）— Jon Bentley

- **评价**：最被低估的算法好书。**薄**（230 页），但读一遍能顶三本厚书。
- **优点**：不讲理论，只讲「一个真实工程问题 → 用算法解决」的故事。每一章都能让你拍大腿说「原来还可以这样想」
- **缺点**：例子偏老（1980 年代），但思想完全不过时
- **本课程对应**：第 8 章（滑动窗口思想最早出自这里）、第 9 章（代码优化原则）

## 进阶（想深入理解某一方向时读）

### 《算法设计指南》（The Algorithm Design Manual）— Steven Skiena

- **评价**：CLRS 的实用版。前半部分是算法设计策略，后半部分是一本「算法目录」——遇到某个类型的问题，直接查目录找到对应的解法
- **适合**：工作中需要快速上手解决问题的工程师
- **特点**：作者幽默风趣，读起来不累

### 《Competitive Programming》— Halim 兄弟

- **评价**：算法竞赛圈的「红宝书」。强烈推荐它的**决策表**——每个问题类型对应的最优解法
- **适合**：想刷题拿面试、或者打比赛的读者
- **特点**：代码以 C++ 为主，但思路通用

### 《具体数学》（Concrete Mathematics）— Graham, Knuth, Patashnik

- **评价**：如果你想知道「为什么 O(n log n) 的底数不 log₂ 是 log e？」——这本书里有答案
- **注意**：数学要求较高，不适合初学

## 在线资源

### CP-Algorithms (cp-algorithms.com)

- **评价**：免费的算法百科全书。每个主题都有详细的文字解释 + 代码实现
- **特点**：比维基百科更聚焦算法竞赛，质量极高
- **推荐章节**：DP、Bit manipulation、Sparse Table、LCA

### USACO Guide (usaco.guide)

- **评价**：美国信息学奥赛的官方配套教程。结构清晰，习题质量高
- **适合**：想系统刷题的读者（按难度分级，从青铜到白金）

### LeetCode 题解区

- **评价**：最实用的学习资源之一。每一个高频题都有多语言、多解法对比
- **建议**：看「Solution」标签下的官方题解，再看「Discuss」标签下的高票回答。同一个题看 3-5 种解法，理解为什么「这个解法更快」

## 一句话选书指南

| 你的目标 | 读这本 |
|---------|-------|
| 打基础，系统学习 | CLRS（算法导论）部分章节 |
| 快速上手解决工作问题 | 《算法设计指南》 |
| 想看最有趣的故事 | 《编程珠玑》 |
| 准备面试刷题 | LeetCode 官方题解 + 《Competitive Programming》 |
| 追求极致的数学理解 | 《具体数学》 |

> **最后一条建议**：读书 ≠ 学会算法。最好的学习方式是：读一个概念 → 在 LeetCode 上找 3 道对应的题做 → 回来再看一遍书。重复三轮，你就真的懂了。
