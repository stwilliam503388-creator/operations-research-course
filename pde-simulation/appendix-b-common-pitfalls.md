# 附录B：TOP 5 必踩坑

> 以下五个坑，每一个初学 PDE 的人至少会踩三个。踩了不可怕——可怕的是踩了不知道自己踩了。读一遍，也许能帮你省掉三天的调试时间。

---

## 坑 1：边界条件写错了 / 写反了 / 忘了

**症状**：结果看起来「差不多」但不对；或者完全不合物理规律（比如温度越算越高）。

**典型翻车现场**：

```
热传导方程，芯片散热问题。
你设了「左边界温度 100°C，右边界温度 25°C」。
结果算出来的温度分布是对称的——最高温度在中间。
你心想：导体内部的焦耳热嘛，中间最热，合理！
实际上你的边界条件设的是「左右边界都 100°C」，而不是「左 100°C 右 25°C」。
```

**更深层的坑**：Dirichlet 和 Neumann 的符号。

```
Neumann 条件：-k ∂T/∂n = h(T - T_amb)
```

这个负号「-k ∂T/∂n」是傅里叶定律的标准形式。忘记它，你的对流边界就会变成「反向加热」边界——热量从环境往你的物体流，而不是从物体往环境散。

**自救方案**：
1. 每个边界用大白话写一遍（「左边固定 100°C」「右边绝热」）
2. 先做简单检验：全部 Dirichlet、全部 Neumann、混合各跑一遍，看结果是否物理
3. 初始条件也统一检查一遍——你设了 u(x,0) = 0 吗？还是忘记了导致从零开始？

## 坑 2：CFL 条件被忽略

**症状**：显式格式跑热传导，刚启动几秒就「NaN explosion」——矩阵里全是 `nan` 或 `inf`。

**原因**：热传导显式格式的稳定条件是：

\[
\alpha \frac{\Delta t}{\Delta x^2} \leq 0.5
\]

如果你把 Δx 从 0.1 降到 0.01（精度），Δt 必须从 0.1s 降到 0.001s——100 倍的代价。很多人只缩了网格，忘了缩步长。

**量化案例**：

```
α = 1e-5 m²/s (典型土壤)
Δx = 0.01 m  →  Δt ≤ 0.5 × 0.01² / 1e-5 = 5 秒  ✅ 还行
Δx = 0.001 m →  Δt ≤ 0.5 × 0.001² / 1e-5 = 0.05 秒  😰 开始痛了
Δx = 0.0001 m→  Δt ≤ 0.5 × 0.0001² / 1e-5 = 0.0005 秒  🤯 炸了
```

对于网格加密 10 倍，时间步长缩短 100 倍。这就是为什么热传导的显式格式在工程中很少用——隐式格式在这里是救星。

**自救方案**：始终在代码开头计算并打印 CFL 数。如果它接近或超过了稳定性极限，程序会自己告诉你——而不是等你发现结果炸了。

## 坑 3：矩阵组装的索引错位

**症状**：解振荡、不对称、或者在某些节点出现「毛刺」。

**典型翻车现场**：

```
你用五点差分：w_{i-2} - 4w_{i-1} + 6w_i - 4w_{i+1} + w_{i+2}
写代码：
  A[i, i-2] = 1
  A[i, i-1] = -4
  A[i, i]   = 6
  A[i, i+1] = -4
  A[i, i+2] = 1

看上去没问题对吧？但是你的循环从 i=0 到 i=N-1。
当 i=0 时，i-2 = -2 —— Python 自动变成倒数第二个索引。
于是边界上的列被错误填入，你的矩阵变成了一个奇怪的「带」。
```

Python 的负索引是双刃剑：平时很方便，在矩阵组装时是绝对的噩梦。这种 bug 不报错（语法上完全正确），只会让结果莫名其妙地错。

**自救方案**：
1. 始终从 i=2 循环到 i=N-3，边界单独处理
2. 组装完后打印矩阵的前几行和最后几行手动检查
3. 用 `plt.spy()` 或 `plt.matshow(A)` 可视化矩阵结构——一眼看出哪里不对

## 坑 4：网格无关性没有验证

**症状**：跑了一次结果就信了，不知道自己的解跟真实解差多少。

**真实案例**：

```
你用 20×20 的网格算一个芯片温度场，得到最高温 95°C。
报告里写「芯片最高温 95°C，在安全范围内」。
实际上用 100×100 的网格算出来是 125°C——根本不是安全范围。
你的「95°C」只是网格太粗的离散化误差。
```

网格越粗，数值耗散越强——就像一个低分辨率 JPEG，细节全被抹掉了。热点的峰值温度在粗网格上系统性偏低。

**自救方案**：至少用三种网格密度（N, 2N, 4N）做收敛性分析：
- 如果 N→2N 的变化 < 5%，你的网格够密
- 如果变化 > 20%，继续加密
- 记录收敛阶（外推估计精确解）

## 坑 5：量纲不一致（唯一一个完全不该犯的错误）

**症状**：解的数量级完全不对——挠度算出来是 10⁹ 米、温度是 10⁻¹² °C、速度是光速的 1000 倍。

**原因**：单位混用。最常见的翻车组合是「米 + 毫米」或「瓦 + 千瓦」。

```
你的弹性模量 E = 200 GPa = 2e11 Pa
你的梁长度 L = 1.0 m
你的截面宽度 b = 50 mm … 等等，你写代码时写的是 b = 50 还是 b = 0.05？
```

50 mm = 0.05 m。如果你用了 b=50（以为 mm）而 I 公式里的其他量都是 m——惯性矩 I = bh³/12 就差了 50⁴×... 倍。挠度 w ∝ 1/I，你算出来的结果会偏小几千倍。

**自救方案**：
1. 所有物理量全部用 SI 基本单位（m, kg, s, K, Pa, N）
2. 输入输出时再换算（1mm = 0.001m, 1MPa = 1e6Pa）
3. 在代码注释里标注每个参数的单位
4. 算一个简单 case 跟解析解对比——量纲错误会立刻暴露

## 踩坑急救卡

| 坑 | 快速自检 | 一句话自救 |
|----|---------|-----------|
| 边界条件 | 检查每个边界的大白话描述 | 打印边界值，跟预期对比 |
| CFL 爆炸 | 打印 CFL 数 | 换隐式格式或缩 Δt |
| 索引错位 | `np.spy(A)` 可视化矩阵 | 循环从 2 到 N-3 |
| 网格无关性 | 加密网格看解的变化 | 做收敛性分析 |
| 量纲 | 解的数量级离谱 | 全部转 SI 单位 |
