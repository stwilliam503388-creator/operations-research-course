## 场景：手机芯片满负荷运行

想象一下：你正在用手机打一局大型游戏，芯片全速运转——功耗直奔 10W+。芯片只有指甲盖大小（约 10mm × 10mm），热量在这么小的面积上高度集中。

你关心的问题是：

- **最高温度在哪？** — 靠近芯片核心的区域温度最高
- **温度分布达到稳态需要多久？** — 几毫秒还是几秒？
- **散热设计合理吗？** — 左端靠近散热器，右端在芯片深处

### 物理建模

我们做一个合理的简化：把芯片近似为一根 **1D 细棒**，长度为 L。

```
┌────────────────────────────────────────┐
│                                        │
│  散热器         芯片内部               │
│  x=0     → 热量传播方向 →     x=L     │
│  85°C                             绝热  │
│  (固定温)                        (无热流)│
│                                        │
└────────────────────────────────────────┘
```

**边界条件**：
- **左边界** (x=0)：Dirichlet 条件，温度固定为 85°C（热点区域）
- **右边界** (x=L)：Neumann 条件，绝热（∂u/∂x = 0），芯片内部深处热量散不出去

**初始条件**：整根芯片初始温度均匀，为环境温度 25°C。

---

## 物理方程

**热传导方程（1D 形式）**：

\[
\frac{\partial u}{\partial t} = \alpha \frac{\partial^2 u}{\partial x^2}
\]

其中：

| 符号 | 含义 | 典型值 |
|------|------|--------|
| u(x,t) | 温度 [°C] | 随时间变化 |
| α | 热扩散率 [m²/s] | 硅 ~ 1.0×10⁻⁴ m²/s |
| x | 空间坐标 [m] | 0 ~ 10 mm |
| t | 时间 [s] | 0 ~ 若干秒 |

### 物理直觉

这个方程说的是：**某点的温度变化率，正比于该点温度分布的「弯曲程度」（二阶导数）**。

- 如果温度分布是拱形（中间凸起）→ 凸起会逐渐变平 → 温度下降
- 如果温度分布是凹形（中间凹陷）→ 凹陷会逐渐填平 → 温度上升
- 最终达到直线分布（二阶导数为零）→ 稳态

这就像你把一滴墨水滴进水里——墨水的浓度梯度驱动扩散过程，最终均匀分布。

## 数值方法：FDM 显式 FTCS 格式

### 离散化

把空间分成 nx 个网格点，时间推进 n 步：

**FTCS = Forward in Time, Central in Space**

```
时间前向差分:    ∂u/∂t ≈ (u^{n+1}_i - u^n_i) / Δt
空间中心差分:    ∂²u/∂x² ≈ (u^n_{i-1} - 2u^n_i + u^n_{i+1}) / Δx²
```

代入热传导方程，得到显式更新公式：

\[
u^{n+1}_i = u^n_i + r \cdot (u^n_{i-1} - 2u^n_i + u^n_{i+1})
\]

其中：

\[
r = \frac{\alpha \Delta t}{\Delta x^2}
\]

### 生活类比

你有一排城市排成一条线，每个城市都知道自己和邻居今天的温度。你用以下规则来预测明天的温度：

> **明天我的温度 = 今天我的温度 + r × (左边比我热多少 + 右边比我热多少 - 2×我比平均热多少)**

r 越大，扩散越快。但如果 r 太大，预测就会「过头」——数值解会震荡甚至爆炸。

---

## 稳定性条件

对于 FTCS 显式格式，稳定性条件是：

\[
r = \frac{\alpha \Delta t}{\Delta x^2} \leq 0.5
\]

### 为什么？

想象一个极端温度分布：`[100, 0, 100]`。FTCS 更新中间点：

```
u_new(中间) = 0 + r × (100 - 0 + 100 - 0) = 200r
```

如果 r > 0.5，中间点的温度会超过 100——物理上不可能！下一时间步，这种违反物理的分布会指数级放大，最终「爆炸」。

### 网格加密的代价

热传导方程的稳定性条件非常苛刻：

- 网格加密一倍（Δx 减半）→ Δt 需要缩小到 **1/4**
- 网格加密 10 倍 → 时间步长缩小到 **1/100**

这就是为什么工程中通常用 **隐式格式** 来求解热传导问题——隐式格式无条件稳定，可以用大步长推进。

---

## 边界条件处理

### Dirichlet 边界（左边界，固定温度）

直接赋值即可：

```python
u[n+1, 0] = T_left  # 固定为 85°C
```

### Neumann 边界（右边界，绝热）

绝热条件：∂u/∂x = 0，意味着边界点的温度等于相邻内点的温度：

```python
u[n+1, -1] = u[n+1, -2]  # 绝热：梯度为零
```

也可以用更精确的虚拟网格点方法，但对于入门案例，一阶近似已经足够。

### 核心计算代码（显式 FTCS）

```python
# 内部点: FTCS 更新
u[n+1, 1:-1] = (u[n, 1:-1]
                + r * (u[n, :-2] - 2*u[n, 1:-1] + u[n, 2:]))

# 左边界: Dirichlet (固定温度)
u[n+1, 0] = T_left

# 右边界: Neumann (绝热)
u[n+1, -1] = u[n+1, -2]
```

就这三行——这就是 1D 热传导方程 FTCS 格式的全部核心。

随时间推进，热量从左边界向右扩散。芯片温度分布逐渐从「左高右低」变为「整体接近 85°C」——因为右边界绝热，热量无法逸出，最终整个芯片温度均匀化到左边界温度。

## 洞察

### 1. 热传导是「抹平」的过程

热传导方程不像波动方程那样「传播信号」——它是在「抹平差异」。这是由方程本身决定的：一阶时间导数 + 二阶空间导数 → 系统能量单调递减。

### 2. 为什么热传导需要隐式格式

对于显式格式，稳定性条件 r ≤ 0.5 意味着：
- 网格加密一倍 → 时间步长缩到 1/4 → 计算量增加 **8 倍**
- 网格加密 10 倍 → 时间步长缩到 1/100 → 计算量增加 **1000 倍**

而隐式格式（Crank-Nicolson）无条件稳定，可以用大步长推进到稳态。

### 3. 二维/三维扩展

1D 代码可以直接扩展到 2D/3D：

- 2D 热传导：uₜ = α(uₓₓ + u_yᵧ)
- 空间中心差分扩展到 2D：5 点模板
- 稳定性条件：αΔt(1/Δx² + 1/Δy²) ≤ 0.5

## 延伸

### 1. 非线性热传导

真实材料的热扩散率 α 通常随温度变化：α = α(u)。方程变为非线性：

\[
\frac{\partial u}{\partial t} = \frac{\partial}{\partial x} \left( \alpha(u) \frac{\partial u}{\partial x} \right)
\]

处理方法：在 FTCS 中，r 在空间上不再均匀——每个网格点有不同的 α 值。

### 2. 3D 芯片散热

真实芯片的散热是 3D 问题，涉及：
- 芯片本身（硅）
- 散热器（铜或铝）
- 热界面材料（TIM）
- 多层结构 → 需要处理材料界面的热传导

### 3. 与有限元（FEM）对比

对于简单几何（矩形芯片），FDM 足够。但对于真实芯片的复杂走线结构和散热鳍片，FEM 更合适。

### 4. 瞬态热分析在工程中的典型应用

- **电子设备**：芯片结温计算、PCB 散热设计
- **汽车**：发动机热管理、刹车盘温升
- **建筑**：墙体传热、保温材料设计
- **制造**：焊接热影响区分析、淬火过程模拟
