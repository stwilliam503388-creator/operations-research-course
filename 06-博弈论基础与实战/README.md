# 博弈论基础与实战课程

> 一个周末，从零到能看懂博弈论的核心概念、把竞争/合作场景建模成博弈、以及用 Python 求解纳什均衡。

---

## 这份教程是什么

这是一份给**有基础数学和 Python 能力、但对博弈论完全陌生**的技术人/产品经理/策略分析师写的自学教程。它不讲学院派的定理证明——每一章都围绕「这个博弈模型描述了什么样的现实冲突」和「这个均衡概念告诉你该怎么决策」展开。

读完以后，你不一定会成为博弈论专家，但你一定会：

- 跟同事说清楚「纳什均衡、帕累托最优、占优策略」各自在说什么
- 看到一个商业竞争场景，判断是囚徒困境、协调博弈还是鹰鸽博弈
- 用 Python 求解小型博弈的纳什均衡，并做灵敏度分析
- 理解拍卖、谈判、机制设计的基本逻辑

## 术语速查表

✅ **核心术语**（先记住这 15 个，其余的以后查）

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 博弈 | Game | 你的收益不仅取决于你怎么选，还取决于别人怎么选 |
| 参与者 | Player | 做决策的人——你、你的竞争对手、拍卖中的出价人 |
| 策略 | Strategy | 你在所有可能情况下打算怎么做 |纯策略 | Pure Strategy | 确定的选择——我选 A 就是 A |混合策略 | Mixed Strategy | 随机化选择——我 60% 概率选 A，40% 概率选 B |
| 收益 | Payoff | 每种策略组合下的结果——利润、得分、效用 |
| 纳什均衡 | Nash Equilibrium | 给定别人的选择，没有人想单方面改变自己的选择 |
| 占优策略 | Dominant Strategy | 不管别人怎么选，我选这个总是最好的 |
| 囚徒困境 | Prisoner's Dilemma | 每个人都选对自己最好的，结果对所有人都不好 |
| 帕累托最优 | Pareto Optimality | 不可能让任何人更好而不让另一个人更差 |
| 零和博弈 | Zero-Sum Game | 我赢多少，你就输多少——蛋糕大小固定 |
| 非零和博弈 | Non-Zero-Sum Game | 合作可以把蛋糕做大——双赢或双输 |
| 序贯博弈 | Sequential Game | 你先出招，我看到后再回应——下棋、谈判 |
| 子博弈完美均衡 | Subgame Perfect Equilibrium | 在博弈的每个阶段，你的选择都是理性的 |
| 可信威胁 | Credible Threat | 说到做到——如果我威胁你但你不在乎，这个威胁不可信 |
| 共同知识 | Common Knowledge | 我知道你知道我知道你知道…——所有参与者都知道的事 |

🔧 **进阶术语**（用到时回来查）

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 贝叶斯博弈 | Bayesian Game | 我不知道你是什么类型，只知道概率分布 |
| 信号传递 | Signaling | 通过你的行动推测你的隐藏信息 |
| 逆向归纳 | Backward Induction | 从最后一步往前推——你在每一步都会选最优 |
| 贴现因子 | Discount Factor | 未来的收益在今天值多少钱 |
| 无名氏定理 | Folk Theorem | 如果博弈重复足够多次，任何可行的收益都可以成为均衡 |
| 沙普利值 | Shapley Value | 每个参与者在合作中应得的公平份额 |
| 核仁 | Nucleolus | 另一种公平分配方案 |
| 机制设计 | Mechanism Design | 反过来想——你想要什么结果，设计规则让人自愿走向它 |
| 显示原理 | Revelation Principle | 机制设计可以简化为「让人们说实话」 |
| 拍卖收入等价定理 | Revenue Equivalence | 不同拍卖形式下卖家的期望收入一样（在一定条件下）|
