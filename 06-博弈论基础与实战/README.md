# 博弈论基础与实战课程

> 一个周末，从零到能看懂博弈论的核心概念、把竞争/合作场景建模成博弈、以及用 Python 求解纳什均衡。

## 内容质量标准

本课程遵循根目录的 [课程内容质量规范](../COURSE_CONTENT_GUIDE.md)：用人话解释核心直觉，同时明确模型假设、适用边界和可运行验证方式。

## 这份教程是什么

这是一份给**有基础数学和 Python 能力、但对博弈论完全陌生**的技术人/产品经理/策略分析师写的自学教程。它不讲学院派的定理证明——每一章都围绕「这个博弈模型描述了什么样的现实冲突」和「这个均衡概念告诉你该怎么决策」展开。

读完以后，你不一定会成为博弈论专家，但你一定会：

- 跟同事说清楚「纳什均衡、帕累托最优、占优策略」各自在说什么
- 看到一个商业竞争场景，判断是囚徒困境、协调博弈还是鹰鸽博弈
- 用 Python 求解小型博弈的纳什均衡，并做灵敏度分析
- 理解拍卖、谈判、机制设计的基本逻辑

## 术语速查表

✅ **核心术语**（先记住这 15 个，其余的以后查）

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 博弈 | Game | 像几个人同时下棋：你赚不赚，不只看你怎么走，也看别人怎么走 |
| 参与者 | Player | 局里的决策者：你、竞争对手、平台、买家、卖家都可能是玩家 |
| 策略 | Strategy | 不是只想一步，而是提前写好“遇到什么情况我就怎么做”的计划 |
| 纯策略 | Pure Strategy | 明牌做一个确定选择：我就选 A，不掷骰子 |
| 混合策略 | Mixed Strategy | 把选择随机化：像石头剪刀布，不能老出同一招 |
| 收益 | Payoff | 每种出招组合带来的结果，可以是利润、分数、面子或满意度 |
| 纳什均衡 | Nash Equilibrium | 像大家都卡在一个局面：别人不变时，谁单独改主意都不划算 |
| 占优策略 | Dominant Strategy | 像万能优选项：不管别人怎么出，我选它都不吃亏 |
| 囚徒困境 | Prisoner's Dilemma | 每个人都按个人利益做“聪明选择”，最后大家一起变差 |
| 帕累托最优 | Pareto Optimality | 已经没有白捡的改进了：想让一个人更好，至少要让别人更差 |
| 零和博弈 | Zero-Sum Game | 像分固定大小的蛋糕：我多拿一口，你就少一口 |
| 非零和博弈 | Non-Zero-Sum Game | 蛋糕大小会变：合作可能一起变好，也可能互坑一起变差 |
| 序贯博弈 | Sequential Game | 像下棋或谈判：你先动，我看见你的动作后再回应 |
| 子博弈完美均衡 | Subgame Perfect Equilibrium | 不只开局理性，走到任何分支以后，后面的选择也得说得通 |
| 可信威胁 | Credible Threat | 威胁要真的舍得执行才有用；如果对方知道你不敢做，就只是嘴硬 |
| 共同知识 | Common Knowledge | 不只是“我知道”，而是我知道你知道、你也知道我知道，大家都心照不宣 |

🔧 **进阶术语**（用到时回来查）

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 贝叶斯博弈 | Bayesian Game | 像打牌时不知道对方手里是什么，只能根据概率猜他的类型 |
| 信号传递 | Signaling | 通过行动透露隐藏信息，比如高学历、质保、报价都可能是在发信号 |
| 逆向归纳 | Backward Induction | 像倒着推棋局：先看最后一步谁会怎么选，再一步步倒推回来 |
| 贴现因子 | Discount Factor | 未来收益打几折算到今天；越没耐心，未来的钱越“不值钱” |
| 无名氏定理 | Folk Theorem | 如果以后还要长期打交道，很多“今天忍一下、以后赚回来”的合作都可能稳定 |
| 沙普利值 | Shapley Value | 像算合伙人贡献：谁给团队多带来价值，谁就该多分一点 |
| 核仁 | Nucleolus | 像调解分蛋糕，尽量让最不满意的联盟也别太委屈 |
| 机制设计 | Mechanism Design | 反过来设计游戏规则，让大家为了自己好，最后也走向你想要的结果 |
| 显示原理 | Revelation Principle | 很多复杂机制可以改写成“让大家说真话也最划算”的形式 |
| 拍卖收入等价定理 | Revenue Equivalence | 在一堆严格条件下，不同拍卖玩法给卖家的平均收入可能差不多 |

### 概念边界：均衡不等于最好结果

博弈论最容易误解的一点是：**纳什均衡只说明“没人愿意单方面改”，不说明这个结果对大家最好。**

- 纳什均衡可能很糟，例如囚徒困境里双方都背叛。
- 帕累托最优说的是“没有白捡的改进”，但它不一定公平。
- 占优策略是“无论别人怎么选我都不吃亏”，但很多真实博弈根本没有占优策略。
- 机制设计不是预测别人会怎么玩，而是反过来设计规则，让人自愿走向你想要的结果。
