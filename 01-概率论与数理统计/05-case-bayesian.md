# 案例3：贝叶斯推断 ★★★☆☆

> 一种罕见病发病率 0.1%，检测准确率 99%。如果你检测阳性，实际得病的概率是多少？

---

## 场景

你体检做了一个罕见病的筛查。报告显示"阳性"。

医生告诉你："这个检测的准确率是 99%——病人中 99% 会检测出阳性，健康人中 99% 会检测出阴性。"

你吓坏了——99% 准确率，意味着你有 99% 的概率得了这个病？

**不是。** 让我们用贝叶斯定理来算一下。

## 建模

### 定义事件

- D：得了该病
- T+：检测结果为阳性

### 已知参数

- P(D) = 0.001（先验：发病率 0.1%）
- P(T+ | D) = 0.99（灵敏度：得病的人 99% 检测出阳性）
- P(T+ | ¬D) = 0.01（假阳性率：健康的人 1% 检测出阳性）

### 贝叶斯定理

```
P(D | T+) = P(T+ | D) × P(D) / P(T+)
```

其中：
```
P(T+) = P(T+|D)P(D) + P(T+|¬D)P(¬D)
       = 0.99 × 0.001 + 0.01 × 0.999
       = 0.01098
```

所以：
```
P(D | T+) = 0.99 × 0.001 / 0.01098 ≈ 0.0902
```

**你实际得病的概率只有约 9%**——远低于直觉的 99%。

### 直觉解释

这个病很罕见（0.1%），所以即使检测很准，绝大多数阳性结果其实是假阳性。在 1000 人中：
- 1 个真病人 → 0.99 个真阳性
- 999 个健康人 → 9.99 个假阳性

总共约 11 个阳性中只有 1 个是真阳性——所以约 9%。

## 方法

**贝叶斯更新的完整过程**：

1. **设定先验**：P(D) = 0.001（你最初的信念）
2. **收集证据**：第一次检测阳性
3. **计算后验**：P(D|T+) ≈ 9%（更新后的信念）
4. **把后验作为新先验**：P(D) = 0.09 用于下一次更新
5. **收集新证据**：第二次检测阳性
6. **再次计算**：P(D|T+₂) ≈ 91%

这就是贝叶斯更新的核心：**你每次看到新证据，就把当前的后验当作下一次的先验**。

## 洞察

1. **贝叶斯定理纠正了直觉偏差**。我们的大脑不擅长处理"一个准确率 99% 的测试用于罕见病"这种情况。贝叶斯公式给出了反直觉但正确的答案。

2. **先验：朋友的偏见还是领域知识？**
   - "坏的"先验：没有依据的偏见（"我直觉这药有效"）
   - "好的"先验：基于领域知识的先验（"类似药物的历史成功率是 10%"）
   - 数据量够大时，先验的影响消失

3. **贝叶斯 vs 频率学派在这个场景中**：
   - 频率学派：无法直接回答 P(D|T+)，只能给出检测的灵敏度/特异度
   - 贝叶斯学派：直接给出 P(D|T+)=9%，这正是你想要的答案

4. **贝叶斯更新在实战中的应用**：
   - 推荐系统：先验是用户历史偏好，证据是新的点击行为
   - 搜索引擎：先验是语言模型，证据是用户输入的查询
   - 强化学习：先验是策略的初始估计，证据是环境反馈
