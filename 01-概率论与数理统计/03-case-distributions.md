# 案例1：常见分布拟合与识别 ★★☆☆☆

> 你有一批历史需求数据，不知道它服从什么分布。用 QQ 图和统计检验判断它最可能是正态、对数正态还是指数分布。

---

## 场景

你是某零售公司的数据分析师。运营团队给你一份过去 365 天的日需求记录。为了让案例开箱即跑，`code/python/case03_distributions.py` 会在脚本中生成一份模拟需求数据，而不是依赖外部 CSV。你要回答："这些数据服从什么分布？我们该用哪种模型来预测未来的安全库存？"

你的直觉是需求可能服从正态分布——但你需要用数据验证这个直觉。

## 建模

**PMF vs PDF vs CDF 速查**：

- **PMF（概率质量函数）**：离散变量取值 x 的概率。P(X=x) = p(x)。总和 = 1。
- **PDF（概率密度函数）**：连续变量在 x 处的"密度"。∫f(x)dx = 1。**PDF 的值可以 >1。**
- **CDF（累积分布函数）**：P(X ≤ x)。从 0 到 1 单调递增。任何分布的 CDF 都在同一量级上。

**识别分布的两种方法**：

### 方法1：QQ 图（分位数-分位数图）

- 把数据的分位数和理论分布的分位数做对比
- 如果数据服从该分布 → 点落在 y=x 对角线上
- 尾部偏离 → 厚尾（数据点在对角线下方 = 尾部更胖）

**为什么 QQ 图比直方图好**：
- 直方图的形状受 bin 宽度影响
- QQ 图直接比较分位数，不受人工参数干扰
- QQ 图能更清楚地显示尾部行为

### 方法2：正态性统计检验

| 检验 | 适用场景 | 注意事项 |
|------|---------|---------|
| Shapiro-Wilk | 小样本（n<5000）| 最推荐 |
| Kolmogorov-Smirnov | 任意分布 | 需要指定分布参数 |
| Anderson-Darling | 尾部敏感 | 比 KS 更敏感于尾部偏差 |

**实战经验**：对于大数据集，统计检验几乎总是拒绝正态性（因为真实数据几乎从不完美正态）。实战中看 QQ 图比看 p 值更实用。

## 方法

**拟合步骤**：

1. 加载数据，计算基本统计量（均值、标准差、偏度、峰度）
2. 绘制直方图和核密度估计（KDE）
3. 假设为正态分布，绘制 QQ 图
4. 假设为对数正态分布，对数据取 log 后再做 QQ 图
5. 假设为指数分布，绘制指数 QQ 图
6. 用 Shapiro-Wilk 检验正态性
7. 综合判断：哪个分布的 QQ 图最接近对角线

1. ✅ 从正态分布生成的数据能被 QQ 图正确识别（点在对角线上）
2. ✅ 从指数分布生成的数据不会被误判为正态（QQ 图明显偏离对角线）
3. ✅ 数据量增大时分布识别准确率提高

## 洞察

1. **真实世界的数据很少完美正态**。它们在大多数区域是正态的，但在尾部（极端值）常常更胖（厚尾）。这就是为什么金融数据中"黑天鹅"事件比正态分布预测的多得多。

2. **对数正态分布比正态分布更常见于正数数据**。价格、收入、需求——这些只能为正的数据经常服从对数正态。取 log 后它们就正态了。

3. **QQ 图的尾部比中心更重要**。如果你的应用关心的是极端风险（比如安全库存中的缺货概率），关注尾部行为比关注中心更重要。

4. **不要迷信统计检验**。当 n=1000 时，Shapiro-Wilk 检验几乎一定会拒绝正态性（因为真实数据几乎从不完美正态）。实战中"近似正态"就够了。

## 延伸

- 如果你发现数据是厚尾的，考虑 t 分布或混合分布
- 用 Kolmogorov-Smirnov 检验来比较多个候选分布
- 对于建模目的，考虑是否真的需要知道精确分布——CLT 说样本均值总是正态的
