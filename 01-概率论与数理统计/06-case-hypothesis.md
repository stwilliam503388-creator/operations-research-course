# 案例4：假设检验与A/B测试 ★★★☆☆

> 你的 APP 改了推荐算法，想验证新算法是否显著提高了点击率。

---

## 场景

你是某内容平台的产品经理。推荐算法团队提出了一个新版本，声称能提高用户点击率（CTR）。当前版本的 CTR 是 5%。

你们做了 A/B 测试：
- 对照组（旧算法）：10000 用户，500 次点击（5%）
- 实验组（新算法）：10000 用户，530 次点击（5.3%）

问：提升了 0.3 个百分点，是真实有效还是随机波动？

## 建模

### 假设设定

```
H₀: p_new = p_old    新算法无效
H₁: p_new > p_old    新算法有效（单侧检验）
```

### 检验方法：双比例 z 检验

比较两个比例的检验统计量：

```
z = (p̂_new - p̂_old) / SE

其中 SE = √[p̂(1-p̂)(1/n₁ + 1/n₂)]
      p̂ = (x₁ + x₂) / (n₁ + n₂)  （合并比例）
```

### 决策规则

- z > z_α：拒绝 H₀
- α = 0.05 时，z_0.05 = 1.645（单侧）
- 或者看 p 值：p < 0.05 → 拒绝 H₀

### 效应量

p 值告诉你是"是/否"显著。效应量告诉你"差多少"：

```
Cohen's h = 2 × arcsin(√p₁) - 2 × arcsin(√p₂)
```

解释：
- h=0.2：小效应
- h=0.5：中效应
- h=0.8：大效应

## 方法

**A/B 测试完整流程**：

1. **设计**：确定最小效应量、α、β → 算出所需样本量
2. **随机化**：确保用户被随机分配到两组
3. **运行**：收集数据直到达到预设样本量
4. **分析**：
   - 计算两组比例
   - 进行 z 检验，计算 p 值
   - 计算效应量的置信区间
5. **决策**：p < 0.05 → 拒绝 H₀，认为新算法有效
6. **报告**：给出差异的点估计和区间估计

## 验证标准

1. ✅ 当新旧算法无差异时，p>0.05（不误报——控制第一类错误）
2. ✅ 当新算法确实更好时，p<0.05（能检测出差异——有检验效力）
3. ✅ 样本量增大时检验效力提高
4. ✅ 效应量越大，相同样本量下检验效力越高

## 洞察

1. **p 值不是真相检测器**。p=0.04 并不意味着"有 96% 的概率有效"。它意味着"如果 H₀ 为真，看到这么极端的数据的概率是 4%"。

2. **统计显著 ≠ 实际显著**。当 n=100000 时，0.1% 的差异也可能 p<0.001。但 0.1% 的转化率提升值得你上线一个新版本吗？不一定。

3. **多重比较问题**。如果你同时测了 20 个指标，按 α=0.05 的标准，平均会有 1 个指标显示"显著"——即使所有指标都没有真实变化。**Bonferroni 校正**：α' = α/m（m 为比较次数）。

4. **提前停止是作弊**。每过一段时间看一下 p 值，等 p<0.05 就停止——这会让第一类错误率暴增到 30-50%。预先确定样本量，不到就不看结果。

5. **样本量计算很重要**。效应量越小，需要的样本量越大。把 CTR 从 5% 提到 5.3%（提升 6%）需要大约 42,000 个用户每组（80% power, α=0.05）。

## 延伸

- 如果数据不是比例（比如平均购买金额），用双样本 t 检验
- 如果样本量小，用 Fisher 精确检验代替 z 检验
- 贝叶斯 A/B 测试：直接给出"新算法更好的概率"
- 序贯检验：允许在测试过程中监控结果而不膨胀第一类错误
