# 🏆 毕业项目：库存决策的完整概率分析

> 把本课程学的所有技能串起来，解决一个真实的供应链决策问题。

---

## 项目场景

你是某电商公司的供应链分析师。公司经营一款热门商品，你需要帮运营团队制定**库存策略**。

### 已知数据

运营团队希望分析两类历史数据：

1. **日需求数据**：过去 500 天的每日销售量
2. **供应商补货时间**：两个潜在供应商过去各自的补货天数记录

为了让课程代码开箱即跑，`code/capstone.py` 会用纯 Python 标准库生成一份可复现的模拟数据。真实项目中，可以把这两组模拟数据替换为业务系统导出的 CSV。

你需要解决以下问题：

### 问题 1：需求分布识别（使用案例1的方法）

加载日需求数据，判断它服从什么分布。

- 计算基本统计量（均值、标准差、偏度、峰度）
- 绘制需求直方图和 QQ 图
- 判断：正态、对数正态还是指数分布？
- 根据识别结果，给出均值和方差的估计

### 问题 2：月总需求的分布（使用案例2的 CLT）

根据问题1识别的日需求分布，用 CLT 估计**月总需求（30 天）**的分布。

- 月总需求的均值是多少？
- 月总需求的标准差是多少？
- 月总需求近似服从什么分布？

### 问题 3：需求参数的不确定性（使用案例3的贝叶斯推断）

运营经理对日需求有一定的经验判断（先验）。将他的经验编码为先验分布，然后用观测数据更新。

- 设定一个合理的先验分布（如 Normal-Inverse-Gamma 共轭先验）
- 计算后验分布的均值和方差
- 对比：贝叶斯估计与 MLE 有什么区别？

### 问题 4：供应商选择（使用案例4的假设检验）

两个供应商 A 和 B 的补货时间数据已知。新供应商 A 声称比现有供应商 B 更快。

- 设定 H₀ 和 H₁
- 进行双样本 t 检验
- 给出结论：A 是否显著快于 B？

### 问题 5：安全库存与服务水平（使用案例5的蒙特卡洛）

结合以上分析，用蒙特卡洛仿真评估不同安全库存水平下的服务水平。

- 仿真参数：日需求分布（来自问题 1）、补货时间分布（来自最佳供应商）
- 连续订货策略：(s, Q) 策略——当库存低于 s 时，订货 Q 件
- 仿真 10000 天，评估：
  - 服务水平（不缺货的天数比例）
  - 平均库存水平
  - 缺货次数和缺货量
- 找到满足 95% 服务水平的最小安全库存 s

## 输出要求

提交一个完整的分析报告（`08-capstone.md` 可以是你最终报告的一部分），以及一个可运行的 Python 脚本（`code/capstone.py`）。

脚本应：
1. 使用纯 Python 标准库生成模拟数据（因为实际数据未提供）
2. 实现上述 5 个问题的完整分析
3. 输出清晰的文本结果
4. 注释充分，方便他人理解和复现

## 提示

- 时间紧张？先完成问题 1 和 5，这是最核心的
- 问题 3（贝叶斯）和问题 4（假设检验）可以相对独立地完成
- 所有分析需要在同一个脚本中完成，便于整体运行
- **不要追求完美分布拟合**——近似正态就足够了
- 最终的答案不只是一个数——而是一个包含了不确定性的决策建议
