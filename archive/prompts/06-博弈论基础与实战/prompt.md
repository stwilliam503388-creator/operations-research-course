# 博弈论基础与实战课程

> 一个周末，从零到能看懂博弈论的核心概念、把竞争/合作场景建模成博弈、以及用 Python 求解纳什均衡。

---

## 这份教程是什么

这是一份给**有基础数学和 Python 能力、但对博弈论完全陌生**的技术人/产品经理/策略分析师写的自学教程。它不讲学院派的定理证明——每一章都围绕「这个博弈模型描述了什么样的现实冲突」和「这个均衡概念告诉你该怎么决策」展开。

读完以后，你不一定会成为博弈论专家，但你一定会：

- 跟同事说清楚「纳什均衡、帕累托最优、占优策略」各自在说什么
- 看到一个商业竞争场景，判断是囚徒困境、协调博弈还是鹰鸽博弈
- 用 Python 求解小型博弈的纳什均衡，并做灵敏度分析
- 理解拍卖、谈判、机制设计的基本逻辑

# 第一部分：博弈论概览

> 在这一章，你不会看到一行代码。你会看到：博弈论是怎么从冯·诺依曼的书房里走进每一个拍卖行、每一场贸易谈判、每一次价格战的？一个竞争/合作场景怎么被拆解成「参与者 / 策略 / 收益」这三个要素？以及——博弈论和运筹学是什么关系？

---

## 1.1 没有博弈论之前，世界是什么样子的

先想象几个场景。

**场景 A：价格战**。你家楼下的两家奶茶店。如果都卖 15 元，各赚 5000/天。如果一家降到 12 元、另一家保持 15 元——降价的抢走大部分顾客赚 8000，不降价的只剩 2000。但如果都降到 12 元——两败俱伤，各赚 3000。结果你每天看到的是什么？两家都在降价。这就是囚徒困境——**每个人都选对自己最好的，结果对所有人都不好**。

**场景 B：拍卖**。你参加一块土地拍卖。你心里估价 500 万。叫价到 400 万的时候，别人还在加。你该跟到多少？出高了怕亏，出低了怕输。这不是一个孤立的决策问题——你的最优出价取决于**别人怎么出价**。

**场景 C：谈判**。你要和供应商谈一个年度合同。你知道对方的成本底线大概在 80 万左右，但你不确定。你说 70 万，对方说 100 万。你们来回拉锯。**每一方的出价策略取决于对对方底牌的猜测。**

这三个场景的共同点：**你的收益不仅取决于你怎么选，还取决于别人怎么选。**

这就是博弈论的研究对象——**多人决策互动**。

### 从冯·诺依曼到诺贝尔奖：一段简史

| 年份 | 人物 | 贡献 |
|------|------|------|
| 1928 | 冯·诺依曼 | 证明零和博弈的 minimax 定理 |
| 1944 | 冯·诺依曼 & 摩根斯坦 | 《博弈论与经济行为》——奠基之作 |
| 1950 | 纳什 | 纳什均衡——非合作博弈的核心概念 |
| 1960s | 塞尔滕 | 子博弈完美均衡——序贯博弈的精炼 |
| 1960s | 海萨尼 | 贝叶斯博弈——不完全信息下如何推理 |
| 1970s~ | 应用爆发 | 拍卖设计、匹配市场、机制设计获多个诺贝尔奖 |

## 1.2 博弈的三要素

所有博弈，拆开来看都是三件事：

```
参与者 (Player)  →  谁在做决策？
策略 (Strategy)  →  每个人有哪些选择？
收益 (Payoff)    →  每种结果下每个人得到什么？
```

**生活类比**：下围棋。
- 参与者 = 黑棋和白棋
- 策略 = 每一手的落子位置
- 收益 = 赢或输

## 1.4 博弈论与运筹学的关系

博弈论是运筹学的一个分支——在 OR 课程的第 1.2 节中，我们提到 OR 的三个基本问题之一是「决策问题：在不确定下怎么选」。博弈论处理的是**策略不确定**——不确定性来自其他理性参与者的选择。

| | 运筹学（通常） | 博弈论 |
|--|-------------|--------|
| 参与者数量 | 1 个决策者 | 2+ 个决策者 |
| 目标 | 优化自己的目标 | 在别人也优化的前提下优化自己 |
| 数学工具 | LP、MIP、DP | 均衡求解、不动点定理 |
| 不确定性来源 | 自然/概率 | 他人的策略 |

**一句话区别**：运筹学教你怎么做最好的决策。博弈论教你在别人也做最好决策的时候怎么做最好的决策。

## 1.5 建模心法：五步法

### 第一步：列参与者（30 秒）

「谁在决策？」列出所有人。别忘了「潜在进入者」、「政府监管机构」——凡是能影响你收益的人都是参与者。

### 第二步：列策略（1 分钟）

每个人有哪些选择？用大白话写：
- 奶茶店 A：{高价 15 元, 低价 12 元}
- 奶茶店 B：{高价 15 元, 低价 12 元}

### 第三步：写收益矩阵（2 分钟）

画一个表格：行是 A 的策略，列是 B 的策略，格子里写 (A的收益, B的收益)。

### 第四步：找均衡（30 秒）

有没有**占优策略**？如果没有，有没有**纳什均衡**？

### 第五步：检查可信度（30 秒）

如果是动态博弈——**后手的策略是否可信**？如果威胁不可能执行，均衡不成立。

# 第二部分：核心概念

> 不讲学院派的拓扑学证明。只讲「纳什均衡到底是什么、怎么找、什么时候它不唯一」。

---

## 2.1 纳什均衡：没人想单方面改变

### 直觉

纳什均衡是博弈理论的中心概念。它说的是一组策略组合，满足：

> **给定别人的选择，没有人可以通过单方面改变自己的选择得到更高的收益。**

**生活类比**：开车靠左还是靠右？在中国大家都靠右——如果有人单方面改成靠左，他会被撞飞。所以"所有人靠右"是一个纳什均衡（在靠右行驶的国家）。"所有人靠左"在靠右行驶的国家不是一个均衡——单方面改到靠右会更好。

### 找纳什均衡的下划线法

```
                奶茶店 B
              高价    低价
       高价  (5,5)  (2,8)
奶茶店 A
       低价  (8,2)  (3,3)
```

**步骤**：
1. 固定 B 的选择，看 A 的最优反应
   - B 选高价 → A 选低价更好（8 > 5）→ 在 8 下划横线
   - B 选低价 → A 选低价更好（3 > 2）→ 在 3 下划横线
2. 固定 A 的选择，看 B 的最优反应
   - A 选高价 → B 选低价更好（8 > 5）→ 在 8 下划横线
   - A 选低价 → B 选低价更好（3 > 2）→ 在 3 下划横线
3. 看哪些格子两边都被划了线 → (低价,低价) 是唯一的纳什均衡

---

## 2.2 占优策略

### 直觉

> **不管别人怎么选，我选这个总是最好的。**

如果存在占优策略，博弈很简单——你不需要猜别人怎么选。

**例子**：囚徒困境中的"坦白"——不管同伙选什么，坦白总是更好的。

| | 同伙沉默 | 同伙坦白 |
|--|---------|---------|
| 你沉默 | -1, -1 | -10, 0 |
| 你坦白 | 0, -10 | -5, -5 |

你的视角：同伙沉默时坦白（0 > -1），同伙坦白时坦白（-5 > -10）。**坦白是占优策略。**

---

## 2.3 混合策略：当纯策略不存在均衡时

有些博弈没有纯策略纳什均衡。比如**猜拳**——如果我一直出石头，你只要一直出布就赢了。所以唯一合理的策略是**随机化**：石头/剪刀/布各 1/3。

**混合策略** = 以一定概率随机选择纯策略。

### 找混合策略纳什均衡

以鹰鸽博弈为例：

| | 鸽 | 鹰 |
|--|----|----|
| 鸽 | 3,3 | 1,4 |
| 鹰 | 4,1 | 0,0 |

这只混合策略的直觉：如果我总是出鸽，你就总是出鹰（4>3）——所以我必须随机化来让你无法预测我。

均衡条件：**你的混合概率必须让对方"无差别"——他选任何纯策略的期望收益都一样。**

```

设 我出鹰的概率 = p，出鸽的概率 = 1-p

对方选鹰的期望收益 = p·0 + (1-p)·4 = 4 - 4p
对方选鸽的期望收益 = p·1 + (1-p)·3 = 3 - 2p

令两者相等：4 - 4p = 3 - 2p → 2p = 1 → p = 0.5

所以：我以 50% 概率出鹰、50% 概率出鸽时，
对方选鹰和选鸽的收益相同——她没有动机改变。
```

## 2.4 纳什均衡的存在性与多重性

### 存在性定理

纳什（1950）证明：**任何有限博弈（有限参与人、有限策略）至少存在一个纳什均衡**（可能包含混合策略）。

### 多重均衡问题

有些博弈有多个纳什均衡。例如**协调博弈**：

| | 去A店 | 去B店 |
|--|------|------|
| 去A店 | 2,2 | 0,0 |
| 去B店 | 0,0 | 1,1 |

(去A,去A) 和 (去B,去B) 都是纳什均衡。哪个会实际发生？**这就是多重均衡问题**——博弈论本身不告诉你会选哪个，你需要其他工具（聚点、文化、习俗、历史）来精炼。

### 精炼均衡的方法

| 精炼方法 | 直觉 | 适用场景 |
|---------|------|---------|
| 帕累托占优 | 选对所有人更好的那个 | 协调博弈 |
| 风险占优 | 选风险更低的那个 | 不确定时的选择 |
| 聚点 | 选文化/语言上突出的那个 | 日常协调 |
| 子博弈完美 | 剔除不可信的威胁 | 序贯博弈 |

## 2.5 序贯博弈与逆向归纳

当参与者不是同时出手，而是按顺序行动的时候——你需要在每一步考虑对方的理性反应。

### 逆向归纳

```
从博弈树的最后一步开始：
  每步都假设决策者选对自己最有利的
  然后向上推一层
```

**生活类比**：下棋时，高手不是走一步看一步——他们先想"如果我走到这里，对手会怎么应对"，然后再想自己的应对，以此类推。

### 可信威胁 vs 不可信威胁

```
你威胁："如果你降价，我就把价格降到成本以下，大家一起亏。"
但如果对方真的降价了，降到成本以下你也会亏——
这个威胁是「不可信的」，因为执行它对你也不利。
```

子博弈完美均衡（Subgame Perfect Equilibrium）正是在剔除这类不可信威胁——只有那些在每一步都理性的策略才是均衡。
