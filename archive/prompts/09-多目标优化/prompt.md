# 多目标优化基础课程

> 一个周末，从零到能看懂多目标优化在干什么、把多个冲突目标建模成多目标优化问题、以及使用 NSGA-II 等算法求解帕累托前沿。

---

## 这份教程是什么

这是一份给**有运筹学或单目标优化基础、但对多目标优化完全陌生**的技术人/数据科学家/工程师写的自学教程。它不讲学院派的定理证明——每一章都围绕「多个冲突目标怎么同时优化」和「帕累托最优到底告诉你什么」展开。

读完以后，你不一定会成为多目标优化专家，但你一定会：

- 跟同事说清楚「帕累托最优、帕累托前沿、非支配解」各自在说什么
- 看到一个多目标业务问题，判断用加权求和法、ε-约束法还是进化多目标算法
- 理解 NSGA-II 的核心流程：非支配排序 + 拥挤距离 + 锦标赛选择
- 用 Python 实现简单的多目标优化并绘制帕累托前沿
- 知道超体积（HV）、GD、IGD 是用来评价什么

## 多目标优化的基本矛盾

先记住一句话，贯穿整个课程：

> **你不可能在所有目标上都做到最好——多目标优化不是找一个「最优解」，而是找一组「最好的折衷方案」。**

这就是多目标优化的核心矛盾——**冲突目标的权衡**。它不是缺陷，是这门学科的灵魂。当你面对一个业务问题：
- 如果目标之间不冲突 → 其实是个单目标问题，合并即可
- 如果目标冲突但可以提前给权重 → 用加权求和法（先验偏好）
- 如果目标冲突且不知道权重怎么给 → 用 ε-约束法或多目标进化算法（后验偏好）
- 如果想和决策者交互迭代 → 用交互式方法

# 第一部分：多目标优化概览

> 在这一章，你不会看到一行代码。你会看到：为什么现实世界的问题几乎从来不是单目标的？多目标优化怎么从 1906 年的经济学论文走到今天的工业实践？以及——面对一堆冲突的目标，你该怎么思考？

---

## 1.1 为什么单目标不够

先想象几个场景。

**场景 A：买车**。你要买一辆新车。你考虑价格(越便宜越好)、性能(动力越强越好)、油耗(越低越好)、安全性(越高越好)。没有一辆车同时在所有维度上最好——便宜的往往性能弱，性能强的往往油耗高。你必须做出权衡。

**场景 B：投资**。你有 100 万要投资。你希望收益尽可能高，同时风险尽可能低。高收益资产通常伴随高风险。你想只买低风险资产？那收益就低。你想只买高收益资产？那风险就高。你必须折衷。

**场景 C：供应链**。你管理一家制造企业的供应链。你想要成本尽可能低，同时服务水平尽可能高（不缺货），同时碳排放尽可能少。缩减库存降低仓储成本→但缺货风险上升。增加本地供应商降低碳排放→但采购成本上升。三个目标互相拉扯。

这三个场景的共同点：**你有多个目标，它们互相冲突。没有一个方案能让所有目标同时达到最优。**

在工厂里我们经常遇到的情况：车间主任想要产量最大（成本不变时利润最大）、品控经理想要次品率最低（这会拖慢产量）、销售总监想要交付时间最短（需要维持高库存多占资金）。三个人的 KPI 都是 KPI——谁能说哪个更重要？

> 单目标优化教你的：在一条线上找到最高点。
> 多目标优化教你的：在一群方案中找到那条「不能再好了」的曲线（帕累托前沿）。

## 1.2 多目标优化的核心矛盾

**单目标优化问**：哪个解最好？
**多目标优化问**：哪些解是「好的折衷」？

### 一个简单的例子

```
你设计一款无人机：
  - 目标1：最大化续航时间（分钟）
  - 目标2：最小化制造成本（元）
  
方案A：续航30分钟，成本2000元
方案B：续航45分钟，成本3500元
方案C：续航20分钟，成本1800元
方案D：续航40分钟，成本4000元
```

谁更好？
- A vs C：A 续航更长，成本稍高——没有解能说 A 绝对比 C 好（因为成本高了）
- A vs B：A 成本更低但 B 续航更长——没有绝对优劣
- B vs D：B 续航更长且成本更低！→ B 支配 D → D 是支配解，可以被淘汰

这里的核心矛盾是：**续航时间和制造成本是冲突目标**。你不能同时得到最长的续航和最低的成本。多目标优化的目标就是找出所有「非支配解」，让决策者看这份权衡清单来做最终选择。

---

## 1.3 三种处理策略

根据你和决策者的偏好信息在什么时候引入，多目标优化方法分为三大类：

### 1. 先验偏好方法（先给权重，再求解）

你在求解之前就决定了各个目标的相对重要性。

| 方法 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| 加权求和法 | 给每个目标赋权重，合并成单目标 | 简单、快 | 权重难定、非凸前沿找不到 |
| 字典序法 | 按重要性排序，依次优化 | 明确优先级 | 极端的偏好表达 |
| 目标规划 | 设理想值，最小化偏离 | 直观 | 理想值设定影响结果 |

**最适合**：决策者非常清楚自己的偏好，而且偏好稳定。

### 2. 后验偏好方法（先求整个前沿，再选）

你先找出一堆好的折衷方案（帕累托前沿），再拿给决策者挑。

| 方法 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| ε-约束法 | 一个目标作目标，其余转约束 | 可找非凸前沿 | 计算量大（需要调ε参数）|
| NSGA-II | 进化算法生成帕累托前沿 | 一次跑出整个前沿 | 不能保证最优性 |
| MOEA/D | 分解成多个单目标子问题 | 形态可控 | 参数敏感 |

**最适合**：决策者说不清偏好，但看到方案就知道要哪个。

### 3. 交互式方法（边求解边问偏好）

你在求解过程中反复问决策者，逐步缩小搜索范围。

| 方法 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| 参考点法 | 决策者指定偏好点，算法在附近搜索 | 逐步收敛 | 需要决策者多次参与 |
| 逐步折衷法 | 每次展示几个候选，决策者调整 | 灵活 | 耗时长 |

**最适合**：决策者有时间参与、但不知道事先偏好。

## 1.4 数学模型：多目标优化问题的标准形式

所有多目标优化问题都可以写成：

```
min  F(x) = [f₁(x), f₂(x), ..., fₘ(x)]ᵀ
s.t. gⱼ(x) ≤ 0,  j = 1,...,J
     hₖ(x) = 0,   k = 1,...,K
     x ∈ Ω
```

其中：
- **x**：决策变量向量
- **F(x)**：m 个目标函数组成的向量
- **gⱼ(x) ≤ 0**：不等式约束
- **hₖ(x) = 0**：等式约束
- **Ω**：决策空间（如 x 的上下界）

**注意**：最大化问题可以取负号转化为最小化，所有 F(x) 都默认求最小。

> 决策空间与目标空间：x 在决策空间（Ω），F(x) 在目标空间（ℝᵐ）。帕累托前沿是在目标空间里看的——你最后展示给决策者的图，横纵坐标是目标值，不是决策变量值。

## 1.5 建模心法：五步法

### 第一步：列目标清单（2 分钟）

用大白话列出业务上关心的所有指标。不要过滤——全部列出来。然后再看哪些是冗余的（相关性高）、哪些是不冲突的（可以合并）、哪些是真正冲突的。

### 第二步：识别冲突（1 分钟）

目标之间真的冲突吗？如果调整一个目标值会导致另一个目标值朝反方向变——那就是冲突。如果不冲突——直接合并成加权单目标。

### 第三步：确定策略（1 分钟）

问决策者三个问题：
1. 「你能提前告诉我哪个目标更重要吗？」→ 先验偏好
2. 「你不想提前决定，但看到方案能挑？」→ 后验偏好
3. 「你有空和我反复讨论调整？」→ 交互式

### 第四步：归一化（1 分钟）

不同目标的量纲和尺度不同。如果价格是万元级、碳排放是吨级、满意度是 0~1——不归一化没法比较。常用方法：min-max 归一化或目标值除以理想点值。

### 第五步：选方法写模型（30 秒）

根据策略和问题特点选具体方法（02-methods 会详细讲），然后写模型。

## 2.1 帕累托支配关系

### 直觉

> 解 A 支配解 B ⇔ A 在所有目标上不比 B 差，且至少在一个目标上比 B 好。

**生活类比**：你和朋友各自找到了一款电脑。

```
你的电脑：  5000元, 16GB内存, 512GB硬盘
朋友的电脑：6000元, 8GB内存,  512GB硬盘
```

你的电脑更便宜（更好）、内存更大（更好）、硬盘一样——你的电脑**支配**朋友的电脑。朋友不会考虑他那台了。

```
你的电脑：  5000元, 16GB内存, 512GB硬盘
朋友的电脑：4500元,  8GB内存, 1TB硬盘
```

你的更贵但内存大，朋友的更便宜但硬盘大——**互不支配**。这两个都是候选方案。

### 形式定义

对于最小化问题，解 A 支配解 B（记为 A ≺ B）当且仅当：

```
∀i ∈ {1,...,m}: fᵢ(A) ≤ fᵢ(B)   (A 在所有目标上不差于 B)
且 ∃j ∈ {1,...,m}: fⱼ(A) < fⱼ(B)   (至少一个目标严格更好)
```

三个类别：
- **强支配**：A 在所有目标上严格比 B 好
- **弱支配**：A 在所有目标上不差于 B（至少一个相等）
- **互不支配**：A 在某些目标上更好，B 在另一些上更好

---

## 2.2 加权求和法

### 直觉

> 给每个目标一个权重，把它们加起来变成一个目标——然后你就回到了熟悉的单目标优化世界。

```
min  F(x) = w₁·f₁(x) + w₂·f₂(x) + ... + wₘ·fₘ(x)
s.t. 原始约束
      Σ wᵢ = 1, wᵢ ≥ 0
```

**生活类比**：你要买手机，你内心觉得「性能对我来说占 60%，拍照占 30%，外观占 10%」——你给每个维度打了权重，然后算总分。

### 优点

- 极其简单——任何单目标求解器都可以用
- 计算效率高——跑一次得到一个解
- 直观易解释——「你的偏好权重是多少」

### 陷阱：非凸前沿

加权求和法有一个致命的陷阱：**它找不到非凸区域的帕累托最优解**。

```
        │
        │    ╱╲
        │   ╱  ╲        ← 帕累托前沿
        │  ╱    ╲
        │ ╱      ╲
        │╱        ╲
        └─────────────────
            f₁
```

加权求和等价于用一条直线（权重法线）去切可行域，找切点。如果前沿是非凸的——直线切不到凹进去的部分，那些解永远得不到。

**你的权重不可能覆盖所有非支配解，无论你怎么调权重。**

### 什么时候用

| 适用场景 | 不适用场景 |
|---------|-----------|
| 你清楚权重 | 你不知道权重 |
| 前沿近似凸 | 前沿可能是非凸的 |
| 你只需要一个解（不是整个前沿） | 你需要整个帕累托前沿 |
| 快速原型验证 | 正式决策支持 |

---

## 2.3 ε-约束法

### 直觉

> 选一个目标作为目标函数，把所有其他目标转为约束——给它们设上限 εᵢ。

```
min  f₁(x)
s.t. f₂(x) ≤ ε₂
     f₃(x) ≤ ε₃
     ...
     fₘ(x) ≤ εₘ
     原始约束
```

**生活类比**：你买房子。「最看重价格」——先把价格最小化；但增加了约束「面积 ≥ 100㎡」、「离地铁 ≤ 500m」、「房龄 ≤ 10年」。这些约束是你对其他目标的最低要求。

### 怎么得到整个前沿

通过改变 ε₂, ε₃, ..., εₘ 的取值，反复求解，得到不同的最优解——这些解就构成了帕累托前沿的近似。

```
for ε₂ in [min_f₂, max_f₂]:
    for ε₃ in [min_f₃, max_f₃]:
        求解 min f₁ s.t. f₂≤ε₂, f₃≤ε₃...
```

### 优点 vs 缺点

| 优点 | 缺点 |
|------|------|
| 可以找到非凸前沿（克服加权求和的致命缺陷）| 计算量大——每个 ε 组合都要跑一次优化 |
| 每个解有明确含义——「在 f₂ 不超过 ε₂ 时 f₁ 的最小值」| ε 的取值范围需要知道各目标的 min/max |
| 约束式表达符合人类直觉 | 多个 ε 同时变时，组合爆炸 |

### 什么时候用

| 适用场景 | 不适用场景 |
|---------|-----------|
| 目标数量少（2~3 个）| 目标数量多（4+ 个）|
| 你能给其他目标设定合理的 ε 值 | ε 值完全不知道从哪设 |
| 需要展示不同约束水平下的表现 | 需要一次跑出整个前沿（太慢）|

---

## 2.4 目标规划

### 直觉

> 给每个目标设一个理想值（目标值），然后最小化所有偏离的加权和（或最大偏离）。

```
min  Σ wᵢ · |fᵢ(x) - Tᵢ|   （加权和型）
或  min  max { wᵢ · |fᵢ(x) - Tᵢ| }  （极小化最大型）
s.t. 原始约束
```

其中 Tᵢ 是目标 i 的理想值（target）。

**生活类比**：你去买车，心里有三个目标：「价格 15 万以内」、「百公里油耗 6L 以下」、「后备箱至少 500L」。没有一辆车全满足——你选一辆偏离最小的。

### 两种常见形式

| 形式 | 数学 | 适用场景 |
|------|------|---------|
| 加权和型 | min Σ wᵢ·dᵢ⁺ + wᵢ·dᵢ⁻ | 希望平均偏离小 |
| 极小化最大型（切比雪夫型）| min max(wᵢ·dᵢ) | 希望最差的那个目标也不至于太差 |

其中 dᵢ⁺ = max(0, fᵢ - Tᵢ)（超过目标的偏离），dᵢ⁻ = max(0, Tᵢ - fᵢ)（不足的偏离）。

### 什么时候用

| 适用场景 | 不适用场景 |
|---------|-----------|
| 决策者能给出明确的理想值 | 理想值完全没概念 |
| 目标有明确的「满意水平」| 目标是最大化（没有上限）|
| 需要惩罚不同方向的偏离 | 真正的帕累托前沿才是你需要的 |

## 2.5 多目标进化算法（NSGA-II）

### 为什么需要进化算法

加权求和法和ε-约束法每次求解只得到一个解。要得到整个帕累托前沿，需要反复求解。当问题非线性、非凸、非连续时，反复求解代价极高。进化算法可以**一次运行生成整个帕累托前沿**。

### NSGA-II 核心流程

```
初始化 → 非支配排序 → 拥挤距离 → 锦标赛选择 → 交叉变异 → 合并种群 → ... → 终止
```

#### Step 1：非支配排序（Non-dominated Sorting）

把种群中的解分成多个前沿层级（Front）：

```
前沿 1 (Front 1)：不被任何其他解支配的解
前沿 2 (Front 2)：只被前沿 1 中的解支配
前沿 3 (Front 3)：只被前沿 1 或 2 中的解支配
...
```

**直觉**：这就像剥洋葱——最外层是"最好的"（不被任何人支配），剥掉之后下一层出现。

#### Step 2：拥挤距离（Crowding Distance）

同一前沿内的解，越稀疏越好——我们要保留多样性，防止扎堆。

```
拥挤距离 = 解在每个目标方向上与相邻解的距离之和
```

**直觉**：在帕累托前沿上，一个解的两边邻居离它越远，它的拥挤距离越大——说明它处在"人少"的区域，值得保留。

#### Step 3：锦标赛选择 + 精英保留

选择标准：
1. 前沿层级越低越好（优先选 Front 1 的）
2. 同一层级内，拥挤距离越大越好（优先选稀疏区域的）

**精英保留**：父代和子代合并后，按 1-2 标准选 N 个进入下一代——保证不丢失最优解。

### 伪代码

```
P = 随机初始种群(N)
对每一代 t = 1, 2, ..., T:
    Q = 从 P 通过选择+交叉+变异生成子代(N)
    R = P ∪ Q                    # 合并 2N
    fronts = 非支配排序(R)        # 分成多个前沿
    P_new = []
    for front in fronts:
        if len(P_new) + len(front) ≤ N:
            P_new ∪= front       # 全部加入
        else:
            计算拥挤距离(front)
            按拥挤距离降序选满剩余名额
    P = P_new
输出 P 中所有 Front 1 的解 = 帕累托前沿
```

### NSGA-II 的优点

| 优点 | 说明 |
|------|------|
| 一次运行得到整个前沿 | 不需要反复求解 |
| 不需要提前设权重 | 适合后验偏好 |
| 全局搜索能力 | 能跳出局部最优（交叉变异的随机性）|
| 拥挤距离保证多样性 | 前沿上的解均匀分布 |
| 精英保留 | 不丢失已找到的优解 |

### 局限性

| 局限 | 说明 |
|------|------|
| 不保证全局最优 | 进化算法是启发式 |
| 参数敏感 | 种群大小、交叉率、变异率需要调参 |
| 高维目标退化 | 3+ 个目标时，非支配比例爆炸，选择压力变弱 |
| 需要多次运行 | 随机性导致每次结果不同 |

## 2.6 性能指标

你跑出了一个帕累托前沿。它好还是不好？需要三个维度的评价：

### 1. 收敛性（Convergence）——离真正的帕累托前沿有多近

| 指标 | 定义 | 越小越好的方向 |
|------|------|---------------|
| GD (Generational Distance) | 找到的解到真实前沿的平均距离 | ↓ 越小越好 |
| IGD (Inverted GD) | 真实前沿上的点到找到的前沿的平均距离 | ↓ 越小越好 |
| ER (Error Ratio) | 找到的解中不在真实前沿上的比例 | ↓ 越小越好 |

### 2. 多样性（Diversity）——解在前沿上分布得多均匀、多广

| 指标 | 定义 | 方向 |
|------|------|------|
| Δ (Spread) | 相邻解之间的最大距离偏差 | ↓ 越小越好 |
| 前沿宽度 | 各目标方向上的范围 | ↑ 越宽越好（但要在前沿上）|
| 拥挤距离平均值 | 同一前沿内解的平均间距 | 适中为好 |

### 3. 综合指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 超体积 (Hypervolume, HV) | 帕累托前沿和参考点围成的超立方体体积 | ↑ 越大越好——同时衡量收敛性和多样性 |

**HV 为什么好**：它不要求你知道真实帕累托前沿（GD/IGD 需要参考真实前沿）。你只需要设定一个参考点（通常是各目标最差值的组合）。HV 越大→前沿越靠近理想点且覆盖越广。不过计算 HV 的复杂度随目标数指数增长。

## 案例 1：投资组合双目标优化 ★★☆☆☆

**模型**：双目标二次规划
**方法**：加权求和法 + ε-约束法
**目标**：收益（最大化）vs 风险（最小化）

### 1 场景

你有 3 种资产可以投资。每种资产有历史收益率和协方差矩阵。你需要在收益和风险之间找平衡。这是 Markowitz 均值-方差模型的经典多目标版本。

### 2 业务→多目标模型映射表

| 业务概念 | 模型对应 |
|---------|---------|
| 每种资产的配置比例 | 决策变量 xᵢ ∈ [0,1]，Σxᵢ = 1 |
| 期望收益 | 目标1：f₁ = -μᵀx（最大化→转最小化）|
| 投资风险（方差）| 目标2：f₂ = xᵀΣx |
| 收益必须高于最低要求 | 约束：μᵀx ≥ r_min |
| 不做空 | 约束：xᵢ ≥ 0 |

### 3 数学模型

```
决策变量: x ∈ ℝⁿ,  ∑xᵢ = 1, xᵢ ≥ 0

目标1（最小化负收益）: min  f₁(x) = -μᵀx
目标2（最小化风险）:   min  f₂(x) = xᵀΣx

加权求和法:
  min  w₁·(-μᵀx) + w₂·(xᵀΣx)
  s.t. ∑xᵢ = 1, xᵢ ≥ 0
  改变 w = [(1,0), (0.8,0.2), ..., (0,1)]

ε-约束法:
  min  -μᵀx   (或 min xᵀΣx)
  s.t. xᵀΣx ≤ ε  (或 -μᵀx ≤ ε)
       ∑xᵢ = 1, xᵢ ≥ 0
  改变 ε 值得到不同解
```

### 4 求解方法选择

| 方法 | 理由 | 注意事项 |
|------|------|---------|
| 加权求和法 | 前沿是凸的（二次规划），加权求和有效 | 权重等间隔取样，至少 20 个点 |
| ε-约束法 | 对比加权求和，教学生理解「约束法能处理非凸」| 用 f₁ 做目标，f₂ 做约束 |

### 5 代码实现概览

```python
import numpy as np
from scipy.optimize import minimize

# 加权求和法
def weighted_sum(mu, Sigma, w, n):
    def objective(x):
        return w[0] * (-mu @ x) + w[1] * (x @ Sigma @ x)
    # 约束：sum(x)=1, x>=0
    cons = ({'type': 'eq', 'fun': lambda x: np.sum(x) - 1})
    bounds = [(0, 1)] * n
    res = minimize(objective, x0=np.ones(n)/n, 
                   constraints=cons, bounds=bounds)
    return res.x

# 遍历不同权重
weights = [(w, 1-w) for w in np.linspace(0, 1, 21)]
frontier = [weighted_sum(mu, Sigma, w) for w in weights]
```

### 6 运行结果

```
权重 w=(1.0, 0.0): 收益=12.5%, 风险=15.2%  (纯收益型)
权重 w=(0.5, 0.5): 收益=8.3%,  风险=7.1%   (平衡型)
权重 w=(0.0, 1.0): 收益=3.1%,  风险=2.3%   (纯保守型)
```

绘制帕累托前沿图：x 轴为风险（标准差），y 轴为收益——一条单调递增的凸曲线。

### ✅ 验证标准

1. 收益最高解 = 全部投入收益最高的资产（确认极端情况正确）
2. 风险最低解 = 全部投入风险最低的资产
3. 两条曲线（加权求和 vs ε-约束）在凸前沿上重合
4. 前沿单调递增：风险越高，收益越高

### 7 关键洞察

- 加权求和法在这里完美工作——因为收益-风险前沿通常是凸的
- 「最小方差组合」是前沿最左端的点——市场上说的高性价比组合
- 如果加入实际约束（如单只资产上限 30%），前沿会向内收缩

### 8 延伸与变形

- 加入 CVaR 替代方差作为风险度量 → 前沿变成非凸 → 加权求和失效
- 加入交易成本 → 整数变量（买入/卖出）→ MIP
- 多期投资 → 动态优化

## 案例 2：产品设计权衡 ★★★☆☆

**模型**：双目标/三目标优化
**方法**：ε-约束法 + 帕累托前沿可视化
**目标**：成本（最小化）vs 性能（最大化）vs 重量（最小化）

### 1 场景

你设计一款工业零配件。三个主要设计参数：材料强度等级（连续变量x₁）、厚度（x₂）、制造精度（x₃）。它们共同决定了三个目标：成本、性能和重量。成本与材料等级和精度正相关，性能与厚度和材料正相关，重量与厚度负相关。

### 2 业务→多目标模型映射表

| 业务概念 | 模型对应 |
|---------|---------|
| 材料等级、厚度、精度 | 决策变量 x₁, x₂, x₃ |
| 生产成本 | 目标1：f₁(x) = a₁x₁ + a₂x₂ + a₃x₃² |
| 性能评分 | 目标2：f₂(x) = - (b₁x₁ + b₂/x₂ + b₃x₃) → 最小化负性能 |
| 部件重量 | 目标3：f₃(x) = c₁/x₁ + c₂x₂ + c₃x₃ |
| 材料等级上限 | 约束：x₁ ≤ max_level |
| 厚度工艺限制 | 约束：x₂ ∈ [min_thick, max_thick] |

### 3 数学模型（双目标版本：成本 vs 性能）

```
决策变量: x₁, x₂, x₃

min  f₁(x) = 100x₁ + 50x₂ + 200x₃²      (成本)
min  f₂(x) = - (80x₁ + 120/x₂ + 40x₃)    (负性能)
s.t. x₁ ∈ [1, 5], x₂ ∈ [0.5, 3.0], x₃ ∈ [0.1, 1.0]

ε-约束法（f₁为目标，f₂为约束）:
  min  100x₁ + 50x₂ + 200x₃²
  s.t. 80x₁ + 120/x₂ + 40x₃ ≥ ε₂   (性能不低于 ε₂)
       x₁ ∈ [1,5], x₂ ∈ [0.5,3.0], x₃ ∈ [0.1,1.0]
```

### 4 求解方法选择

| 方法 | 理由 | 注意事项 |
|------|------|---------|
| ε-约束法 | 前沿可能非凸，确保能找到所有解 | ε₂ 从性能范围均匀取 30 个值 |
| 加权求和法 | 对比用，展示非凸前沿时的失效 | 仅双目标时使用 |

### ✅ 验证标准

1. ε 越严格（性能要求越高），成本越高——趋势正确
2. 用加权求和法跑相同的例——如果前沿有非凸部分，得不到那些解
3. 三目标时绘制 3D 散点图检查帕累托前沿曲面

### 7 关键洞察

- 制造精度的二次项（x₃²）导致前沿部分非凸——ε-约束法能抓到而加权求和不能
- 「成本-性能-重量」的铁三角——更轻更便宜更高性能不可能同时达到
- 可视化三目标帕累托前沿时，建议用颜色编码第三个目标，散点图在前两个目标平面上

### 8 延伸与变形

- 加入可靠性目标（第4个目标）→ 高维可视化问题
- 用 NSGA-II 替代 ε-约束法，一次跑出整个前沿
- 设计参数为离散值（如材料只能选 3 种等级）→ 混合整数多目标

## 案例 3：生产调度多目标 ★★★☆☆

**模型**：三目标调度优化
**方法**：加权求和法 + 多目标进化算法思路
**目标**：完工时间（最小化）vs 生产成本（最小化）vs 能耗（最小化）

### 1 场景

你有 5 个订单在 2 台机器上加工。每个订单有加工时间、每小时的机器运行成本，以及不同速度档位对应的能耗。你可以选择加工速度（正常档 N / 高速档 H）——高速更快但更耗能更贵。

### 2 业务→多目标模型映射表

| 业务概念 | 模型对应 |
|---------|---------|
| 每个订单在哪个机器、哪个时间加工 | 决策变量（调度分配）|
| 最后一个订单完成的时间 | 目标1：f₁ = makespan = max Cⱼ |
| 总生产成本 | 目标2：f₂ = Σ(运行时间 × 单位成本) |
| 总能耗 | 目标3：f₃ = Σ(运行时间 × 档位能耗) |
| 同一时间一台机器只能加工一个订单 | 约束：无重叠 |
| 订单不可中断 | 约束：一旦开始必须做完 |

### 3 数学模型（简化版）

```
决策变量:
  x_{j,m} = 1 如果订单 j 在机器 m 上加工
  s_j = 订单 j 的开始时间
  v_j ∈ {N, H} 加工速度档位

目标:
  min  f₁ = makespan = max(s_j + t_j(v_j))
  min  f₂ = Σ s_j × c_m + Σ t_j(v_j) × c_speed(v_j)
  min  f₃ = Σ t_j(v_j) × e_speed(v_j)

约束:
  订单不能重叠: s_j + t_j(v_j) ≤ s_k 或 s_k + t_k(v_k) ≤ s_j
               (对同一机器上的任意 j≠k)
  速度选择: v_j ∈ {N, H}
```

### 4 求解方法选择

| 方法 | 理由 | 注意事项 |
|------|------|---------|
| 加权求和（离散优化）| 调度问题 NP-hard，加权求和使用单目标 MILP 求解器 | 权重多组，需要求解器 |
| NSGA-II（示意）| 展示进化算法如何处理离散+连续混合变量 | 编码用排列+速度向量 |

### ✅ 验证标准

1. 所有订单都被分配且时间不重叠
2. 用高速档时 makespan 下降但成本和能耗上升
3. 三目标前沿展示：三个目标两两冲突

### 7 关键洞察

- 生产调度中的三目标是典型的「不可能三角」：快+便宜+绿色
- 高速档在加权求和中是否选中的 threshold 非常敏感——权重微小变化导致解剧烈跳跃
- 帕累托前沿上的调度方案差异巨大——从「最慢最省」到「最快但最贵」

### 8 延伸与变形

- 加入机器故障 → 鲁棒调度
- 加入订单优先级 → 加权形式的目标函数
- 加入碳排放限额 → 约束法

## 案例 4：供应链多目标 ★★★★☆

**模型**：三目标供应链网络设计
**方法**：ε-约束法 + NSGA-II 对比
**目标**：总成本（最小化）vs 服务水平（最大化）vs 碳排放（最小化）

### 1 场景

你管理一个三级供应链：3 个供应商 → 2 个仓库 → 5 个零售区域。每个供应商有产能和单位成本。仓库有固定运营成本和容量。你需要决定：从哪个供应商采购多少、每个仓库备多少库存、配送路线。三个目标互相冲突。

### 2 业务→多目标模型映射表

| 业务概念 | 模型对应 |
|---------|---------|
| 从供应商 s 到仓库 w 的运量 | 决策变量 x_{s,w} ≥ 0 |
| 仓库 w 的库存水平 | 决策变量 y_w ≥ 0 |
| 总成本（采购+运输+仓储+缺货惩罚）| 目标1：f₁ = 总成本 |
| 服务水平 | 目标2：f₂ = 1 - 缺货率（最大化→最小化 1 - 服务率）|
| 碳排放 | 目标3：f₃ = Σ(运量 × 单位碳排放) + 仓储能耗碳排放 |
| 供应商产能 | 约束：Σ x_{s,w} ≤ cap_s |
| 仓库容量 | 约束：y_w ≤ cap_w |
| 需求满足 | 约束：Σ (配送量) ≥ demand_r |

### 3 数学模型（简化）

```
决策变量: x_{s,w} (运量), y_w (库存), z_{w,r} (配送量)

目标:
  min  f₁ = Σ c_{s,w}·x_{s,w} + Σ fixed_w·I(y_w>0) + Σ pen·max(0, d_r - Σ z_{w,r})
  min  f₂ = 1 - (Σ 满足需求 / Σ 总需求)      (1 - 服务率)
  min  f₃ = Σ e_{s,w}·x_{s,w} + Σ e_w·y_w   (碳排放)

约束:
  Σ x_{s,w} ≤ cap_s                  ∀s
  Σ z_{w,r} = y_w                    ∀w (库存等于配送总和)
  Σ z_{w,r} ≥ demand_r · service_min  ∀r
  x_{s,w}, y_w, z_{w,r} ≥ 0
```

### 4 求解方法选择

| 方法 | 理由 | 注意事项 |
|------|------|---------|
| ε-约束法 | 单目标 MILP（固定碳排放上限，最小化成本+最大化服务）| 两次取 ε 网格 |
| NSGA-II | 对比用，展示同时考虑三目标的效果 | 染色体编码 = 所有 x_{s,w} 和 y_w |

### ✅ 验证标准

1. 碳排放约束越紧 → 成本上升 → 前沿向右上移动（合理）
2. 服务水平趋近 100% → 需要更多库存 → 成本上升 → 碳排放上升
3. NSGA-II 得到的帕累托前沿应覆盖 ε-约束法得到的解（或更好）

### 7 关键洞察

- 供应链三目标的权衡是「可持续性三角形」的经典案例
- 碳排放约束在 0~20% 范围内边际成本低——简单措施就能显著减碳
- 超过 40% 减碳时，成本急剧上升——需要供应链结构重塑

### 8 延伸与变形

- 加入供应商选择（0/1 变量）→ 混合整数多目标
- 需求不确定 → 随机多目标优化
- 多周期 → 动态多目标优化

## 案例 5：NSGA-II 实战 ★★★★☆ [进阶]

**模型**：双目标标准测试函数（ZDT1 / ZDT2 / ZDT3）
**方法**：从零实现 NSGA-II
**目标**：教学演示——理解非支配排序和拥挤距离的核心流程

### 1 场景

你将从零实现 NSGA-II，在经典多目标测试函数上验证，并绘制帕累托前沿。

### 2 三个测试函数

```
ZDT1（凸前沿）:
  f₁(x) = x₁
  f₂(x) = g(x) · [1 - √(f₁/g(x))]
  g(x) = 1 + 9/(n-1)·Σ_{i=2}^{n} xᵢ
  xᵢ ∈ [0,1], n=30

ZDT2（非凸前沿）:
  f₁(x) = x₁
  f₂(x) = g(x) · [1 - (f₁/g(x))²]
  g(x) 同上

ZDT3（不连续的前沿）:
  f₁(x) = x₁
  f₂(x) = g(x) · [1 - √(f₁/g(x)) - (f₁/g(x))·sin(10πf₁)]
  g(x) 同上
```

### 3 核心实现模块

```python
import numpy as np

def non_dominated_sorting(pop):
    """非支配排序：返回每个个体的前沿层级"""
    n = len(pop)
    # 计算每个个体被谁支配、支配谁
    S = [[] for _ in range(n)]   # 被 i 支配的集合
    n_count = [0]*n               # 支配 i 的个数
    fronts = [[]]                 # 第一层
    
    for p in range(n):
        for q in range(n):
            if dominates(pop[p], pop[q]):
                S[p].append(q)
            elif dominates(pop[q], pop[p]):
                n_count[p] += 1
        if n_count[p] == 0:
            fronts[0].append(p)
    
    # 逐层剥洋葱
    i = 0
    while fronts[i]:
        next_front = []
        for p in fronts[i]:
            for q in S[p]:
                n_count[q] -= 1
                if n_count[q] == 0:
                    next_front.append(q)
        i += 1
        fronts.append(next_front)
    return fronts[:-1]

def crowding_distance(front, objs):
    """计算前沿内个体的拥挤距离"""
    n = len(front)
    m = objs.shape[1]
    dist = np.zeros(n)
    
    for obj_idx in range(m):
        # 按目标值排序
        idx = np.argsort([objs[i, obj_idx] for i in front])
        dist[idx[0]] = dist[idx[-1]] = np.inf  # 边界点
        f_range = objs[idx[-1], obj_idx] - objs[idx[0], obj_idx]
        for i in range(1, n-1):
            dist[idx[i]] += (objs[idx[i+1], obj_idx] - 
                            objs[idx[i-1], obj_idx]) / f_range
    return dist

def tournament_selection(pop, fronts, distances, k=2):
    """锦标赛选择：先比前沿层级，再比拥挤距离"""
    # 随机选 k 个，按 rank 小 > 距离大 选
    selected = []
    for _ in range(len(pop)):
        candidates = np.random.choice(len(pop), k, replace=False)
        best = candidates[0]
        for c in candidates[1:]:
            if (fronts[c] < fronts[best] or 
                (fronts[c] == fronts[best] and distances[c] > distances[best])):
                best = c
        selected.append(best)
    return selected

def nsga2(n_var, n_pop, n_gen, test_func='ZDT1'):
    # 初始化
    pop = np.random.rand(n_pop, n_var)
    
    for gen in range(n_gen):
        # 计算目标值
        objs = np.array([test_func(ind) for ind in pop])
        
        # 非支配排序
        fronts = non_dominated_sorting(objs.tolist())
        
        # 计算拥挤距离
        ranks = np.zeros(n_pop)
        distances = np.zeros(n_pop)
        for i, front in enumerate(fronts):
            for idx in front:
                ranks[idx] = i
            dist = crowding_distance(front, objs)
            for j, idx in enumerate(front):
                distances[idx] = dist[j]
        
        # 选择 → 交叉 → 变异（SBX + 多项式变异）
        parent_indices = tournament_selection(pop, ranks, distances)
        parents = pop[parent_indices]
        offspring = sbx_crossover(parents)      # 模拟二进制交叉
        offspring = polynomial_mutation(offspring)  # 多项式变异
        
        # 合并 + 环境选择
        combined = np.vstack([pop, offspring])
        # ...（非支配排序+拥挤距离选择前 N 个）
    
    return pop[fronts[0]]  # 返回帕累托前沿
```

### 4 关键参数

| 参数 | 建议值 | 说明 |
|------|-------|------|
| 种群大小 | 100 | 经典值 |
| 代数 | 250 | ZDT 系列一般 200~300 代收敛 |
| 交叉率 | 0.9 | SBX 交叉概率 |
| 变异率 | 1/n_var | 多项式变异概率 |
| 交叉分布指数 | 20 | SBX 分布的 η |
| 变异分布指数 | 20 | 多项式变异分布的 η |

### ✅ 验证标准

1. ZDT1：找到的前沿是凸曲线，GD < 0.01
2. ZDT2：找到的前沿是非凸曲线（凹的），GD < 0.01
3. ZDT3：找到的前沿是不连续的（分为 5 段），验证算法能处理不连续前沿
4. 种群多样性：拥挤距离平均值 > 0.01（没有扎堆）
5. 重复运行 10 次：每次的 HV 波动 < 5%

### 7 关键洞察

- 非支配排序的核心复杂度是 O(M·N²)——种群越大越慢，但这是值得的
- 拥挤距离是 NSGA-II 的「秘密武器」——没有它，所有解会扎堆在帕累托前沿的角落
- ZDT3 的不连续前沿是测试算法多样性维持能力的关键——解必须分布到多段
- 边界点（拥挤距离 = ∞）保证算法不会丢失前沿的端点

### 8 延伸与变形

- 约束支配法（Constrained-Domination）：加入约束处理
- 参考点法（NSGA-III）：处理 3+ 目标的退化问题
- 自适应变异率：根据种群多样性调整变异强度

### 项目设计

**主题：三目标城市物流优化**

你是一家同城物流公司的数据科学家。公司有 50 辆电动车，每天要配送 200 个订单。你需要设计一个多目标优化方案，同时优化三个目标：

| 目标 | 说明 |
|------|------|
| 总配送成本 | 行驶里程 × 每公里电费 + 司机工时费（最小化）|
| 准时率 | 在约定时间窗内送达的订单比例（最大化 → 最小化负比例）|
| 碳排放 | 总行驶里程 × 每公里碳排放（最小化）|

**约束**：
- 每辆车有最大载重和续航里程限制
- 每个订单有配送时间窗
- 每辆车每天最多工作 8 小时
- 所有订单必须被配送

**要求**：

1. **方法1（ε-约束法）**：将碳排放作为约束，最小化成本和最大化准时率
   - 取 10 个不同的碳排放上限，求解 10 次
   - 绘制双目标帕累托前沿

2. **方法2（加权求和法）**：将三个目标合并为单目标
   - 尝试至少 5 组权重
   - 对比和 ε-约束法的结果

3. **方法3（NSGA-II 示意）**：用进化算法一次跑出三目标帕累托前沿
   - 种群大小 100，代数 200
   - 绘制三目标 3D 散点图（颜色编码第三个目标）

4. **分析**：
   - 三个目标之间的 trade-off 量化（如「降低 10% 碳排放会使成本上升多少」）
   - 敏感性分析（电动车续航变化 ±20% 对结果的影响）
   - 哪组解最适合当前的业务情景？（公司正处于扩张期，现金流紧张）

### ✅ 毕业项目验证标准

1. ε-约束法：碳排放约束越紧，成本越高，准时率越低——趋势单调性验证
2. 加权求和法：不同权重得到不同区域的最优解
3. NSGA-II：三目标前沿覆盖 ε-约束法的双目标前沿投影
4. 灵敏度分析结论合理：续航下降 20% 时成本上升比例 < 续航下降比例（非线性效应）

## 附录A：什么时候不需要多目标优化

| 场景 | 理由 |
|------|------|
| 目标之间不冲突 | 如果 f₁ 提高时 f₂ 也自动提高——实际是单目标问题，不要人为复杂化 |
| 决策者能明确给权重且不改变 | 直接加权求和跑一次就行，不需要整个前沿 |
| 决策变量少于 3 个 | 直接穷举所有方案画雷达图给决策者看，更快 |
| 数据完全不可靠 | 多目标优化的结果精度受限于输入质量——先解决数据问题 |
| 公司文化只认「一个数」 | 多目标输出是一组方案——这不是一个数字能交差的 |

## 附录B：TOP 5 必踩坑

1. **没归一化就加权求和** → 量纲不同的目标直接加——量级大的目标主导结果，权重白设
2. **加权求和找非凸前沿** → 调了一百组权重，前沿中间还是缺一块——不是权重的问题，是方法的问题，换 ε-约束法或 NSGA-II
3. **ε-约束法中 ε 选太松或太紧** → 太松 = 约束不起作用（所有解一样），太紧 = 无可行解。先用各目标的单目标最优值确定 ε 的合理范围
4. **NSGA-II 种群太小** → 种群 < 50 时，前几代就收敛到局部区域，整个前沿都覆盖不全。至少 100
5. **把帕累托前沿上所有解直接扔给决策者** → 100 个点？决策者会晕。先用聚类或 TOPSIS 降维到 5~7 个代表性解

| 结合方式 | 说明 | 典型应用 |
