# 物理方程求解基础课程

> 一个周末，从零到能理解偏微分方程是什么、物理问题怎么建模、以及用数值方法求解真实世界的物理现象。

---

## 这份教程是什么

这是一份给**有高等数学基础（微积分、线性代数）、会写 Python、但对偏微分方程和物理模拟完全陌生**的工科/技术人写的自学教程。它不讲虚的——每一章都围绕「这个方程描述了什么物理现象」和「这个东西在工程中到底在干什么」展开。读完以后，你不一定会成为计算物理学家，但你一定会：

- 跟同事解释清楚「偏微分方程是干什么的、为什么物理世界需要它来描述」
- 看懂有限差分、有限元的基本思路，知道什么时候该用哪种方法
- 用 Python + 数值库把工程中的 PDE 问题离散化并求解

## 术语速查表

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 偏微分方程 | Partial Differential Equation (PDE) | 描述一个物理量在空间和时间上如何变化的数学方程。像天气预报——告诉你未来的温度分布 |
| 阶数 | Order | 方程中最高阶偏导数的阶数。一阶=只关注变化率，二阶=关注变化率的变化率 |
| 线性 PDE | Linear PDE | 未知函数和它的导数都是线性的——好解，可以用叠加原理 |
| 非线性 PDE | Nonlinear PDE | 未知函数和它的导数之间有乘积——难解，真实世界大多是这种 |
| 椭圆型方程 | Elliptic PDE | 像一张拉紧的膜，每个点的值由周围所有点共同决定（稳态热传导） |
| 抛物型方程 | Parabolic PDE | 像一个慢慢冷却的咖啡杯——时间向前走，温度逐渐均匀（瞬态热传导、扩散） |
| 双曲型方程 | Hyperbolic PDE | 像敲鼓时声波传播——信息以有限速度沿着特征线传播（波动方程、激波） |
| 边界条件 | Boundary Condition (BC) | 物理问题在边界上已知的信息——边界温度已知？边界固定不动？ |
| 初始条件 | Initial Condition (IC) | 时间相关问题的起始状态——t=0 时的温度场是什么样的？ |
| Dirichlet 边界 | Dirichlet BC | 边界上的函数值已知——比如锅边温度固定为 100°C |
| Neumann 边界 | Neumann BC | 边界上的函数梯度已知——比如锅边是绝热的（热流量为零） |
| Robin 边界 | Robin BC | 边界上函数值和梯度的组合已知——比如表面对流换热 |
| 有限差分法 | Finite Difference Method (FDM) | 用相邻点的差商代替导数——最简单的数值离散方法 |
| 有限元法 | Finite Element Method (FEM) | 把区域切成小单元，在每个单元上假设解的形式——适合复杂几何 |
| 有限体积法 | Finite Volume Method (FVM) | 把区域切成小控制体，在每个体上保证守恒——流体力学的最爱 |
| 网格 | Mesh / Grid | 把连续的空间切成离散的小块——分辨率越高越精确但也越慢 |
| 稳定性 | Stability | 数值解不会随着时间步进越跑越偏——不稳定的方法会爆炸 |
| 收敛性 | Convergence | 网格越细，数值解越接近真实解 |
| CFL 条件 | CFL Condition | 时间步长不能太大，否则信息跑得比网格快——算法会炸 |
| 刚度 | Stiffness | 系统中不同物理过程的时间尺度相差巨大——你必须用很小的步长才能跟上最快的过程 |
| 特征线 | Characteristic Line | 信息在 PDE 中传播的路径——波动沿特征线走，扩散没有明显的特征线 |

> 读到任何一个术语忘了什么意思，回到这里看「一句话人话」列，3 秒搞定。

# 第一部分：物理背景与 PDE 简介

> 在这一章，你不会看到一行代码。你会看到：为什么 PDE 是物理学的通用语言？一个物理问题是怎么从「热咖啡变凉」这样的日常现象变成一组数学方程的？以及——当你面对一个需要模拟的物理场景，从哪开始？

---

## 1.1 没有 PDE 之前，世界是什么样子的

先想象一个场景。

你是一家芯片设计公司的散热工程师。老板拿过来一块新的芯片设计——指甲盖大小、功耗 150W，比一台电烙铁还热。你的问题是：芯片在工作时，内部温度分布是什么样的？最高温会不会烧掉？散热器有没设计对？

你拿到的数据是：芯片功耗 150W，体积 1cm³，硅的导热系数 130 W/(m·K)。你怎么算温度？

**传统做法**：假设整个芯片温度一样，用一维公式简单地算一下平均温升——发现大约温升 30°C。报告上写「在安全范围内」。结果样片出来，一台机运行 30 分钟后直接过热保护。为什么？因为芯片内部有一个 0.1mm × 0.1mm 的「热点」，那里的局部功耗密度是平均值的 20 倍——温度在那一小块地方直接飙到了 180°C，而不是你算的平均 30°C。

这正是问题的核心：**物理世界是三维 + 时间的连续体，你不能用一个数来描述它。** 温度不是「一个温度」，它是一个场——空间中的每个点都有一个温度值，且这些值随时间变化。

**PDE 就是描述场的语言。** 没有 PDE，你只能用「全局平均」来糊弄——糊弄得了报告，糊弄不了物理。

### 从直觉到方程：三个概念

PDE 不像 MIP 那样有「变量/约束/目标」三个要素。它有自己的一套概念：

| 概念 | 生活类比 | 数学含义 |
|------|----------|---------|
| **场** | 你用手摸一块铁板——每个位置温度不一样，这就是温度场。 | 一个关于空间坐标（x,y,z）和可能时间 t 的函数 u(x,y,z,t) |
| **梯度** | 你从山顶往下走——越陡的地方「下降越快」。梯度就是告诉你哪个方向变化最快。 | ∇u = (∂u/∂x, ∂u/∂y, ∂u/∂z)——函数在空间中的变化率向量 |
| **通量** | 你往水池里滴一滴墨，墨向四面八方散开——单位时间穿过某个截面的墨的量就是通量。 | 某物理量通过单位面积单位时间的流动速率 |
| **散度** | 一个区域内的「净流量」——流入的多还是流出的多？散度为正表示源（向外喷），为负表示汇（向内吸）。 | ∇·F——向量场 F 的散度（源/汇密度） |
| **拉普拉斯算子** | 想象一个橡皮膜，你在一点按压它——凹陷的形状大致由周围点的曲率决定。拉普拉斯就是这种「平均曲率」。 | ∇²u = ∂²u/∂x² + ∂²u/∂y² + ∂²u/∂z² |
| **边界条件** | 煮鸡蛋——锅的内壁温度恒定为 100°C，这就是边界上的「已知信息」。 | 在区域边界上规定的条件，使解唯一 |
| **初始条件** | 你开始计时的那一瞬间——水杯里的水温分布是什么样的？ | t=0 时整个空间上的场分布 |

## 1.2 一段八卦：PDE 是怎么从流体力学变成现代工程模拟基石

### 18 世纪：弦的振动——第一个 PDE

1746 年，达朗贝尔（d'Alembert）研究了绷紧的弦的振动问题。他推导出了**波动方程**：

$$
\frac{\partial^2 u}{\partial t^2} = c^2 \frac{\partial^2 u}{\partial x^2}
$$

这可能是人类历史上第一个 PDE。达朗贝尔证明了：弦的形状可以写成两个行波的叠加——一个向右走，一个向左走。这就是著名的达朗贝尔公式。

生活类比：你把一根绳子一端的抖动传到了另一端。这就是波——信息以有限速度在介质中传播。你扔一块石头进池塘，涟漪散开——也是波动方程。

### 1822：傅里叶的热传导方程

傅里叶（Fourier）在研究热传导问题时提出了**热传导方程**：

$$
\frac{\partial u}{\partial t} = \alpha \nabla^2 u
$$

他的观点极其深刻：热量流动的速度正比于温度梯度——温差越大，传热越快。这套理论虽然当时被拉普拉斯、拉格朗日等人质疑（认为「把热分解为正弦波」是荒谬的），但最终成为了数学物理的基石。

更重要的是，傅里叶在这个过程中发明了**傅里叶级数**——任何一个周期函数都可以分解为正弦波的叠加。这个发现后来被用到从 JPEG 压缩到量子力学的所有领域。

### 什么时候你需要 PDE：现实信号

```text
你该用 PDE 的信号：
- 问题涉及「空间中的分布」（不是一个数，是一个场）
- 问题涉及「随时间的变化」（不光是稳态）
- 物理量之间有「梯度驱动的流动」（热流、水流、电流）

你不该用 PDE 的信号：
- 只需要一个全局平均值（那用 ODE 或代数方程就够了）
- 系统可以看作 0 维黑箱（那连微分方程都不需要）
- 精度不重要，经验公式可以糊弄过去
```

## 1.3 三大经典 PDE：物理学的三个基本原型

所有的物理 PDE——无论看起来多复杂——都可以归结为三种数学类型之一。识别出类型，你就知道它的「脾气」和「解法偏好」。

### 类型一：热传导方程 / 扩散方程（抛物型）

**原型方程**：
$$
\frac{\partial u}{\partial t} = \alpha \nabla^2 u
$$

**物理直觉**：一个区域内的「不均匀」会随时间被抹平。热量从高温到低温流动，直到变成均匀的稳态。

**关键特性**：
- 时间不可逆——你不能从均匀的温度场「反推」出原来的热点在哪。熵增的方向。
- 信息传播速度无限快（理论上）——你在一点加热，隔热上所有的点会「同时」感受到。
- 稳态解满足 ∇²u = 0（拉普拉斯方程）

**你何时会遇到它**：
- 热传导（芯片散热、建筑保温、金属淬火）
- 扩散（污染物在水中的扩散、药物在组织中的释放）
- 金融（Black-Scholes 方程的本质就是热传导方程的反向形式）

### 类型二：波动方程（双曲型）

**原型方程**：
$$
\frac{\partial^2 u}{\partial t^2} = c^2 \nabla^2 u
$$

**物理直觉**：扰动以有限速度传播。你敲一下鼓——鼓面振动以声速传出去。

**关键特性**：
- 有时间可逆性——理论上你可以从现在的波反推过去的状态
- 信息传播有速度上限 c——这是物理因果律的数学表达
- 有特征线——沿着某些方向，方程会简化为一阶常微分方程

**你何时会遇到它**：
- 声波（音响设计、噪声控制、地震勘探）
- 电磁波（天线设计、雷达、光通信——Maxwell 方程的本质是波动方程）
- 机械振动（桥梁振动、汽车悬挂、地震中建筑物的响应）

### 类型三：拉普拉斯方程 / 泊松方程（椭圆型）

**原型方程**：
$$
\nabla^2 u = 0 \quad \text{(拉普拉斯)} \qquad \nabla^2 u = f \quad \text{(泊松)}
$$

**物理直觉**：稳态、平衡。已经没有随时间的变化了——系统已经达到了「最终形状」。

**关键特性**：
- 没有时间变量——描述的是稳态
- 满足极值原理——解的最大值和最小值都在边界上（内部不会出现比边界更大的值）
- 解是无限可微的——只要边界条件足够光滑，内部解也是光滑的

**你何时会遇到它**：
- 稳态热传导（温度不再变化时的温度分布）
- 静电场（电荷分布产生的静电势）
- 不可压缩无旋流动（势流——绕机翼的流动近似）
- 扭转问题（一根柱子的扭转角度分布）

### 三大 PDE 速查对照表

| | 热传导/扩散 (抛物型) | 波动 (双曲型) | 拉普拉斯/泊松 (椭圆型) |
|--|---------------------|--------------|---------------------|
| 典型方程 | uₜ = α∇²u | uₜₜ = c²∇²u | ∇²u = 0 |
| 物理含义 | 扩散、抹平 | 波传播 | 稳态、平衡 |
| 时间方向 | 单向（不可逆） | 双向（可逆） | 无时间 |
| 信息速度 | 无限快 | 有限速度 c | N/A |
| 需要初始条件 | 是 | 是（位置+速度） | 否 |
| 需要边界条件 | 是 | 是 | 是 |
| 是否耗散 | 是（能量减少） | 否（能量守恒） | 稳态 |
| 数值方法偏好 | 隐式格式 | 显式格式 + CFL | 迭代法（如 Gauss-Seidel） |

## 1.4 边界条件：现实世界在哪告诉你「停下来」

PDE 本身描述的是物理规律——热量怎么流动、波怎么传播。但如果没有边界条件，PDE 有无数个解。边界条件就是**把数学放回现实世界**的锚。

### 三种最常见的边界条件

| 类型 | 数学表达 | 物理含义 | 生活类比 |
|------|---------|---------|---------|
| Dirichlet | u = g (在边界上) | 边界上的值已知 | 锅边温度固定为 100°C——你设定锅边的值 |
| Neumann | ∂u/∂n = q (在边界上) | 边界上的梯度已知 | 锅边是绝热的——没有热量穿过锅边（q=0）|
| Robin | ∂u/∂n + k(u - u_ext) = 0 | 边界上的换热情已知 | 锅边暴露在空气中散热——传热系数已知 |

**为什么边界条件这么重要？** 因为物理世界不是无限的。一块芯片有边缘，一跟管道有壁面，一个地震波的传播区域有地球表面。边界条件就是这些「截断」的数学描述。选错了边界条件，你的模拟结果可能完全相反。

---

## 1.5 建模心法：从物理现象到 PDE 的五步法

不是你写不写得出方程的问题。是你根本不知道从哪里开始。

### 第一步：确定控制方程（1 分钟）

回答三个问题：
1. **我关心什么物理量？**（温度 T？位移 u？压强 p？电势 φ？）
2. **随时间变化吗？**（是 → 抛物型或双曲型；否 → 椭圆型）
3. **有「扩散」还是「波动」行为？**（抹平 → 扩散方程；振荡传播 → 波动方程）

| 你观察到的行为 | 对应的 PDE 类型 |
|--------------|----------------|
| 冷热物体逐渐达到相同温度 | 抛物型（热传导/扩散方程）|
| 敲了会响，会传出去 | 双曲型（波动方程）|
| 放那不动了，稳定了 | 椭圆型（拉普拉斯/泊松方程）|

### 第二步：确定空间维度（30 秒）

真实世界是 3D 的。但：
- 如果沿某个方向没有变化（比如无穷长管道沿轴向温度恒定）→ 降一维
- 如果对称（圆柱对称、球对称）→ 用极坐标/球坐标降低一维
- **经验法则**：从 1D 开始，跑通再上 2D/3D。1D 试错成本最低。

### 第三步：列出参数（30 秒）

把问题中所有的已知数值列出来：
- 材料参数（导热系数 α、波速 c、弹性模量 E、泊松比 ν）
- 几何参数（长度 L、宽度 W、厚度 H）
- 边界条件（边界温度、边界热流量、固定端、自由端）
- 初始条件（t=0 时的场分布）

### 第四步：列出边界条件（1 分钟）

用大白话描述每个边界的情况：
- 「左边界：温度固定为 100°C」→ Dirichlet BC
- 「右边界：绝热，没有热量流出去」→ Neumann BC (q=0)
- 「上边界：暴露在空气中，散热，散热系数已知」→ Robin BC
- 「下边界：对称面——热流量为零」→ Neumann BC (q=0)

### 第五步：检查量纲（30 秒）

方程的每一项必须有相同的量纲。这是最快速的不合理检测——如果量纲对不上，方程一定写错了。

```
举例：热传导方程 uₜ = α∇²u
  uₜ 的量纲：K/s（温度随时间的变化率）
  ∇²u 的量纲：K/m²（温度的二阶空间导数）
  α 的量纲：m²/s（热扩散系数——为了凑出 K/s 的右边）
  左右都是 K/s ✓
```

## 2.2 有限差分法 (FDM)：最简单的入门

### 核心直觉

不用求导数——用相邻点的函数值「近似」导数。

```
精确导数：  ∂u/∂x = lim (u(x+Δx) - u(x)) / Δx      当 Δx→0
数值导数：  ∂u/∂x ≈ (u(x+Δx) - u(x)) / Δx         Δx 是网格间距
```

### 三种基本差分格式

```
前向差分:    ∂u/∂x ≈ (u_{i+1} - u_{i}) / Δx       精度 O(Δx)
后向差分:    ∂u/∂x ≈ (u_{i} - u_{i-1}) / Δx       精度 O(Δx)
中心差分:    ∂u/∂x ≈ (u_{i+1} - u_{i-1}) / (2Δx)  精度 O(Δx²)
```

二阶导数（三点中心差分）：
```
∂²u/∂x² ≈ (u_{i-1} - 2u_{i} + u_{i+1}) / Δx²    精度 O(Δx²)
```

### 一维热传导方程的离散化

**原始 PDE**：
$$
\frac{\partial u}{\partial t} = \alpha \frac{\partial^2 u}{\partial x^2}
$$

**时间用前向差分，空间用中心差分**：
```
(u^{n+1}_i - u^{n}_i) / Δt = α (u^{n}_{i-1} - 2u^{n}_i + u^{n}_{i+1}) / Δx²
```

整理得到**显式时间推进格式**：
```
u^{n+1}_i = u^{n}_i + α·Δt/Δx² · (u^{n}_{i-1} - 2u^{n}_i + u^{n}_{i+1})
```

**生活类比**：你有一排城市，每个城市知道自己和邻居今天的温度。你用「我今天和邻居的温差」来估计「明天我有多热」。这就是显式格式——每个点只看邻居，就能算出下一步。

### 一维波动方程的离散化

**原始 PDE**：
$$
\frac{\partial^2 u}{\partial t^2} = c^2 \frac{\partial^2 u}{\partial x^2}
$$

**时间用二阶中心差分，空间用中心差分**：
```
(u^{n+1}_i - 2u^{n}_i + u^{n-1}_i) / Δt² = c² (u^{n}_{i-1} - 2u^{n}_i + u^{n}_{i+1}) / Δx²
```

整理：
```
u^{n+1}_i = 2u^{n}_i - u^{n-1}_i + (cΔt/Δx)² · (u^{n}_{i-1} - 2u^{n}_i + u^{n}_{i+1})
```

**生活类比**：拨动一根吉他弦。你根据「弦现在的形状」和「上一时刻的形状」来推算「下一时刻的形状」。振动就是这样传播的。

## 2.3 稳定性与 CFL 条件

### 为什么显式格式会「炸」？

如果你时间步长太大，信息在一个时间步内传播了超过一个网格单元——数值解就会指数级放大误差，最终表现为结果「爆炸」到无穷大。

### CFL 条件：Courant-Friedrichs-Lewy

**物理直觉**：信息在 PDE 中传播有速度（波速 c）。数值方法在一个时间步 Δt 内最多传播一个网格间距 Δx。所以你必须保证：

```
信息走的距离 ≤ 网格间距
c · Δt ≤ Δx
```

即：
```
CFL 数 = c·Δt / Δx ≤ 1
```

### 三大 PDE 的稳定性条件速查

| 方程 | 显式格式的稳定性条件 | 含义 |
|------|---------------------|------|
| 热传导（抛物型） | α·Δt / Δx² ≤ 0.5 | 时间步长受 Δx² 限制——网格加密一倍，步长要缩小到 1/4 |
| 波动（双曲型） | c·Δt / Δx ≤ 1 | 时间步长受 Δx 限制——网格加密一倍，步长缩小到 1/2 |
| 拉普拉斯（椭圆） | N/A（无时间步） | 用迭代法收敛即可 |

**工作类比**：
- 热传导方程中，稳定性随 Δx² 缩放——这意味着如果你把网格精度提高一倍（Δx 减半），你需要用 4 倍的时间步数。这是**隐式格式**登场的原因——隐式格式无条件稳定，允许更大的时间步长。
- 波动方程中，稳定性随 Δx 线性缩放——显式格式通常够用。

---

## 2.4 显式 vs 隐式：工程中的关键选择

| | 显式格式 | 隐式格式 |
|--|---------|---------|
| 时间推进 | 每个时间步直接算 | 每个时间步需要解方程组 |
| 稳定性 | 有条件（CFL） | 无条件稳定 |
| 每步开销 | O(n) | O(n) ~ O(n³)（取决于求解器） |
| 适合场景 | 波动、快速瞬态 | 热传导、慢速瞬态、稳态 |
| 并行 | 易并行 | 难并行 |

**选择原则**：
- 如果问题的「自然时间尺度」远小于 CFL 允许的最大步长 → 用显式（波动方程通常是这种情况）
- 如果 CFL 允许的步长远小于你想研究的物理时间范围 → 用隐式（热传导方程通常是这种情况）
- 如果你只关心稳态 → 直接解椭圆型方程，或用隐式大步长推过去

---

## 2.5 有限元法 (FEM)：复杂几何的秘密武器

### 为什么 FDM 不够？

有限差分法需要**结构化网格**（网格点按规则排列）。当几何形状复杂时——比如发动机缸体、飞机机翼——FDM 难以处理边界。

### FEM 的核心思想

1. **剖分**：把复杂区域切成简单的小块（三角形、四面体）
2. **假设**：在每个小块上，假设解的形式是一个简单的多项式（通常是线性或二次）
3. **组装**：把所有小块的方程组装成一个全局方程组
4. **求解**：解这个大型稀疏线性系统

**生活类比**：FDM 像一个方格纸——只能画方形的图案。FEM 像乐高——用三角形、四边形任意拼出复杂形状。

---

## 2.6 有限体积法 (FVM)：流体力学的最爱

### FVM 的核心差别

FVM 直接把 PDE 的**积分形式**（而非微分形式）离散化：

$$
\int_V \frac{\partial u}{\partial t} dV + \oint_S \mathbf{F} \cdot \mathbf{n} dS = 0
$$

**优势**：天然保证守恒——从每个控制体流出的通量恰好等于相邻控制体流入的通量。对于流体力学（质量、动量、能量必须精确守恒）来说，这是巨大的优势。

---

## 2.7 三大数值方法对比

| | FDM（有限差分） | FEM（有限元） | FVM（有限体积） |
|--|---------------|-------------|---------------|
| 核心思想 | 差商代替导数 | 分片多项式近似 | 守恒形式的积分 |
| 网格要求 | 结构化 | 非结构化 | 结构化/非结构化均可 |
| 复杂几何 | ❌ 困难 | ✅ 擅长 | ✅ 擅长 |
| 守恒性 | ❌ 不保证 | ❌ 不保证 | ✅ 天然保证 |
| 实现难度 | ★☆☆☆☆ 简单 | ★★★☆☆ 复杂 | ★★☆☆☆ 中等 |
| 典型应用 | 学术、简单几何 | 结构力学、多物理 | 流体力学、传热学 |
| 开发时间 | 几小时 | 几周 | 几天 |
