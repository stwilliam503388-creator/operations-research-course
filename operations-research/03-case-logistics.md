# 案例1：物流网络优化 — 运输问题 / LP / 最小费用流

> 难度：★★☆☆☆ | 前置知识：无 | ⏱ 阅读时间：30 分钟

---

## 1. 场景：从工厂到客户的 2.5 亿元账单

你是一家消费品公司的供应链负责人。公司有 **3 个工厂**、**10 个区域仓库**、**200 个城市的客户**。

每个月，你的任务是：

1. **从 3 个工厂把货送到 10 个仓库**
2. **从 10 个仓库把货送到 200 个客户手里**

总配送成本——**2.5 亿元/年**。老板说：今年砍 10%。

你之前的做法：靠调度员凭经验压价、砍路线——省个 2-3% 就吃力了。今年你打算换个路子：上运筹学。

---

## 2. 业务问题 → 数学模型

先把这个业务问题拆成运筹学的三要素：

| 要素 | 业务对应 | 数学形式 |
|------|---------|---------|
| **决策变量** | 从每个工厂往每个仓库送多少货 | $x_{ij} \geq 0$（连续变量） |
| **决策变量** | 哪个仓库服务哪个客户 | $y_{jk} \in \{0, 1\}$（二进制） |
| **约束** | 工厂产能有限 | $\sum_j x_{ij} \leq S_i$ |
| **约束** | 仓库进出平衡 | $\sum_i x_{ij} = \sum_k D_k \cdot y_{jk}$ |
| **约束** | 每个客户只由一个仓库服务 | $\sum_j y_{jk} = 1$ |
| **约束** | 仓库容量限制 | $\sum_k D_k \cdot y_{jk} \leq W_j$ |
| **目标** | 最小化总配送成本 | $\min \sum_i \sum_j c_{ij} x_{ij} + \sum_j \sum_k d_{jk} D_k y_{jk}$ |

> **一句话**：在满足所有客户需求的前提下，找到从工厂经仓库到客户的最省钱路线。

### 为什么这是个运输问题？

把工厂看成"供给节点"，仓库看成"需求节点"——这不就是经典的**运输问题**吗？

运输问题的特殊之处在于：
- 所有约束都是等式（或不等式），且系数矩阵全是 0 或 1
- 这个矩阵是**幺模矩阵（totally unimodular）**
- **数学魔法**：即使你把它当 LP 解（允许小数），得到的解天然就是整数

这就是为什么运输问题"好解"——不需要整数规划，LP 就够了。

## 3. 数学模型（完整版）

### 集合

| 符号 | 含义 |
|------|------|
| $I = \{1, 2, 3\}$ | 工厂集合 |
| $J = \{1, 2, ..., 10\}$ | 仓库集合 |
| $K = \{1, 2, ..., 200\}$ | 客户集合 |

### 参数

| 符号 | 含义 | 单位 |
|------|------|------|
| $S_i$ | 工厂 $i$ 的月产能 | 吨 |
| $W_j$ | 仓库 $j$ 的月容量 | 吨 |
| $D_k$ | 客户 $k$ 的月需求量 | 吨 |
| $c_{ij}$ | 从工厂 $i$ 到仓库 $j$ 的单位运费 | 元/吨 |
| $d_{jk}$ | 从仓库 $j$ 到客户 $k$ 的单位运费 | 元/吨 |

### 决策变量

$$
\begin{aligned}
x_{ij} &\geq 0 \quad &&\text{工厂 } i \rightarrow \text{仓库 } j \text{ 的货物量} \\
y_{jk} &\in \{0, 1\} \quad &&\text{仓库 } j \text{ 是否服务客户 } k
\end{aligned}
$$

### 目标函数

$$
\min \sum_{i=1}^{3} \sum_{j=1}^{10} c_{ij} \cdot x_{ij} \;+\; \sum_{j=1}^{10} \sum_{k=1}^{200} d_{jk} \cdot D_k \cdot y_{jk}
$$
> **第一项**：工厂到仓库的运输成本
> **第二项**：仓库到客户的配送成本

### 约束

$$
\begin{aligned}
\sum_{j=1}^{10} x_{ij} &\leq S_i, \quad \forall i \quad &&\text{① 工厂产能约束} \\
\sum_{i=1}^{3} x_{ij} &= \sum_{k=1}^{200} D_k \cdot y_{jk}, \quad \forall j \quad &&\text{② 仓库进出平衡} \\
\sum_{j=1}^{10} y_{jk} &= 1, \quad \forall k \quad &&\text{③ 客户唯一仓库} \\
\sum_{k=1}^{200} D_k \cdot y_{jk} &\leq W_j, \quad \forall j \quad &&\text{④ 仓库容量约束} \\
x_{ij} &\geq 0, \; y_{jk} \in \{0, 1\} \quad &&\text{⑤ 非负与整数}
\end{aligned}
$$

### 逐行注释

| 行 | 它在说什么 | 业务含义 |
|----|-----------|---------|
| ① | 每个工厂运出去的总货 ≤ 产能 | 别让工厂超负荷，工人会造反 |
| ② | 仓库收到的 = 仓库送出的 | 仓库只是个中转站，不囤货 |
| ③ | 每个客户只对应一个仓库 | 客户不需要知道你的仓库结构 |
| ④ | 仓库处理的货 ≤ 仓库容量 | 仓库堆不下也白搭 |

## 4. 求解方法选择

| 方法 | 能否用 | 原因 |
|------|--------|------|
| **单纯形法** | ✅ 基础 | 运输问题是 LP，单纯形法秒解 |
| **运输问题专用算法（MODI）** | ✅ 更优 | 比通用单纯形法快 10-100 倍 |
| **最小费用流** | ✅ 等价 | 可以建模为网络流，用最短路增广 |
| 内点法 | ✅ 杀鸡用牛刀 | 规模不大，不需要 |
| 分支定界 | ❌ 没必要 | 运输问题的 LP 解天然整数 |
| 启发式 | ❌ 没必要 | 精确解秒出，不需要近似 |

**为什么选 MODI 法？**

MODI（Modified Distribution）法是运输问题的专用算法，核心思路：

1. **先找一个初始可行解**（西北角法，简单粗暴）
2. **计算每个非基变量的检验数**（如果所有 ≥ 0 → 最优）
3. **如果某个检验数 < 0 → 找到闭回路，调整流量**
4. **重复直到最优**

这个算法本质上就是**单纯形法在运输问题上的特化版本**，但利用了运输问题的特殊结构（系数矩阵全是 0/1），省去了通用单纯形法的很多计算量。

## 6. 运行结果解读

### 6.1 工厂→仓库：最优配送方案

MODI 法经过几次迭代就找到了最优解。结果告诉你：

- **工厂 0 主要供应仓库 0、1、2**（因为它离这些仓库近，运费低）
- **工厂 1 供应仓库 2~6**（覆盖中部区域）
- **工厂 2 供应仓库 7~9**（覆盖远端区域）

每个工厂和仓库之间的配对，都是**全局最优的**——不存在任何一个调整能让总成本更低。

### 6.2 影子价格：告诉你瓶颈在哪

影子价格（对偶变量）是运输问题输出中最有业务价值的信息：

```
仓库0: v=16.12 → 这个仓库扩容 1 单位，总成本可降 16.12 元
仓库4: v=91.07 → 这个仓库扩容 1 单位，总成本可降 91.07 元
```

**业务含义**：
- 仓库 4 的影子价格很高（91.07）→ 它是网络中的**瓶颈节点**
- 把仓库 4 的容量扩大 100 单位，理论上可节省 9107 元/月
- 相比之下，仓库 0 的影子价格低很多（16.12）→ 扩容优先级低

这就给了管理者一个**投资决策的量化依据**：优先扩容影子价格高的仓库。

### 6.3 客户分配：200 个客户各归其位

每个客户都被分配到了成本最优的仓库：
- 仓库 0: 21 个客户（最近）
- 仓库 3: 26 个客户（覆盖密集区域）
- 仓库 9: 31 个客户（偏远地区的大仓）
- ...

所有客户都分配到了**且只分配到一个仓库**，✅ 验证通过。

### 6.4 灵敏度分析：运费波动

对每个工厂→仓库的运费做了 ±10% 的变动，重新求解。结果：

**所有 60 个扰动场景（3×10×2）中，最优基变量全部不变。**

这说明当前的配送方案对运费波动有很强的鲁棒性。业务层面的意义：
- 跟运输公司谈合同的时候，**±10% 以内的价格波动不影响方案**
- 不需要因为运费微调而重新规划整个网络
- 当然，如果某个路线运费暴涨 50% 以上，方案可能会变

## 8. 📖 洞察

### 为什么运输问题这么"好解"？

运输问题的约束矩阵长这样：

```
[1 1 1 0 0 0 ...]  工厂1的产能约束
[0 0 0 1 1 1 ...]  工厂2的产能约束
[1 0 0 1 0 0 ...]  仓库1的需求约束
[0 1 0 0 1 0 ...]  仓库2的需求约束
```

每列只有两个 1（一个工厂、一个仓库），其余全是 0。这种结构叫**幺模矩阵**——它保证了：
1. LP 的可行域顶点坐标全是整数
2. 用单纯形法求解，得到的一定是整数解
3. 不需要昂贵的整数规划算法

> **通俗理解**：运输问题就像拼乐高——每个积木块都正好能严丝合缝地拼在一起，不会出现"半块"的情况。

### 影子价格的真实业务价值

很多管理者觉得运筹学是"黑箱"，但影子价格让这个黑箱透明化：

- **高影子价格的仓库** → 那是你网络的"咽喉" → 赶紧扩容或建新仓
- **低影子价格的仓库** → 多它不多，少它不少 → 可以关掉/外包
- **工厂 u_i 为负** → 这个工厂是低成本供给源 → 扩大产能

我们的案例里，仓库 8 的影子价格高达 109.35 元/单位——说明它严重供不应求，扩容的**边际收益极高**。

### 灵敏度分析在谈判桌上的作用

运费谈判时，你最怕的是什么？签了低价合同，结果配送方案要重做。

灵敏度分析告诉你：
- 哪些路线的运费波动不影响方案 → **大胆压价**
- 哪些路线一涨价就得重新规划 → **锁死价格**

## 9. 延伸思考

学完这个案例，你可以在这些方向上继续深入：

1. **设施选址问题**（如果引入仓库的固定建设成本 $F_j$）
   - 模型变成：要不要建这个仓库（0/1 决策）+ 怎么配送
   - 方法变成：MIP（混合整数规划）
   - 更贴近现实，但也更难解

2. **多周期动态网络流**（如果考虑时间维度）
   - 每个周期的需求和产能都在变化
   - 可以囤货：这个周期的货下个周期送
   - 方法变成：时间扩展网络 + 最小费用流

3. **需求不确定下的鲁棒优化**
   - 如果只知道需求的范围，不知道具体值
   - 需要在最坏情况下也有可行方案
   - 方法变成：鲁棒对应（robust counterpart）

4. **库存路径问题（IRP）**
   - 把库存决策和运输决策同时优化
   - 经典 NP-hard 问题，工业界非常常见
