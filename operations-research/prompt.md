# 运筹学基础与实战课程

> 一个周末，从零到能看懂运筹学在干什么、把业务问题建模成优化模型、以及选择合适的算法求解。

---

## 这份教程是什么

这是一份给**有高中数学和基础 Python、但对运筹学完全陌生**的技术人/业务决策者写的自学教程。它不讲学院派的定理证明——每一章都围绕「这个模型解决什么业务问题」和「这个算法为什么在这个场景下好使」展开。

读完以后，你不一定会成为运筹学专家，但你一定会：

- 跟同事说清楚「线性规划→整数规划→启发式」的递进关系
- 看到一个业务问题，判断它适合用 LP、MIP、网络流还是启发式
- 看懂求解器日志，知道什么时候该停、什么时候该换方法

## 运筹学的基本矛盾

先记住一句话，贯穿整个课程：

> **你做得越精确，花的时间越长。你做得越快，离最优解就越远。**

这就是运筹学的核心矛盾——**精度 vs 速度**。它不是缺陷，是这门学科的灵魂。当你面对一个业务问题：
- 如果时间够，用精确算法（LP/MIP 求解器）找到最优解
- 如果时间紧但规模小，也用精确算法
- 如果规模大到精确算法算不完，用启发式换速度
- 如果连模型都建不出来，用仿真评估

每当你在这门课里看到"选哪个方法"，背后都是在回答同一个问题：**你愿意为精度付出多少时间？**

# 第一部分：运筹学概览

> 在这一章，你不会看到一行代码。你会看到：运筹学是怎么从一个军事后勤工具变成今天千亿美元产业的核心引擎的？一个业务问题怎么被拆解成「决策变量 / 约束 / 目标」这三个要素？以及——你应该从哪开始学？

---

## 1.1 没有运筹学之前，世界是什么样子的

先想象三个场景。

**场景 A：物流**。你的公司在 50 个城市有仓库，每天要把 10 万件货送到 2000 个客户手里。运费一年 3 个亿。老板说「今年砍 10% 运费」。你怎么做？没有运筹学——靠调度员凭经验压价、砍路线——省个 2-3% 就已经很吃力了。

**场景 B：生产**。你的工厂有 10 条产线、5 种产品、每种有各自的需求和截止时间。排程员排了一个通宵的方案——结果第三周发现原料不够。重新排。

**场景 C：金融**。你有 1 个亿的资金要分配到股票、债券、期货三种资产上。每种收益率不确定。怎么分配才能让收益尽可能高、同时风险可控？

这三个场景看起来完全不同，但它们的底层数学结构是一样的：**你有选择（决策变量）、你有限制（约束）、你有目标（最优化）**。这就是运筹学的全部。

### 从军事到商业：一段简史

运筹学（Operations Research）这个词本身来自二战。英国军方请了一群数学家来优化雷达部署和反潜巡逻路线——「我们要在有限的资源下达到最大的军事效果」。战后，这些人去了企业界：同样的数学，不同的战场。

- **1947**：George Dantzig 发明单纯形法（LP 的核心算法）
- **1954**：旅行商问题被证明是 NP-hard——某些问题天生难解
- **1960**：分支定界法诞生——整数规划的核心引擎
- **1984**：Karmarkar 发明内点法——大规模 LP 的突破
- **1990s~**：CPLEX / Gurobi / COPT 等商业求解器崛起
- **2010s~**：运筹学 + ML 的交叉——端到端决策优化

## 1.2 运筹学的三个基本问题

所有运筹学问题，问的其实是三件事：

| 问题类型 | 你问的问题 | 数学对应 |
|---------|-----------|---------|
| 分配问题 | 有限资源怎么分给多个需求方？ | LP / 网络流 / 整数规划 |
| 排程问题 | 什么顺序、什么时间做哪些事？ | MIP / 约束规划 / 启发式 |
| 决策问题 | 在不确定下怎么选？ | 随机规划 / MDP / 仿真 |

每个问题都可以拆成三要素：

```
决策变量 → 你要填的格子
约束     → 现实不允许的事
目标     → 你想最大化/最小化的东西
```

> **逃生通道**：无论问题看起来多复杂，先问自己这三个问题。70% 的建模问题都出在「决策变量没想清楚」。

## 1.3 运筹学的工具箱：四层递进

```
                ┌──────────────────────────────────────┐
                │      层 4：仿真（你的方案到底行不行）    │
                │  Monte Carlo / 离散事件仿真            │
                └──────────────────┬───────────────────┘
                                   │
                ┌──────────────────▼───────────────────┐
                │    层 3：启发式（找不到最优解的时候）    │
                │  模拟退火 / 遗传算法 / 禁忌搜索 / LNS   │
                └──────────────────┬───────────────────┘
                                   │
                ┌──────────────────▼───────────────────┐
                │    层 2：精确算法（求解器能做的事）      │
                │  单纯形法 / 分支定界 / 割平面 / Benders │
                └──────────────────┬───────────────────┘
                                   │
                ┌──────────────────▼───────────────────┐
                │    层 1：建模（把业务变成数学）         │
                │  LP / MIP / NLP / 网络流 / DP          │
                └──────────────────────────────────────┘
```

**使用原则**：永远从层 1 开始。如果层 1 的模型可以精确求解——直接用。如果规模太大——退化到层 2（更聪明的精确算法）或层 3（放弃精确最优，换速度）。如果连数学建模都不现实——用层 4 仿真评估。

## 1.4 四大基础模型（必须掌握）

### 1. 线性规划 (LP) — 最基础的发动机

```
min  c·x
s.t. A·x ≤ b
     x ≥ 0
```

- 所有关系都是线性的——直线的世界
- 单纯形法或内点法可解
- 百万级变量仍然秒解

### 2. 整数规划 (IP/MIP) — 真实世界

```
min  c·x + d·y
s.t. A·x + B·y ≤ b
     x ≥ 0 连续
     y ∈ {0,1} 整数
```

- 「建不建仓库」（0/1）+ 「运多少货」（连续）
- NP-hard——求解器用分支定界 + 割平面

### 3. 网络流 — 特殊结构，超快求解

```
节点 + 弧 + 流量守恒
```

- 运输、分配、最短路径、最大流
- 即使百万节点也能秒解（有专用算法）

### 4. 动态规划 (DP) — 序贯决策

```
V(s) = max_a { R(s,a) + γ·V(s') }
```

- 多阶段决策——今天的选择影响明天的空间
- 可以是确定性或随机（MDP）

### 进阶模型（先了解，用到再深学）

读者须知：以下两种模型在理论上是运筹学的重要分支，但对入门阶段来说，**你不需要在第一天就搞懂它们**。先知道它们解决什么问题、长什么样就好：

**🔹 非线性规划 (NLP)** — 当成本和收益不是直线时

```
min  f(x)    其中 f 可能是非线性的
s.t. g(x) ≤ 0
```

- 梯度下降、牛顿法、KKT 条件
- 没有「通用求解器」——需要具体问题具体分析
- **你什么时候会遇到**：投资组合（收益非线性）、定价优化（价格弹性非线性）

**🔹 随机规划 / 鲁棒优化** — 当未来不确定时

```
min  E[f(x, ξ)]            ← 假设分布已知
或  min  max_{ξ∈U} f(x, ξ)  ← 只假设范围已知
```

- 需求不确定、价格不确定、天气不确定
- 随机规划：假设分布已知；鲁棒优化：只假设集合已知
- **你什么时候会遇到**：供应链（需求不确定）、能源（可再生能源出力不确定）

## 1.5 建模心法：五步法

### 第一步：画图（2 分钟）

实体（方框）+ 关系（箭头）+ 数字（标注）。

### 第二步：找决策变量（1 分钟）

「我最后要交的那张 Excel 表里，哪些格子需要填？」

### 第三步：列约束清单（1 分钟）

用大白话列「不能」——「不超过产能、不超截止时间、不能同时排两个任务…」

### 第四步：写目标（30 秒）

一句线：最大化利润 / 最小化成本 / 最短完成时间

### 第五步：检查量纲（30 秒）

目标函数的单位 = 所有成本项的单位一致。不一致就是模型写错了。

# 第二部分：建模与算法框架

> 不讲学院派的定理证明。只讲「一个实际问题怎么变成模型」和「模型该用什么方法解」。

---

## 2.1 建模三要素再深入

### 决策变量的三种类型

| 类型 | 取值 | 典型含义 | 例子 |
|------|------|---------|------|
| 连续变量 | x ≥ 0 的任何实数 | 量、金额、时间 | 运多少货、生产多少吨 |
| 整数变量 | x ∈ {0,1,2,...} | 不可分割的数量 | 用多少辆车、雇多少人 |
| 0/1 变量 | x ∈ {0,1} | 是/否决策 | 建不建厂、选不选方案 |

**经验法则**：能用连续变量的地方不要用整数。整数越多，求解越慢。

### 约束的四种常见模式

| 约束类型 | 数学形式 | 业务含义 |
|---------|---------|---------|
| 资源上限 | Σ a_i·x_i ≤ b | 不超过产能、预算、容量 |
| 需求满足 | Σ x_i ≥ d | 必须满足的需求下限 |
| 比例关系 | x_j ≤ M·y_j | 如果不建仓库，就不能从那里发货（Big-M）|
| 互斥选择 | Σ y_i ≤ 1 | 五个方案中只能选一个 |

### 目标函数的三种常见形式

| 目标形式 | 适用场景 | 注意事项 |
|---------|---------|---------|
| 最小化成本 | 运营优化 | 别忘记固定成本 vs 可变成本 |
| 最大化利润 | 商业决策 | 收入 - 成本，两者都可能是非线性 |
| 最小化最大 | 均衡负载 | min-max 形式，可以线性化 |

```
       ┌──────────┐     ┌──────────┐     ┌──────────────┐

     │  LP         │ │  MIP         │  │  NLP         │
     │ 单纯形/内点 │ │ 分支定界+割  │  │ 梯度法/牛顿  │
     │ 超大规模    │ │  中大规模    │  │  小规模/特定 │

     └──────────────┘ └──────────────┘  └──────────────┘

                    ┌─────────┴─────────┐

```

### 各类方法适用规模速查

| 方法 | 变量数上限 | 求解时间 | 最优性保证 |
|------|-----------|---------|-----------|
| LP（单纯形/内点） | 百万+ | 秒~分钟 | ✅ 全局最优 |
| MIP（分支定界） | 万~十万 | 分钟~小时 | ✅ 可证明最优 |
| 网络流专用算法 | 百万+ | 秒 | ✅ 全局最优 |
| DP | 取决于状态数 | - | ✅ 全局最优 |
| NLP（局部优化） | 万~十万 | 秒~分钟 | ❌ 可能局部最优 |
| 元启发式（SA/GA） | 任意 | 分钟~小时 | ❌ 无保证 |
| 仿真优化 | 任意 | 小时~天 | ❌ 无保证 |

---

## 2.3 精确算法 vs 启发式 vs 仿真：怎么选

| 精度需求 | 时效要求 | 问题规模 | 建议方案 |
|---------|---------|---------|---------|
| 必须最优 | 不限 | 中小 | 精确算法（MIP 求解器）|
| 必须最优 | 快 | 小 | 精确算法 |
| 近似即可 | 快 | 大 | 启发式（邻域搜索/贪心）|
| 近似即可 | 不限 | 超大 | 元启发式 |
| 只评估 | 不限 | 任意 | 仿真 |

**关键原则**：不要一上来就用启发式。先用 LP/MIP 求解器跑一遍——哪怕跑不完，它给的**下界（dual bound）**就是你启发式结果的「评分参考」。

## 2.4 运筹学项目的标准流程

```
业务问题
   │
   ▼
Step 1: 问题定义 ← 跟业务方对齐：决策是什么？限制是什么？目标是什么？
   │
   ▼
Step 2: 数据收集 ← 没有数据就没有模型。先确认数据可得性
   │
   ▼
Step 3: 建模 ← 五步法（画图→变量→约束→目标→量纲）
   │
   ▼
Step 4: 求解 ← 算法决策树选方法
   │
   ▼
Step 5: 验证 ← 已知解注入、极端测试、敏感性分析
   │
   ▼
Step 6: 部署 ← 集成到业务系统、自动化
   │
   ▼
Step 7: 监控 ← 实际效果 vs 模型预测、参数漂移时重训
```

## 案例 1：物流网络优化（LP + 网络流）★★☆☆☆

**模型**：线性规划 / 运输问题 / 最小费用流

**场景**：3 个工厂、10 个仓库、200 个客户。工厂产能有限，仓库有固定成本，客户需求已知。求最小化总成本（生产成本 + 运输成本）的配送方案。

**教学点**：
- LP 的单纯形法直觉——为什么解一定在顶点上
- 运输问题的特殊结构——为什么比一般 LP 好解
- 对偶的经济解释——影子价格=每条路线的边际成本
- 灵敏度分析——运费变了，方案要不要变

**延伸**：
- 加入仓库选址决策 → 设施选址问题（MIP）
- 加入时间窗 → VRP

**✅ 验证标准**：
1. 求解结果中每个客户被分配到一个仓库，总供应≤总产能
2. 影子价格解释：产能增加 1 单位，目标值改善多少
3. 运费变动 ±10%，方案不变（验证灵敏度分析结论）

---

## 案例 2：库存策略优化（MIP + DP）★★★☆☆

**模型**：动态规划 / 混合整数规划

**场景**：你管理一家零售商的库存。每周需求不确定（已知分布），每次订货有固定成本、有仓储容量上限、缺货有惩罚成本。求最小化总成本（订货+持有+缺货）的多周期订货策略。

**教学点**：
- 报童（Newsvendor）问题的直觉——最优订货量 = 需求分布的某个分位数
- (s, S) 策略——库存降到 s 时订货到 S，为什么这种策略是最优的
- DP 求解多周期库存——状态 = 当前库存，动作 = 订多少
- MIP 建模库存约束——库存平衡方程 x_{t+1} = x_t + order_t - demand_t

**延伸**：
- 多产品共享仓库容量 → 加入联合补货约束
- 供应商有最小订货量 → 整数决策变量
- 考虑提前期（lead time）→ 安全库存计算

**✅ 验证标准**：
1. 单周期报童问题：最优订货量 = F^{-1}(c_u/(c_u+c_o)) 与 DP 结果一致
2. 多周期 DP：库存不会超过仓库容量（约束检查）
3. 固定成本高时，订货频率降低（经济直觉成立）

---

## 案例 3：投资组合优化（NLP）★★★☆☆

**模型**：二次规划 / 均值-方差模型

**场景**：Markowitz 均值-方差模型——给定 N 种资产，在目标收益率下最小化风险（方差）。

**教学点**：
- 二次目标函数的几何直觉——有效前沿
- KKT 条件的直观解释
- 从确定到随机——蒙特卡洛模拟
- 鲁棒优化的动机——参数估计误差导致的组合偏移

**延伸**：
- 加入实际约束（不得卖空、行业集中度限制）
- CVaR 替代方差作为风险度量

**✅ 验证标准**：
1. 有效前沿单调递增：收益率越高，风险越大
2. 最小方差组合的权重和为 1（约束检查）
3. 蒙特卡洛模拟的预期收益率 ≈ 模型的目标收益率（误差 < 5%）

---

## 案例 4：服务系统排队优化（仿真）★★★☆☆

**模型**：排队论 / 离散事件仿真

**场景**：你管理一家医院的急诊科。病人到达率随时间变化（早高峰 vs 夜间的），不同严重程度的分诊级别需要不同的服务时间。你有 N 个医生，每个医生一次只能看一个病人。目标是：在给定医生数量下最小化病人平均等待时间，同时保证危重病人的等待时间不超过 X 分钟。

**教学点**：
- M/M/c 排队模型的直觉——λ（到达率）、μ（服务率）、c（服务台数）
- Little 定律：L = λW ——队列长度 = 到达率 × 等待时间（简单到可怕但极其强大）
- 仿真 vs 解析排队论——什么时候解析够用，什么时候必须仿真
- 到达率随时间变化 → 非平稳排队 → 仿真几乎是唯一选择

**延伸**：
- 加入医生排班（一天中医生数量可调）→ 两阶段优化
- 多优先级队列——危重病人插队机制
- 扩展到呼叫中心、银行柜台、服务器集群

**✅ 验证标准**：
1. M/M/1 特殊情况：仿真结果与解析解 L = λ/(μ-λ) 偏差 < 5%
2. Little 定律验证：仿真统计的 L / λ ≈ W（误差 < 1%）
3. 增加一个医生后平均等待时间下降——方向正确

---

## 案例 5：人员排班与约束规划★★★★☆ [进阶]

**模型**：约束规划（CP）/ 整数规划

**场景**：医院护士排班——50 名护士、7 天、每班次需要的人数不同。约束包括：每人每周最多夜班 3 次、每人至少休息 2 天、技能等级匹配、尽量公平。

**教学点**：
- 约束规划 vs 整数规划——什么情况下 CP 更好
- 软约束和硬约束——「尽量公平」怎么建模
- 对称性破坏——如何避免求解器在等价解上浪费时间
- 大邻域搜索（LNS）——当精确解不可行时

**延伸**：
- 加入员工偏好 → 多目标优化
- 实时排班 → 在线优化

**✅ 验证标准**：
1. 所有硬约束全部满足（夜班次数、休息天数、技能匹配）
2. 所有班次的人员需求被满足（每班人数≥下限）
3. 对称性破坏前后解的质量一致但求解时间下降

## 附录A：什么时候不需要运筹学

| 场景 | 理由 |
|------|------|
| 决策只涉及 3 个以下选择 | 直接穷举比建模快 |
| 数据不可靠/不可得 | "输入垃圾，输出垃圾" |
| 问题一周变三次 | 优化跟不上变化 |
| 业务方只接受简单方案 | 运筹学的解释成本太高 |

## 附录B：TOP 5 必踩坑

1. Big-M 取太大 → LP 松弛太松，分支定界膨胀
2. 忘记对称性破坏 → 求解器在等价解上浪费 90% 时间
3. 灵敏度分析跳过 → 数据稍微一变，最优解面目全非
4. 把业务限制硬编码进目标（而非约束）→ 权重调几个月
5. 一上来就用启发式 → 不知道离最优解差多远

## 附录C：运筹学 × ML 交叉点

| 结合方式 | 说明 | 典型应用 |
|---------|------|---------|
| ML 预测 → OR 决策 | 用 ML 预测不确定参数，输入优化模型 | 需求预测+库存优化 |
| OR 生成 → ML 特征 | 优化解作为 ML 的输入特征 | 排班方案→员工流失预测 |
| ML 替代 OR 求解器 | 用 NN 学习求解器的映射 | 端到端学习优化 |
| 端到端决策优化 | 决策层+预测层联合训练 | 价格弹性学习+定价优化 |
