# 附录 C：运筹学 × ML 的交叉点

> 全文约 800 字 | 阅读时间：4 分钟

---

## 为什么要关心这个交叉点

运筹学（OR）和机器学习（ML）以前是两拨人在做：OR 解决「在已知参数下怎么做最优决策」，ML 解决「从数据中学习未知的模式」。但现实世界需要两者结合——**用 ML 预测参数，用 OR 做决策**。

---

## C.1 预测 + 优化（Predict-then-Optimize）

**典型流程**：

```
数据 → ML 预测模型 → 预测参数 → OR 优化模型 → 决策方案
```

**例子**：
- **物流**：ML 预测各城市下周的需求量 → MIP 模型优化补货计划
- **能源**：LSTM 预测明天风力发电量 → 机组组合优化
- **零售**：XGBoost 预测每个 SKU 的销量 → 库存优化

**⚠️ 常见坑**：ML 的预测误差会被 OR 模型放大。一个需求预测误差 5%，可能导致采购决策误差 20%。解决方案：**端到端学习**（Decision-focused Learning），让 ML 的损失函数直接是最优决策损失，而不是预测精度。

---

## C.2 学习优化（Learning to Optimize）

**用 ML 加速求解器的搜索过程**：

| ML 应用 | 效果 |
|--------|------|
| 学习分支策略（Branching） | 替代求解器的手工分支规则，减少 30-50% 节点数 |
| 学习割平面选择 | 哪些割平面有效？ML 帮你选 |
| 学习初始解 | 给 MIP 求解器一个好的初始解，冷启动 → 热启动 |
| 学习启发式参数 | SA 的初始温度和冷却速率、GA 的交叉率——ML 自动调参 |

**代表工作**：Google 的「Learning to Branch」、DeepMind 的「Neural MIP Solver」。

---

## C.3 端到端决策优化（End-to-End Decision Optimization）

**传统方式**：

```
ML 预测 → 固定预测值 → OR 求解
```

**端到端方式**：

```
ML 模型（输出决策变量）→ 决策损失（包含约束和目标）
```

核心技术是**可微优化层**：让 OR 求解器变成一个可微分的「层」，嵌入到神经网络中。这样梯度就可以从最终决策损失传到 ML 模型的参数。

**代表工作**：
- OptNet（可微二次规划层）
- cvxpylayers（可微凸优化层）
- Deep Learning via MIP（用 MIP 作为神经网络的一层）

---

## C.4 强化学习 + 运筹学

RL 特别适合**序贯决策问题**——而运筹学中很多问题也是序贯的（库存、调度、路由）。

| 交叉方式 | 说明 |
|---------|------|
| RL 做求解策略 | 用 PPO / DQN 代替分支定界的启发式规则 |
| OR 做 RL 的环境模型 | RL agent 在运筹学仿真环境中训练 |
| 混合方法 | 先用 OR 求解器得到一个 baseline，RL 在 baseline 基础上做微调 |

**经典案例**：Google 用 RL 优化数据中心冷却——本质是一个运筹学问题（温度控制），用 RL 找到了比传统 PID 更好的策略。

---

## C.5 选择建议

| 你的场景 | 推荐交叉方向 |
|---------|------------|
| 有历史数据，需要做决策 | Predict-then-Optimize |
| 需要加速已有的求解器 | Learning to Optimize |
| 预测和决策深度耦合 | End-to-End Learning |
| 序贯环境、可交互 | RL + OR |

---

## 一句话总结

> **运筹学告诉你「给定参数怎么做最优决策」，机器学习告诉你「参数从哪里来」——两者结合才是完整的决策智能。**
