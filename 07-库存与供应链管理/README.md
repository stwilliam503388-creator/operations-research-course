# 库存与供应链管理基础课程

> 一个周末，从零到能看懂库存管理的核心模型、把供应链问题建模成优化问题、以及用 Python 求解补货策略与供应链协调方案。

## 这份教程是什么

这是一份给**有基础数学和 Python 能力、但对库存与供应链管理完全陌生**的技术人/供应链分析师/运营管理者写的自学教程。它不讲学院派的定理证明——每一章都围绕「这个库存模型解决什么业务问题」和「这个供应链协调机制为什么在这里好使」展开。

读完以后，你不一定会成为供应链总监，但你一定会：

- 跟同事说清楚「EOQ→报童→(s,S)→安全库存→牛鞭效应」的递进关系
- 看到一个库存/供应链问题，判断它适合用哪个模型、该不该做优化
- 用 Python + numpy 实现补货策略仿真，并做灵敏度分析
- 理解 VMI、CPFR、回购契约等供应链协调机制的核心逻辑
- 看懂一个供应链网络设计问题，知道怎么从设施选址到库存配置一步步求解

## 术语速查表

**核心术语**（先记住这 15 个，其余的以后查）

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 经济订货批量 | EOQ | 像找“每次进货多少最省钱”：订太勤花手续费，订太多又压库存 |
| 安全库存 | Safety Stock | 像家里多备一袋米：需求突然变多或到货晚了，也不至于断粮 |
| 牛鞭效应 | Bullwhip Effect | 像甩鞭子，越到尾端摆动越大：顾客小波动，到工厂可能变大震荡 |
| 报童问题 | Newsvendor Problem | 像早上订报纸：订多卖不掉，订少错过生意，要在浪费和缺货间找平衡 |
| 服务水平 | Service Level | 像承诺“不缺货”的概率，95% 就是 100 次里大约 95 次能满足需求 |
| 订货点 | Reorder Point (ROP) | 像油表红线：库存降到这个数，就该下单补货了 |
| (s,S) 策略 | (s,S) Policy | 像水箱补水：低于 s 就补，一口气补到 S |
| 批量折扣 | Quantity Discount | 买多单价便宜，但仓库和资金也被占住，所以不是越多越好 |
| 联合补货 | Joint Replenishment | 像几个人拼车分摊车费：多个产品一起订能省固定成本，但也可能多屯货 |
| 持有成本 | Holding Cost | 货放在仓库里不是免费的：占钱、占地、会过期、会损坏 |
| 缺货成本 | Stockout Cost | 没货带来的损失：客户走了、订单丢了、罚款来了、口碑变差 |
| 提前期 | Lead Time | 从下单到到货的等待时间；等得越久，越需要多准备库存垫着 |
| 供应商管理库存 | VMI | 像把补货管家交给供应商：你给销售数据，他来决定补多少 |
| CPFR | Collaborative Planning, Forecasting & Replenishment | 供需双方一起看预测、一起排补货计划，不再各猜各的 |
| 回购契约 | Buyback Contract | 供应商说“卖不掉我回收一部分”，零售商就更敢多订 |

**进阶术语**（用到时回来查）

| 中文 | English | 一句话人话 |
|------|---------|-----------|
| 收益共享契约 | Revenue Sharing Contract | 供应商先便宜供货，但之后要分一部分销售收入，大家一起赌销量 |
| 弹性订货契约 | Quantity Flexibility Contract | 先约个大概数量，等需求更清楚后允许你小范围加减 |
| 数量折扣契约 | Volume Discount Contract | 买得越多折扣越大，用低价诱导你多订 |
| 可选契约 | Option Contract | 像先买一张“订货门票”，以后需要时可以按约定价格拿货 |
| 多级库存 | Multi-Echelon Inventory | 不只一个仓库，而是工厂、区域仓、门店一串库存互相牵连 |
| 风险分担 | Risk Pooling | 把多个地方的需求合起来看，东边少西边多能互相抵消，整体更稳 |
| 延迟策略 | Postponement | 先做通用半成品，等订单来了再上最后颜色、包装或配置 |

### 概念边界：库存模型不是一个万能公式

库存管理最容易误解的一点是：**EOQ、报童、安全库存、(s,S) 解决的是不同问题，不能互相乱套。**

- EOQ 主要处理需求相对稳定时“每次订多少最省钱”。
- 报童问题处理一次性、卖不完会亏、订少会缺货的随机需求。
- 安全库存处理需求和提前期的不确定性，是为了降低缺货风险。
- (s,S) 策略是持续补货规则：低于 `s` 就补到 `S`。

真实项目还要先问预测准不准、提前期稳不稳、仓库容量够不够、缺货到底损失多少。
