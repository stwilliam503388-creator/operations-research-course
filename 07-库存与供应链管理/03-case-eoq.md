## 1 场景

你是一家制造企业的采购经理。公司每年需要某种标准零件 10,000 个，每个零件采购价 20 元。每次向供应商下单会产生固定订货成本 500 元（包含运输、质检、行政处理），每个零件每年的持有成本为 2 元（包含仓储费、资金占用、损耗）。你的目标是找到最优订货批量，使年总库存成本最小。

**教学点**：
- EOQ 公式的完整推导——年总成本 = 订货成本 + 持有成本
- 最优解的本质：边际订货成本 = 边际持有成本
- 灵敏度分析——K 或 h 变化对 Q* 和总成本的影响程度
- 与枚举法对比验证 EOQ 公式的正确性

---

## 2 库存建模（决策 / 成本 / 权衡）

| 要素 | 说明 |
|------|------|
| **参与者** | 企业（采购部门）与供应商之间的单方决策问题 |
| **企业策略** | 选择订货批量 Q（连续决策变量） |
| **供应商策略** | 被动响应：按订单交付，批发价 w 外生给定 |
| **企业收益** | 年总成本 TC(Q) = (D/Q)×K + (Q/2)×h，最小化 TC(Q) |
| **博弈性质** | 单方优化（非博弈），但可看作 Stackelberg 博弈中跟随者的最优反应 |

虽然 EOQ 传统上不是博弈问题，但其核心逻辑——在两种成本之间权衡——为后续博弈论契约设计（案例 4）奠定了基础。

## 3 数学模型（逐行注释式）

### 业务→模型映射表

| 业务要素 | 模型符号 | 数值 | 说明 |
|---------|---------|------|------|
| 年需求量 | D | 10,000 个 | 确定型需求，全年均匀 |
| 每次订货固定成本 | K | 500 元 | 运输+质检+行政，与 Q 无关 |
| 单位年持有成本 | h | 2 元/个/年 | 仓储+资金占用+损耗 |
| 采购单价 | w | 20 元 | 外生给定，不影响最优 Q* |
| 订货批量（决策变量） | Q | 待求 | 目标是最小化 TC(Q) |

### 模型公式

```
总成本 TC(Q) = 年订货次数 × K + 平均库存 × h
             = (D / Q) × K + (Q / 2) × h

一阶导数求极值：
  d(TC)/dQ = -D×K/Q² + h/2 = 0
  => D×K/Q² = h/2
  => Q* = sqrt(2×D×K / h)

代入数值：
  Q* = sqrt(2 × 10,000 × 500 / 2)
     = sqrt(5,000,000)
     ≈ 2,236.07 个

此时：
  年订货成本 = D/Q* × K = 10,000/2,236.07 × 500 ≈ 2,236.07 元
  年持有成本 = Q*/2 × h = 2,236.07/2 × 2 ≈ 2,236.07 元
  年总成本   = 4,472.14 元
```

**关键观察**：在最优解处，订货成本 = 持有成本。这是 EOQ 平衡条件的必然结果，也是验证 EOQ 计算是否正确的最简便方法。

## 4 模型分析

### 枚举法验证

为了验证 EOQ 公式的正确性，我们枚举 Q 从 500 到 5000 的所有整数值，计算对应的 TC(Q)，与公式结果对比：

| Q | 订货成本 | 持有成本 | 总成本 |
|---|---------|---------|-------|
| 1,000 | 5,000 | 1,000 | 6,000 |
| 1,500 | 3,333 | 1,500 | 4,833 |
| 2,000 | 2,500 | 2,000 | 4,500 |
| **2,236** | **2,236** | **2,236** | **4,472** ✅ |
| 2,500 | 2,000 | 2,500 | 4,500 |
| 3,000 | 1,667 | 3,000 | 4,667 |

枚举最优 Q = 2,236，与 EOQ 公式结果一致。

### 灵敏度分析

| 参数变化 | Q* 变化 | 总成本变化 |
|---------|---------|-----------|
| K↑20%（500→600） | +9.5%（2,236→2,449） | +0.83% |
| h↑20%（2→2.4） | -8.7%（2,236→2,041） | +0.83% |
| K 和 h 各偏 ±20% | - | 总成本变化 < 1% |

EOQ 对参数误差不敏感——这是 EOQ 模型最实用的性质之一。

## 5 代码实现概览

```python
import numpy as np

def eoq(D, K, h):
    """EOQ 公式计算最优订货批量"""
    return np.sqrt(2 * D * K / h)

def total_cost(D, K, h, Q):
    """年总成本 = 订货成本 + 持有成本"""
    order_cost = (D / Q) * K
    holding_cost = (Q / 2) * h
    return order_cost + holding_cost

def enumerate_optimal(D, K, h, Q_min=500, Q_max=5000):
    """枚举法验证 EOQ 公式"""
    best_Q = Q_min
    best_TC = float('inf')
    for Q in range(Q_min, Q_max + 1):
        tc = total_cost(D, K, h, Q)
        if tc < best_TC:
            best_TC = tc
            best_Q = Q
    return best_Q, best_TC
```

完整代码见 `code/case03_eoq.py`，包含：
- EOQ 公式计算 + 总成本验证
- 枚举法对比（确认公式最优 = 枚举最优）
- 灵敏度分析：K 和 h 变化对 Q* 和总成本的影响
- 总成本曲线可视化

## ✅ 验证标准

1. **公式验证**：EOQ = sqrt(2×10000×500/2) = sqrt(5,000,000) ≈ 2,236，与枚举最优一致
2. **平衡条件**：在 Q* 处，年订货成本 = 年持有成本 = 2,236 元
3. **灵敏度**：K 或 h 偏离 20%，总成本增幅 < 1%（EOQ 对参数误差不敏感）
4. **成本曲线**：Q 偏离 Q* 时总成本上升，且在 Q* 处曲线最平缓

## 7 关键洞察

- **EOQ 对参数误差高度不敏感**：K 或 h 偏离 20%，总成本只增加不到 1%。这意味着使用近似估计算出的批量也很接近最优。这是 EOQ 在实际中被广泛使用的重要原因。
- **订货成本 = 持有成本是最优的本质特征**：任何偏离这个平衡点的选择都意味着存在更优的决策。
- **EOQ 是决策方向指南，不是精确数字**：在实际中，考虑包装规格、运输最小起订量后，将 Q* 四舍五入到整数规格即可。没有必要追求"精确到个位"。
- **EOQ 假设需求恒定**：当需求波动较大时，需要转到报童模型（案例 2）或 (s,S) 策略。

---

## 8 延伸与变形

- **批量折扣（全量折扣 vs 增量折扣）**：供应商给出阶梯价格时，需要分段计算 EOQ 并检查可行性
- **有限生产率模型（EPQ）**：一边生产一边消耗，Q* = sqrt(2×D×K / (h×(1 - D/P)))
- **考虑缺货允许（backorder）的 EOQ**：允许偶尔缺货时，最优批量会变化
- **多产品联合补货**：多种产品共享运输资源时，一次订多种可以分摊固定成本
- **概率需求下的 (s,S) 策略**：EOQ 的推广——当需求随机时，用 (s,S) 策略替代固定批量
