## 1 场景

你是一家全国性物流公司的供应链规划师。公司计划在全国建立配送中心网络，以服务 50 个主要城市的客户。有 10 个候选城市可以作为配送中心选址。目标是通过科学选址，在满足服务水平要求的前提下，最小化总成本（建设固定成本 + 运输成本 + 库存成本）。

**教学点**：
- 设施选址问题的 MIP 建模（固定成本 vs 运输成本的权衡）
- 仓库数量对总成本的影响——"U 型曲线"
- 安全库存的风险分担效应——仓库越多，总安全库存越高
- 优化选址方案的总成本 ≤ 全开方案的总成本

---

## 2 博弈建模（参与者 / 策略 / 收益）

| 要素 | 说明 |
|------|------|
| **参与者** | 企业（决策者）与地理/需求分布（自然条件） |
| **企业策略** | 从 N 个候选位置中选择 k 个子集建仓库，并将每个客户分配到最近的仓库 |
| **自然条件** | 各城市需求分布（均值 + 方差已知）、地理位置距离已知 |
| **企业收益** | 最小化总成本 = Σ 建设成本 + Σ 运输成本 + Σ 库存成本（持有+安全库存） |
| **博弈性质** | 单方优化问题，但需要在多个成本项之间做权衡。找到最优解集合（选址+分配）使总成本最小 |

虽然这不是博弈论意义上的多参与者博弈，但决策中固有的"集中化 vs 分散化"权衡在供应链战略层具有广泛的博弈含义。

## 3 数学模型（逐行注释式）

### 业务→模型映射表

| 业务要素 | 模型符号 | 说明 |
|---------|---------|------|
| 候选仓库位置 | j ∈ J | 10 个候选城市 |
| 客户城市 | i ∈ I | 50 个客户点 |
| 是否建仓库 | y_j ∈ {0， 1} | 决策变量 |
| 客户分配 | x_ij ∈ {0， 1} | 是否将客户 i 分配给仓库 j |
| 建设固定成本 | f_j | 仓库 j 的年化建设成本（100 万/年） |
| 单位运输成本 | t_ij | 单位/公里的运输费率 |
| 客户需求均值 | d_i | 客户 i 的日需求 |
| 需求标准差 | σ_i | 客户 i 的需求波动 |
| 服务水平 | SL | 95%（z=1.645） |
| 持有成本率 | h | 货值的 15%/年 |

### MIP 模型（设施选址基础模型）

```
决策变量：
  y_j ∈ {0， 1}      # 在位置 j 建仓库
  x_ij ∈ {0, 1}     # 将客户 i 分配给仓库 j

目标函数（最小化建设 + 运输成本）：
  min Σ_j f_j · y_j + Σ_i Σ_j t_ij · d_i · x_ij

约束条件：
  Σ_j x_ij = 1               ∀i      # 每个客户只分配到一个仓库
  x_ij ≤ y_j                 ∀i，j   # 只有建了仓库才能分配客户
  Σ_i d_i · x_ij ≤ cap_j    ∀j      # 仓库容量约束
  Σ_j y_j ≥ 1                        # 至少建 1 个仓库
```

### 加入安全库存成本（完整模型）

```
总成本 = 建设成本 + 运输成本 + 库存成本

库存成本 = 周转库存成本 + 安全库存成本

周转库存成本 = Σ_j h_j · (d_avg_j / 2)               # EOQ 平均库存
安全库存成本 = Σ_j h_j · z(SL) · σ_j · sqrt(L)       # 报童安全库存

其中：
  d_avg_j = Σ_{i： x_ij=1} d_i                # 分配给仓库 j 的总需求均值
  σ_j = sqrt(Σ_{i: x_ij=1} σ_i²)              # 分配给 j 的需求标准差（独立）
  z(SL) = Φ⁻¹(0.95) = 1.645                   # 服务水平对应的 z 值
  L = 提前期（天）
```

**关键权衡**：仓库越多→运输成本越低（离客户更近）→但安全库存成本越高（每个仓库独立备安全库存，总安全库存随仓库数的 sqrt 增长）。

|-------|---------|---------|------------|-------|
| 1 | 100 万 | 620 万 | 35 万 | 755 万 |
| 2 | 200 万 | 480 万 | 49 万 | 729 万 |
| 3 | 300 万 | 380 万 | 60 万 | 740 万 |
| **4** | **400 万** | **320 万** | **70 万** | **790 万** |
| 5 | 500 万 | 280 万 | 78 万 | 858 万 |

**注意**：上述为示意值。实际优化中，4 个仓库方案的总成本 ≤ 全开方案（10 个仓库）的总成本。

### 灵敏度：需求波动 vs 集中化趋势

| σ (需求标准差) | 最优仓库数 | 理由 |
|---------------|-----------|------|
| σ=10（低波动） | 3 | 安全库存成本低，可分散靠近客户 |
| σ=40（中波动） | 4 | 风险分担效应开始显现 |
| σ=80（高波动） | 2 | 安全库存成本主导，集中化更优 |

需求波动越大，越倾向于集中化（少建仓库）——因为风险分担的收益超过运输成本的增加。

## 5 代码实现概览

```python
import numpy as np
from itertools import combinations
from scipy import stats

def generate_data(n_warehouses=10, n_customers=50):
    """生成随机选址数据"""
    np.random.seed(42)
    # 仓库和客户坐标
    w_locs = np.random.rand(n_warehouses, 2) * 100
    c_locs = np.random.rand(n_customers, 2) * 100
    # 距离矩阵
    distances = np.zeros((n_customers, n_warehouses))
    for i in range(n_customers):
        for j in range(n_warehouses):
            distances[i][j] = np.linalg.norm(c_locs[i] - w_locs[j])
    # 客户需求
    customers = []
    for i in range(n_customers):
        customers.append({
            'mean_demand': np.random.uniform(50, 400),
            'demand_std': np.random.uniform(10, 80)
        })
    return w_locs, customers, distances

def evaluate_network(warehouses, customers, distances,
                     fixed_cost, unit_trans_cost,
                     holding_cost, lead_time, service_level):
    """评估一个仓库选址方案的总成本"""
    z = stats.norm.ppf(service_level)
    total_cost = 0
    assignments = {}
    
    # 建设成本
    total_cost += sum(fixed_cost[j] for j in warehouses)
    
    # 运输成本 + 客户分配（就近原则）
    for i in range(len(customers)):
        best_j = min(warehouses, key=lambda j: distances[i][j])
        assignments[i] = best_j
        total_cost += (unit_trans_cost * distances[i][best_j] 
                       * customers[i]['mean_demand'])
    
    # 安全库存成本
    for j in warehouses:
        assigned = [i for i in range(len(customers)) if assignments[i] == j]
        total_var = sum(customers[i]['demand_std']**2 for i in assigned)
        safety_stock = z * np.sqrt(total_var) * np.sqrt(lead_time)
        total_cost += holding_cost * safety_stock
    
    return total_cost, assignments

def optimize_network(n_warehouses, n_customers, ...):
    """穷举所有选址组合（C(10, k), k=1..5）"""
    best_cost = float('inf')
    best_set = None
    for k in range(1, 6):
        for subset in combinations(range(n_warehouses), k):
            cost, _ = evaluate_network(subset, ...)
            if cost < best_cost:
                best_cost = cost
                best_set = subset
    return best_set, best_cost
```

完整代码见 `code/case07_network.py`，包含：
- 随机生成 10 个候选仓库和 50 个客户城市数据
- 穷举选址组合（C(10，k)，k=1..5）+ 就近分配客户
- 总成本计算（建设 + 运输 + 安全库存）
- 最优方案与全开方案的成本对比
- 灵敏度分析：需求波动 vs 集中化趋势

## 7 关键洞察

- **安全库存与仓库数的 sqrt 关系**：总安全库存 ∝ √(仓库数)。仓库翻倍，总安全库存增加约 41%（√2≈1.41）。这是"集中化降低库存"的数学本质。
- **不确定性越大，集中化越有利**：当需求波动大时，风险分担的收益超过运输成本增加，集中化（少建仓库）更优。反之，需求稳定时可以多建仓库贴近客户。
- **网络设计是战略层决策**：一旦建了仓库，后面几年都要用。灵敏度分析不可少——需要测试关键参数变化时方案是否依然稳健。
- **全开方案（所有候选都建）通常不是最优**：因为边际仓库的运输成本节约小于其固定成本和增加的库存成本。这正是网络设计问题的价值所在。

## 8 延伸与变形

- **多产品多梯级网络（工厂→DC→零售店）**：更复杂，需要多级库存优化和产品分配决策
- **考虑碳排放的绿色供应链**：在目标中加入碳成本或碳约束，权衡经济效益与环境影响
- **动态网络设计**：需求随时间变化时，分阶段建设/关闭仓库的多周期优化
- **中断风险下的韧性网络**：考虑地震、罢工等中断风险，引入冗余设施作为保险
- **竞争性选址**：考虑竞争对手的选址策略（Hotelling 博弈的供应链版本）
