# 附录B：TOP 5 必踩坑

这些坑你大概率会踩——提前知道，就少走弯路。

---

## 🕳️ 坑 1：假设需求恒定

**现象**：拿 EOQ 直接套用在所有场景上，不管需求有没有趋势和季节性。

EOQ 很好用，但前提是需求稳定。季节性（双十一、春节）、促销期（降价 30% 的销量翻 3 倍）、新品上市（无历史数据）——这些场景的需求不是恒定的。在这些场景用 EOQ，要么缺货、要么积压。

**根本原因**：EOQ 是最简单的模型，所以最容易默认使用。初学者容易认为「需求恒定」是一个无害的假设——直到库存积压或缺货才发现问题。

**怎么办**：先看需求曲线。画一张简单的时间序列图。如果有明显趋势或季节波动，用 (s,S) 或动态规划——别偷懒用 EOQ。如果难以判断，可以用变异系数（CV = σ/μ）衡量：CV < 0.2 可以用 EOQ，CV > 0.5 必须用随机模型。

---

## 🕳️ 坑 2：忽略提前期的组成部分

**现象**：把提前期当成一个简单的数字，结果安全库存总是偏少。

「今天订明天到」和「今天订三个月后到」的安全库存天差地别。提前期每增加一天，安全库存就要加一天的需求量。但问题在于：很多人把提前期只算成「供应商说的时间」，忽略了内部流程。

**提前期的真正构成**：

```
实际提前期 = 订单审批时间 + 供应商生产时间 + 运输时间 + 清关时间 + 质检时间 + 入库上架时间
```

每个环节都可能延迟。如果供应商说 7 天，但审批就要 2 天、质检又拖了 3 天——你的实际提前期是 12 天，不是 7 天。

**怎么办**：用历史数据的 P95 或 P99 提前期，而不是平均值。把提前期的方差也纳入安全库存计算——提前期不确定性的影响和需求不确定性一样大。

---

## 🕳️ 坑 3：服务水平设太高

**现象**：管理层说「我们要 99% 服务水平」，库存成本翻了一倍。

95% 到 99% 听起来只差 4%，但库存可能要多 50%。最后那 1% 是最贵的。为什么？安全库存公式中的 z 值：

```
95% 服务水平 → z = 1.645
99% 服务水平 → z = 2.326
安全库存比例 = 2.326 / 1.645 = 1.41（多 41%）
```

如果需求波动大、提前期长，这个差距会更大。

**怎么办**：先算算达到 99% 服务水平需要多花多少钱，再跟业务方确认——他们可能并不知道「99%」意味着什么。一个实用的方法：画出「服务水平 vs 总成本」曲线，让管理层自己看拐点在哪里。通常 95%-97% 是性价比最高的区间。

**另外**：不同 SKU 应该有不同的服务水平目标。A 类（高价值/高销量）可以设 99%，C 类（低价值/低销量）设 90% 就够了。

---

## 🕳️ 坑 4：独立优化各级库存

**现象**：每个部门/每个节点都在追求自己的「最优」，结果整个供应链更糟了。

零售商优化自己 → 批发商优化自己 → 加起来就是牛鞭效应。每级的「最优」加在一起不等于全局最优。这种「局部理性导致全局非理性」的现象在供应链中极为普遍。

**一个典型场景**：
- 零售商为了省运费，把订单从每周 1 次改成每 2 周 1 次
- 零售商的库存成本下降了 10%（省了运费）
- 批发商看到订单波动变大，多备了 20% 的安全库存
- 批发商加了价 → 零售商进货成本上升 → 最终零售商的总成本反而上升了

**怎么办**：
1. 至少做信息共享——共享 POS 数据、库存水平、产能数据
2. 最好的情况是供应链契约协调——让每级的激励与全局一致（关于契约设计，见第 2.4 节）
3. 如果无法实现完全的契约协调，至少做联合绩效评估（不再只看各自的 KPI，而是看供应链整体利润）

---

## 🕳️ 坑 5：忘记数据质量

**现象**：花大量时间优化模型，但用的数据是手工录入的 Excel 表。

「输入垃圾，输出垃圾」——库存模型的精度不会超过需求预测的精度。如果你用的需求数据是手工录入的 Excel，手工记录经常漏记、错记、延期记——模型再精也没用。

**更隐蔽的版本**：
- 需求数据包含了缺货期间的「虚假低需求」（因为没货，所以没卖出去——但实际需求是高的）
- 库存数据只记录了账面库存，没有减掉损耗和丢失
- 提前期数据是供应商承诺的时间，不是实际发生的时间

**怎么办**：花 80% 的精力在数据清洗和预测上，20% 在模型上。具体来说：
1. 先做数据审计——确认数据来源、采集频率、记录准确性
2. 处理缺失值和异常值
3. 矫正缺货期间的需求（使用填充估算方法）
4. 再谈建模

**一个实用原则**：如果你花在数据清洗上的时间少于花在模型调参上的时间，你很可能做反了。
