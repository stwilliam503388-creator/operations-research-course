## 2.1 供应链结构：推式 vs 拉式

```
推式（Push）：
  生产 → 库存 → 销售
  「我猜你会买，所以我先生产」
  
拉式（Pull）：
  订单 → 生产 → 交付
  「你买了，我才生产」
```

| 维度 | 推式 | 拉式 |
|------|------|------|
| 需求不确定性 | 低（可以预测） | 高（难以预测） |
| 提前期 | 长 | 短 |
| 规模经济 | 好（批量生产） | 差（按单制造） |
| 库存风险 | 高（可能积压） | 低（零库存） |
| 客户响应速度 | 快（现货） | 慢（生产周期） |

**现实**：大部分供应链是「推拉结合」——前端按库存生产（推），后端按订单组装（拉）。例如 Dell 的直销模式：零件按预测备货（推），最终组装按订单（拉）。

**如何判断你的供应链适合推还是拉？** 一个简单的标准：如果客户愿意等，用拉式；如果客户不愿意等，用推式。但现实中客户既不愿意等又想要定制——这就是供应链设计的根本挑战。

## 2.2 牛鞭效应的数学直觉

### 为什么需求波动会被放大？

假设 4 级供应链：零售商 → 批发商 → 分销商 → 工厂。

每级在做各自的库存决策。每一级都试图从历史订单中「推断」未来需求——但每个推断都加入了安全库存。

**数学直觉**：
- 零售端需求是 \$ D_t = \mu + \epsilon_t \$（均值 + 随机波动）
- 零售商使用移动平均预测，并加安全库存 → 订单波动 > 需求波动
- 批发商看到零售商的订单（已放大），再次预测、再次加安全库存 → 进一步放大
- 到工厂级别的订单波动可能是原始需求的 5-10 倍

### 一个简化的数值例子

```
零售端需求（4 周）：[100, 110, 105, 95]  → 标准差 = 6.5
零售商订单（4 周）：[110, 125, 115, 85]  → 标准差 = 17.1  （放大 2.6x）
批发商订单（4 周）：[120, 140, 125, 75]  → 标准差 = 27.0  （放大 4.2x）
分销商订单（4 周）：[130, 155, 135, 65]  → 标准差 = 37.0  （放大 5.7x）
```

即使每级只加了少量的安全库存（比如 10%），多级放大后效果显著。

### 牛鞭效应的量化指标

```
放大系数 ≈ 工厂端订单标准差 / 零售端需求标准差
```

实验中可以观察到，即使需求是简单的独立同分布，放大系数也 > 1。

### 缓解牛鞭效应的四个策略

1. **信息共享**：零售商实时共享销售数据（POS 数据），避免上下游仅靠订单推断。这是最有效也最便宜的策略。
2. **减少提前期**：提前期越短，安全库存需求越小，放大效应越低。可以通过本地化生产、快速运输实现。
3. **批量缩减**：EOQ 越大的企业越容易放大波动——考虑小批量多批次。虽然单次运输成本上升，但整体供应链波动下降。
4. **价格稳定**：促销和批量折扣会人为制造需求波动。天天低价（EDLP）策略可以从源头消除牛鞭效应。

## 2.3 信息共享与协同

### VMI（供应商管理库存）

供应商替零售商管理库存——供应商能直接看到零售商的库存水平和销售数据。

```
传统模式：零售商→下订单→供应商发货
VMI 模式：零售商→共享数据→供应商决定补货量和时间
```

**优势**：
- 供应商有全局视野，减少牛鞭效应
- 零售商省去采购管理成本
- 供应商可以优化自己的生产计划

**风险**：
- 供应商可能过度补货（多卖货）——需要设计激励机制
- 零售商需要信任供应商的数据使用权——合同约束
- 供应商对零售商的促销活动不敏感——可能漏掉配合

**VMI 在谁身上成功？** 沃尔玛 + 宝洁（P&G）是经典案例——1980 年代开始合作，至今仍是教科书级别的 VMI 实践。

### CPFR（协同规划、预测与补货）

比 VMI 更进一步——供需双方共享预测和补货计划，联合制定。

```
步骤 1：联合需求预测
步骤 2：联合产能/库存规划
步骤 3：联合补货执行
步骤 4：联合偏差评估
```

**适用场景**：零售-供应商长期合作关系，双方都有 IT 能力。CPFR 需要投入的 IT 基础设施和信任成本比 VMI 更高，但收益也更显著。

**VMI vs CPFR 的选择**：如果你的业务关系是新建立的，先从 VMI 开始。当 VMI 运行稳定且双方互信建立后，再逐步升级到 CPFR。

## 2.4 供应链契约设计

供应链契约的目标：让每个节点在追求自身利益最大化的同时，实现供应链整体最优。

### 双重边际效应（Double Marginalization）

```
制造商成本 = c → 出厂价 w → 零售商售价 p → 市场需求 D(p)
```

如果制造商和零售商各自最大化自己的利润，总利润 < 整合供应链的最优利润。这就是双重边际效应——两层都在加价，结果总利润更低。

**数值例子**：
- 整合供应链：最优定价 p* = 15，总利润 = 500
- 分开决策：制造商定价 w = 12，零售商定价 p = 18，制造商利润 = 300，零售商利润 = 150，总利润 = 450
- 效率损失 = (500 - 450)/500 = 10%

契约的作用就是弥补这 10% 的损失。

### 四种核心契约

| 契约类型 | 核心机制 | 协调效果 | 实践复杂度 |
|---------|---------|---------|-----------|
| **批发价契约** | 零售商按 w 进货，承担所有风险 | 不协调——双重边际效应存在 | 简单 |
| **回购契约** | 供应商以 b < w 回购未售商品 | 协调——零售商风险降低，订更多 | 需要处理退货物流 |
| **收益共享契约** | 供应商以低价 w' 供货，分享 φ 比例的收入 | 协调——等价于回购契约 | 需要监控销售额 |
| **数量弹性契约** | 零售商可调增/减订货量一定比例 | 部分协调——需求不确定性分担 | 需要约定弹性范围 |

**关键结论**：好的契约设计可以让零售商的订货决策等价于整合供应链下的最优决策。

**实操建议**：
- 如果退货物流成本低（如电子产品），选回购契约
- 如果退货物流成本高（如重工业设备），选收益共享契约
- 如果需求高度不确定，选数量弹性契约

## 2.5 供应链网络设计（选址-库存-运输联合优化）

### 三要素权衡

```
选址决策（建几个仓、在哪建）
   → 影响运输成本（离客户近 = 运费低）
   → 影响库存成本（仓越多 = 总安全库存越多）
   → 影响服务水平（仓越多 = 响应越快）

运输决策（配送路线、频率）
   → 影响库存（快运减少安全库存）
   → 影响成本（快运更贵）

库存决策（各级库存策略）
   → 影响选址（集中库存 vs 分散库存）
   → 影响运输（批量 vs 频率）
```

### 一个具体的权衡：集中 vs 分散

假设你有两个市场，需求独立同分布 N(100, 20²)：
- **两个分开的仓库**：总安全库存 = 2 × z × 20 = 40z
- **一个集中仓库**：总需求分布 N(200, (20√2)²)，安全库存 = z × 20√2 ≈ 28z

集中库存节省了 30% 的安全库存（风险分担效应），但增加了运输距离和成本。

### 典型建模框架

```python
# 概念框架（非可运行代码）
决策变量:
  y_j ∈ {0,1}      # 是否在位置 j 建仓库
  x_ij ≥ 0         # 从仓库 j 运到客户 i 的量
  S_j ≥ 0          # 仓库 j 的安全库存水平

目标:
  min  运输成本(x) + 固定成本(y) + 库存成本(S)

约束:
  - 每个客户被分配到一个仓库
  - 仓库产能限制
  - 库存平衡
  - 服务水平约束（安全库存 ≥ z·σ·sqrt(L)）
```

这种联合优化在 OR 中是一个 NP-hard 的 MIP，实际中常采用启发式（先定选址，再定库存，再调运输）或 Benders 分解。

**现实中的简化**：绝大部分企业不会做完整的联合优化。标准做法是：
1. 先做选址（用 MIP 或启发式）
2. 然后分配客户到仓库
3. 最后对每个仓库独立优化库存策略

虽然这种分步做法不是全局最优，但在实践中已经能带来显著改进。
